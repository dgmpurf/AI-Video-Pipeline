## 5. 每个文件的用途说明

### 5.1 `AI视频制作_模仿参考库总体设计_V0.1.md`

用途：总纲。它定义：

```text
模仿参考库是什么 / 不是什么
raw_clip / ref_segment / reference package
三层数据结构：资源元数据 / 模仿语法 / 使用记忆
usage_class 与 active_input_allowed
标签体系 L1/L2/L3
搜索体系：facets + FTS5 + semantic later + feedback ranking
context_key 与反馈权重
自动标注与人工确认
Reference Duty
AE/PR 后期特效执行归属
本地文件夹结构
数据库核心表
GUI / Workbench 方向
Codex / Agent 边界
MVP 与未来路线
```

适合：作为模仿参考库 Source 候选，以及后续所有子文件的总依据。

### 5.2 `02_AI视频制作_模仿参考库标签字典_V0.1.csv`

用途：受控双语标签词表。它用于：

```text
SQLite tags 表
tag_aliases 表
GUI facet filter
FTS5 同义词 / alias 搜索
Prompt Compiler 标签映射
Reference Duty Planner 参考职责生成
反馈权重统计
```

使用建议：

```text
全部视频初标只需 L1/L2。
高价值片段再补 L3。
active_input_allowed 不应只由标签决定，必须片段级人工审核。
```

### 5.3 `02_AI视频制作_模仿参考库标签字典_V0.1_说明.md`

用途：解释标签 CSV。它说明标签总数、字段含义、哪些字段用于精确筛选 / 全文搜索 / 语义搜索 / 反馈权重，以及如何处理 active input 默认值。

### 5.4 `03_AI视频制作_模仿参考库数据库Schema_V0.1.sql`

用途：SQLite 建库脚本。它用于创建原始视频、参考片段、标签、自动标注、媒体派生产物、caption、embedding、上下文、检索记录、生成记录、反馈事件和 FTS5 等表。

### 5.5 `03_AI视频制作_模仿参考库数据库Schema_V0.1_说明.md`

用途：解释数据库 Schema。它说明：

```text
视频文件放本地文件夹，不塞进数据库。
SQLite 只存路径、元数据、标签、搜索文本、安全状态、使用反馈。
搜索单位以 ref_segment 为主。
active_input_allowed 默认 false。
外部素材默认 offline_grammar_only。
FTS5 负责第一版搜索引擎。
Context + feedback 负责后续推荐排序。
```

### 5.6 `03_AI视频制作_模仿参考库_empty_refs_with_tags_V0.1.sqlite`

用途：已初始化的空 SQLite 模板库。适合直接复制为：

```text
reference_library/db/refs.sqlite
```

注意：不建议提交进普通 Git；可以本地备份或作为私有工具包保存。

### 5.7 `04_AI视频制作_模仿参考库ReferenceDuty与安全规则_V0.1.md`

用途：安全边界与参考职责规则。它定义：

```text

active_input_allowed 判断表

Reference Duty 通用结构
ACTION / CAMERA / FX / EDIT / PERF / COMP / STYLE / ENV / SOUND 模板

Prompt Compiler 对接规则
Review Recorder 对接规则
Feedback 更新规则
```

这是最适合作为 Source 候选的文件之一。

### 5.8 `05_AI视频制作_模仿参考库GUI与人工标注流程_V0.1.md`

用途：本地工作台与人工流程设计。它定义：

### 5.9 `06_Codex_L1_REF_LIBRARY_INDEXER_SPEC_V0.1.md`

## 6. V0.1 包的推荐执行顺序

### Phase A：人工确认文档包

```text
1. 确认 00 文件清单
2. 确认 01 总体设计
3. 确认 04 安全规则
4. 确认 05 GUI/标注流程
5. 确认 02 标签字典
6. 确认 03 Schema
7. 确认 06 Codex L1 Spec
```

目标：确认这套包的方向没有跑偏。

### Phase B：建立本地目录

```text
reference_library/
  raw_clips/
  ref_segments/
  derived/
  db/
  manifests/
  logs/
  config/
```

目标：先把容器搭好，不急着导入全部 3000–4000 条。

### Phase C：小样本测试

先放入 5–20 条测试视频。测试：

```text
ffprobe metadata
poster / contact sheet
默认 full-length ref_segment
SQLite 写入
FTS5 搜索
CSV export
refsearch CLI
```

目标：验证 Schema 和工具逻辑。

### Phase D：批量导入

```text
导入 3000–4000 条 raw_clip metadata
生成 poster / contact sheet
先不全量深标
```

目标：先做到可搜索、可预览、可粗标。

### Phase E：人工粗标

每条只粗标：

```text
source_type
primary_ref_type
usage_class
risk_level
primary_learning
L1/L2 tags
```

目标：让库从文件夹变成可检索数据库。

### Phase F：高价值片段深标

只对高价值 ref_segment 补：

```text
L3 tags
prompt_words
reference_duty
do_not_copy
review_criteria
post_plan
edit_timeline_note
quality_score
```

目标：产出 prompt_ready 参考。

### Phase G：真实项目反馈闭环

每次使用参考后记录：

```text
attempt_id
project_name
shot_id
context_key
selected_ref_ids
usage_mode
result_status
success_type
failure_type
human_note
```

目标：让库开始学习什么参考在什么场景里真的有效。
