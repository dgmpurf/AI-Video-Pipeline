# 0. 本次更新结论

本次更新不是新增某个具体 prompt，而是补充一种新的场景资产生产规则：

```text
当镜头功能从“远景对峙”切换到“中场交锋 / 战斗 staging”时，
不能默认沿用上一镜头的 A 级背景图。
```

在《赤焰对天穹》SHOT-02 中，直接沿用 SHOT-01 的古寺庭院背景锚点后，虽然角色进入了战斗姿态，但画面仍然没有落在用户期望的“主殿正前方中场战斗区”上。问题本质不是角色 prompt 不够强，而是：

```text
战斗镜头缺少新的场景锚点。
```

因此，新增核心规则：

```text
1. 战斗镜头如果改变空间重心，必须先判断是否需要新的 A 级背景锚点。
2. 远景对峙背景不一定适合近身交锋。
3. 当目标是“主殿前中场战斗”，需要单独生成主殿正面 / 中轴 / 中场可用的空场图。
4. 如果生成图仍沿用旧斜侧机位，应启动“视角纠偏 / 机位纠偏”流程。
5. 视角纠偏不是普通续修，而是同一世界连续性下的新机位重构。
```

# 1. 问题表现

失败图表现为：

```text
1. 战斗动作出现了，但交锋位置不符合预期。
2. 角色像被放在旧场景斜角空间里摆 pose。
3. 主殿正面、中轴、石阶前中场区域不够明确。
4. 背景仍然服务“远景对峙”，不服务“主殿前战斗舞台”。
5. prompt 写了近身交锋，但没有足够强的场景锚点支撑角色落位。
```

本质问题：

```text
这是“场景锚点不适配镜头功能”的问题，不是单纯战斗 prompt 问题。
```

---

# 2. 规则 A：镜头功能变化时，先检查背景锚点是否需要重做

## 2.1 适用情况

当镜头从以下类型切换时，应检查是否需要新的 A-SC 场景锚点：

```text
远景对峙 → 近身交锋
远处观察 → 中场动作
空场建立 → 主体冲突
侧面庭院 → 正面主殿
斜角空间 → 中轴战斗舞台
大空场气氛图 → 角色落脚/交锋图
```

## 2.2 判断标准

如果用户想要的是：

```text
主殿正前方
主殿中轴
石阶前方
中央战斗区
两名角色在中场相遇
某一建筑正面成为背景主体
```

则不能只沿用远景对峙用背景图。应先生成新的：

```text
front_combat_stage
main_hall_front_stage
frontal_axis_stage
combat_stage_anchor
```

# 3. 规则 B：视角纠偏 / 机位纠偏流程

## 3.1 什么时候启动

当已有生成图满足：

```text
世界观正确
美术风格正确
场景元素正确
雨雾 / 光影 / 材质正确
```

但仍存在：

```text
主殿不够正
建筑主体偏斜
中轴不清
战斗区偏到错误位置
镜头仍然沿用旧斜侧视角
```

就不应继续只写“保留场景风格”，而应明确启动：

```text
视角纠偏 / 机位纠偏 / 构图重定向
```

## 3.2 核心表达

```text
这不是普通续修。
这不是简单延续当前构图。
这是基于当前图进行视角纠偏。
目标是生成更正面、更中轴、更适合作为主殿前战斗舞台的新版本。
```

# 4. 规则 C：使用当前错误但接近目标的图作为 primary reference

如果 A-SC-002 虽然仍不够正，但已经比 A-SC-001 更接近目标，则下一轮不应继续只用 A-SC-001，而应使用 A-SC-002 作为 primary reference。

推荐逻辑：

```text
A-SC-001：世界连续性参考
A-SC-002：接近目标但机位仍偏斜的中间版本
A-SC-003：基于 A-SC-002 做视角纠偏后的正面中轴战斗舞台
```

执行时：

```text
使用 A-SC-002 作为 image2image 主输入；
prompt 明确说明目标不是复制 A-SC-002，而是纠偏为更正面的主殿中轴版本。
```

# 5. 视角纠偏 Prompt 模块

## 5.1 任务定义模块

```text
图1 作为当前古寺庭院场景版本参考。基于图1的世界连续性、美术风格、建筑结构、雨雾氛围、湿石板地、主殿、回廊、经幡和积水反光，重新生成一张新的古寺庭院空场图。

注意：这次不是简单延续图1的当前斜侧机位，而是要在保留同一场景连续性的前提下，把镜头调整成更正面对主殿的版本。可以理解为：基于图1，重新生成一张“主殿正视 / 中轴更清楚 / 更适合作为主殿前交锋舞台”的空场图。
```

## 5.2 核心目标模块

```text
这张图要比图1明显更正面地面对主殿。主殿是画面中央背景主体，主殿正门、石阶、石栏杆和正门前中轴区域必须更清楚、更完整、更正地呈现出来。镜头应尽量回到主殿正门前方的中轴视角，而不是继续保持图1那种明显斜侧观看主殿的角度。
```

## 5.3 中场战斗区模块

```text
画面中央中景必须形成一个清楚、开阔、可用的主殿前中央战斗舞台。主殿前的中景湿石板区域，是未来两名角色近身交锋的主要落点区域。画面必须明确服务于“主殿前中场交锋”。
```

## 5.4 反旧机位模块

```text
不要继续沿用图1那种明显斜侧观看主殿的角度。不要让主殿仍然明显偏斜。不要让屋檐透视主导画面。不要让左右建筑边角抢走主殿主体感。要让主殿正门前中轴区域更居中、更正、更稳定。整体效果应接近“主殿正面空场舞台图”，而不是“斜侧空场图”。
```

# 6. 禁止项

视角纠偏场景锚点仍然是场景资产，不是最终 keyframe。

必须禁止：

```text
不要人物
不要角色
不要焚狩
不要霜玑
不要战斗
不要人形剪影
不要黑红男性轮廓
不要银蓝女性轮廓
不要动物
不要僧人
不要香客
不要守卫
不要伞下人形
不要雕像人物
不要壁画人物
不要远处人影
不要武器作为焦点
不要文字
不要水印
不要角色设定卡
不要拼贴图
不要建筑细节特写
不要斜角落视角遮挡主殿
不要只有空地而没有主殿背景锚点
不要改变为其他场景
```

# 8. 生产流程建议

推荐流程：

```text
1. 发现 SHOT-02-KF-001 战斗关键帧空间失败。
2. 标记 SHOT-02-KF-001 为 historical_failed_composition_example。
3. 生成 A-SC-TEMPLE-COURTYARD-002_main_hall_front_combat_stage。
4. 如果 A-SC-002 仍偏斜，则不进入 SHOT-02-KF-002。
5. 使用 A-SC-002 作为 primary reference，生成 A-SC-TEMPLE-COURTYARD-003_main_hall_frontal_axis_stage。
6. 人工审核 A-SC-003。
7. 通过后锁定为 SHOT-02 战斗场景锚点。
8. 再生成 SHOT-02-KF-002。
```

# 9. 证据等级与升级条件

当前等级：B 级暂行规则。

原因：

```text
本项目已经出现明确失败：沿用旧远景对峙场景锚点导致战斗 staging 错位。
解决方向已经明确：新增主殿正面中轴战斗锚点，并进一步做视角纠偏。
但 A-SC-003 和 SHOT-02-KF-002 尚未完成最终验证。
```

升级为 A 级稳定规则需满足：

```text
1. A-SC-003 能成功生成更正面的主殿中轴空场图。
2. SHOT-02-KF-002 使用 A-SC-003 后，战斗落位明显改善。
3. 至少一次后续战斗镜头复用该规则有效。
4. 没有明显副作用，如场景跑偏、人物误生、主殿丢失。
```

# 10. 一句话总结

```text
当镜头目标从远景对峙变成主殿前中场战斗时，不要只改角色 prompt；先检查背景锚点是否服务新的镜头功能。如果主殿角度不对，应基于当前接近目标的图做视角纠偏，生成更正面、更中轴、更适合作为战斗舞台的新场景锚点。
```
