新增核心规则：

```text
1. 传统故事不能只复述剧情，要先确定改编目的：还原、重塑、科普、情绪共鸣、爽感奇观或 IP 化。
2. 神话 / 民间传说必须提炼“文化母题”：禁忌、仪式、象征物、节日由来、英雄牺牲、人与自然冲突。
3. 每个传统故事都要先建立角色资产、场景资产、关键道具和文化符号资产。
4. AI 生图提示词负责锁定单镜头画面，不写连续动作；AI 图生视频提示词负责动作、运镜、情绪、环境互动。
5. 传统故事短片适合使用“黄金 3 秒钩子 + 90 秒左右完整叙事 + 快节奏分镜”。
6. 分镜表字段建议固定为：镜号、时长、运镜方式、景别、AI生图提示词、AI图生视频提示词、对白、旁白、音乐、音效。
7. 规划时长可以是 2–3 秒，但执行层如果模型最短 4 秒，就生成 4 秒后剪辑裁短。
8. 国漫 3D 玄幻、新中式写实 CGI、山海经二次元不能只写风格名，必须转译成角色、材质、光影、色彩、场景结构。
9. 神话重塑必须保留核心母题，同时允许重新定义角色气质，例如“受难者 → 掌控者 / 英雄 / 反抗者”。
10. 武打 / 巨兽 / 神兽 / 变身 / 奔月 / 填海等镜头必须继承 V1.2 的动作、尺度、受力、特效路径规则。
11. 传统故事中的音乐、音效和旁白不是附属信息，而是叙事节奏的一部分，需进入分镜表。
12. 古典文学和名著可提炼通用结构，但不要复制现代影视剧改编里的独特造型、台词、镜头和表演。
```

# 2. 传统故事改编的第一步：确定改编目的

同一个故事可以有完全不同的改编方向。

不要直接问：

```text
帮我把《精卫填海》做成视频。
```

要先确定：

```text
我要还原传统神话？
我要重塑角色？
我要讲节日由来？
我要做国漫燃向短片？
我要做亲子科普？
我要做广告 / 品牌借势？
我要做角色 IP 设定？
我要做动作奇观测试？
```

常见改编目的：

| 改编目的 | 适合故事 | 视觉重点 | 风险 |
|---|---|---|---|
| 传说还原 | 年兽、嫦娥、武松 | 情节清晰、文化符号准确 | 容易平铺直叙 |
| 角色重塑 | 精卫、嫦娥、年兽 | 角色气质、强辨识度、现代审美 | 容易偏离母题 |
| 节日科普 | 年兽、嫦娥 | 利他价值、民俗由来、符号解释 | 容易像说明书 |
| 燃向英雄 | 精卫、后羿、武松 | 巨物、冲击、动作、誓言 | 容易过度特效化 |
| 情感悲剧 | 嫦娥、精卫 | 牺牲、离别、孤独、思念 | 容易只剩唯美画面 |
| 动作奇观 | 武松、后羿、年兽 | 打击感、受力、尺度、环境反馈 | 容易物理不可信 |
| 国风 IP 化 | 神兽、民间传说 | 角色设定卡、符号、道具、色彩 | 容易套皮网游感 |

# 3. 传统故事的核心母题提炼

传统故事进入 AI 视频前，必须先提炼“母题”。

## 3.1 年兽母题

核心不是“怪兽下山”本身，而是：

```text
岁末恐惧
村庄危机
发现规律
火光 / 红色 / 巨响克制年兽
从恐惧到欢庆
节日仪式的由来
```

关键视觉符号：

```text
寒冬古村
积雪屋顶
枯萎灯笼
红色布料
火堆
爆竹 / 巨响
门神 / 对联 / 灯笼
年兽青面獠牙 / 独角 / 黑红鳞甲
村民躲藏
老者发现规律
结尾烟火与红光
```

改编规则：

```text
年兽短片必须让观众看懂：为什么怕红色、火光、巨响。
不要只拍怪兽破坏，要拍“发现弱点 → 村民反制 → 节日诞生”的因果链。
```

---

## 3.2 精卫填海母题

核心不是“小鸟叼石头”本身，而是：

```text
人与大海的冲突
死亡 / 重生 / 执念
弱小身躯对抗巨大自然
永不屈服
从悲剧人物到反抗英雄
```

如果采用现代重塑方向：

```text
精卫不只是受难者，而是掌控海洋、无畏无惧的强者形象。
融合山海经文化和现代二次元审美。
通过宏大叙事衬托英雄主义。
```

关键视觉符号：

```text
少女 / 神鸟双形态
海浪巨墙
黑色大海
赤色羽翼
山海经纹样
石子 / 神石
风暴云
海面裂光
誓言式台词
```

改编规则：

```text
精卫不能只拍“叼石填海”的重复动作。
必须有角色立场和视觉升级：溺亡 / 转化 / 归来 / 对海宣战 / 持续填海 / 象征性胜利或永恒抗争。
```

---

## 3.3 嫦娥奔月母题

核心不是“飞上月亮”，而是：

```text
十日灾难
后羿英雄行动
不死药作为关键道具
逢蒙觊觎制造危机
嫦娥吞药的选择与代价
奔月的奇幻与孤独
广寒宫相思
```

关键视觉符号：

```text
十日高悬
干裂大地
神弓与利箭
仙山与西王母
玉盒 / 不死药
庭院阴影中的逢蒙
烛火摇曳的室内危机
白色仙衣
星辰轨迹
冷月 / 月宫 / 桂树 / 玉兔
```

改编规则：

```text
嫦娥奔月不能只拍唯美飞升。
必须保留“选择、牺牲、分离、相思”的情感链。
奔月特效要服务情绪，不要只做粒子光效。
```

---

## 3.4 武松打虎母题

核心不是“人打老虎”，而是：

```text
英雄气魄
酒家警告
景阳冈危险
醉意与胆气
人与猛兽的肉搏
真实受力和环境反馈
战胜恐惧 / 英雄确立
```

关键视觉符号：

```text
三碗不过冈酒旗
古代客栈
景阳冈山路
斑驳自然阳光
武松高保真角色
老虎体量与压迫感
枯叶 / 山风 / 断枝
拳掌击打
地面尘土 / 碎石 / 衣物震动
```

改编规则：

```text
武松打虎必须继承 V1.2 的打击感和受力规则。
老虎不是普通宠物级动物，要有尺度、重量、扑击路径、受力反馈和环境破坏。
不要只做姿势展示，要写清虎从哪里扑来、武松如何闪避、如何抓住、如何压制、如何反击。
```

# 4. 传统故事短片结构模板

## 4.1 90 秒传统故事短片结构

适合年兽、精卫、嫦娥、武松等 1–2 分钟短片。

```text
0–3s：黄金钩子。危机、奇观、疑问、强动作或强情绪。
3–15s：世界建立。场景、时代、人物和核心冲突。
15–35s：危机升级。怪兽、反派、自然灾难、人物困境。
35–55s：发现 / 选择 / 觉醒。角色找到线索，做出关键选择。
55–75s：高潮行动。对抗、奔月、填海、打虎、驱年。
75–90s：结果与余韵。节日由来、英雄形象、情感收束、文化解释。
```

## 4.2 100 秒动作叙事结构

适合武松打虎、后羿射日等动作重的传统故事。

```text
0–10s：场景和人物登场。
10–25s：危险提示和规则建立。
25–40s：进入危险空间。
40–60s：第一次遭遇。
60–80s：攻防升级。
80–95s：关键胜负手。
95–100s：收束镜头。
```

## 4.3 15 秒段落结构

适合 Dreamina / Seedance 工作流的段落规划，但不一定代表单次生成任务。

```text
段落 1：钩子 + 世界危机。
段落 2：主角 / 怪物 / 关键道具登场。
段落 3：冲突升级。
段落 4：发现规律 / 做出选择。
段落 5：高潮动作。
段落 6：结果与文化解释。
```

注意：

```text
15 秒段落可以用于分镜规划。
实际生成时仍建议拆成多个 3–5 秒 SHOT。
如果平台/CLI 最短生成时长是 4 秒，则生成 4 秒后在剪辑中裁成 2–3 秒。
```

# 6. AI 生图与图生视频分工

## 6.1 AI 生图提示词

传统故事文生图 prompt 应锁定：

```text
场景
人物
道具
景别
视角
主体位置
光影
色彩
风格
材质
情绪
文化符号
```

不要写连续动作。

错误：

```text
年兽冲进村子，撞碎石碾，灯笼被撞飞，村民逃跑，老者发现弱点。
```

正确：

```text
国漫3D玄幻风格，古村落村口夜晚全景，年兽站在破碎石碾前，青面獠牙，独角带暗金纹路，黑红鳞甲，周围积雪和破碎木墙散落，村口红灯笼半落在雪地上，远处木屋门窗紧闭，冷灰色夜景中有微弱火光，平视全景，压迫感强。
```

## 6.2 AI 图生视频提示词

图生视频 prompt 应锁定：

```text
人物动作
运镜方式
情绪
环境互动
特效运动
起始状态
结束状态
禁止变化
```

示例：

```text
@首帧作为起始画面。保持年兽、村口、破碎石碾、木墙、雪地和红灯笼位置不变。
0-1秒：年兽低头嘶吼，肩背微微起伏，黑色雾气在鳞甲边缘缓慢缠绕。
1-3秒：年兽前爪重重踏地，雪尘和碎石向前方弹起，灯笼轻微晃动。
镜头固定，不要切换场景，不要新增村民，不要改变年兽体型。
```

# 7. 角色资产规则

传统故事必须先建角色资产。

## 7.1 常见角色资产

```text
主角：武松 / 精卫 / 嫦娥 / 后羿 / 老者 / 少女。
反派或阻碍：年兽 / 逢蒙 / 海洋意志 / 老虎。
辅助角色：村民 / 老妪 / 孩童 / 西王母 / 玉兔 / 酒家。
神性角色：西王母 / 月宫仙子 / 山海经神鸟形态。
```

## 7.2 角色设定字段

每个角色至少写：

```text
身份
外形
服装
性格
行为特点
标志性道具
标志性色彩
AI生图提示词
是否需要三视图
是否需要表情差分
是否需要动作姿态图
```

## 7.3 普通角色资产

```text
A-CH-NIAN-ELDER 老者
A-CH-WUSONG 武松
A-CH-CHANG-E 嫦娥
A-CH-HOUYI 后羿
A-CH-JINGWEI-HUMAN 精卫少女形态
```

使用：

```text
纯白背景三视图 + 正面头像四联图。
必要时增加 4 个表情差分。
动作角色增加典型姿态图。
```

## 7.4 神兽 / 怪物资产

```text
A-CREATURE-NIAN 年兽
A-CREATURE-TIGER 景阳冈猛虎
A-CREATURE-JINGWEI-BIRD 精卫神鸟形态
```

神兽资产必须写：

```text
体型比例
头部特征
眼睛
角 / 翅 / 爪 / 鳞甲 / 毛发
材质
颜色
与人类或建筑的比例尺
行为特点
弱点或能力
```

不要只写：

```text
巨大年兽，恐怖老虎，神鸟精卫。
```

```text
A-SC 只负责空间、光线、材质、气氛，不放主角。
角色、怪物、关键道具进入 L / C / SHOT 阶段。
```

# 10. 传统故事风格转译规则

## 10.1 国漫 3D 玄幻风格

不要只写：

```text
国漫3D玄幻风格，8K超清。
```

要转译为：

```text
角色为中国动画电影式 3D 建模质感；
面部比例略带国漫审美但不夸张；
服装包含古风布料、云纹、红色节日符号或山海经纹样；
光影为电影级体积光和柔和环境光；
材质为写实布料、鳞甲、木材、雪地、火焰；
色彩根据剧情阶段变化：危机冷灰、反击红金、结尾暖红。
```

## 10.2 新中式写实 CGI

不要只写：

```text
新中式美学，写实CGI，虚幻引擎。
```

要转译为：

```text
写实人体比例；
古代服饰材质可信；
自然阳光、斑驳树影、真实尘土；
场景有木材、酒旗、山路、枯叶、石块等可见材质；
动作重心符合真实人体运动。
```

## 10.3 山海经二次元英雄化

不要只写：

```text
山海经文化，现代二次元审美。
```

要转译为：

```text
角色轮廓清晰，有神鸟羽翼或山海经纹样；
服装和身体局部带赤色、青黑、金色符号；
背景为巨浪、断崖、风暴、神域云纹；
角色姿态具有英雄感：正面迎风、目光坚定、双翼展开、手持神石或海浪环绕。
```

# 11. 年兽故事专项规则

## 11.1 结构

```text
1. 村庄寒冬压抑。
2. 年兽传说出现。
3. 年兽下山破坏。
4. 老者发现弱点：火光、红色、巨响。
5. 村民布置红布、火堆、爆竹。
6. 年兽被吓退。
7. 岁末欢庆，春节习俗诞生。
```

## 11.2 视觉阶段色彩

```text
危机前：冷灰、低饱和、雪、暗门窗。
年兽出现：黑红、暗金、冷雾、红瞳。
发现弱点：火光暖色开始出现。
反击：红色、火光、爆竹金光。
结尾：暖红、烟火、灯笼、欢庆。
```

## 11.3 年兽弱点表现

弱写法：

```text
年兽怕红色、火和声音。
```

稳定写法：

```text
年兽靠近火堆时，头部快速向后退缩，红瞳躲闪，肩背紧绷；火焰跳动照亮它的鳞甲。
年兽看到红色布料时，身体向一侧偏移，脚步停住，眼神避开红布。
爆竹声响起时，年兽耳部后压，身体猛地后撤，黑雾被声浪震散。
```

# 12. 精卫填海专项规则

## 12.1 重塑方向

精卫可以有两种方向：

```text
传统版：弱小但执着，叼石填海，悲壮坚持。
重塑版：掌控海洋，无畏无惧，山海经强者形象。
```

重塑版必须保留母题：

```text
与海的冲突
死亡 / 转化
石与海的对抗
永不停止的意志
```

## 12.2 三段式结构

```text
建置：少女与大海 / 风暴 / 海难。
对抗：海吞噬生命，少女转化为精卫。
解决：精卫归来，携神石、羽翼、风暴之力对抗大海。
```

## 12.3 海洋尺度

不要只写：

```text
大海巨大，巨浪滔天。
```

更稳：

```text
海浪像黑色城墙一样从画面上方压下，占据画面三分之二；
精卫位于画面下方，身形很小但双翼展开，红金羽光形成对抗线；
远处断崖和船只残骸作为比例尺。
```

# 13. 嫦娥奔月专项规则

## 13.1 结构

```text
后羿射日 → 西王母赐药 → 逢蒙觊觎 → 嫦娥吞药 → 奔月之旅 → 月宫相思。
```

## 13.2 情感链

```text
危机：十日并出，大地干裂。
希望：后羿射日，得到不死药。
威胁：逢蒙贪婪。
选择：嫦娥吞药保护不死药。
代价：飞向月宫，与人间分离。
余韵：广寒宫孤独相思。
```

## 13.3 奔月画面规则

弱写法：

```text
嫦娥飞向月亮，梦幻唯美。
```

稳定写法：

```text
嫦娥位于画面中央偏下，白色仙衣被向上气流托起，衣袖和发带向后飘动；
星辰位于背景深处，月亮固定在画面上方中央，清冷光线照亮她的侧脸；
她的身体缓慢向画面上方移动，眼神回望人间方向，表情不舍。
不要让她随机旋转，不要让月亮位置改变，不要生成多个月亮。
```

## 13.4 月宫相思规则

```text
月宫不是普通宫殿，要体现“清冷、空旷、孤独”。
玉石台阶、桂树、玉兔、冷白月光、远处人间灯火可作为情感符号。
嫦娥动作应小：回望、抚袖、低头、轻触栏杆，不要大幅跳舞。
```

# 14. 武松打虎专项规则

## 14.1 场景一致性

武松故事有强连续空间：

```text
客栈 → 酒旗 → 景阳冈山路 → 林间空地 → 打虎 → 战后。
```

必须保持：

```text
武松服装一致。
景阳冈自然光和山林方向一致。
老虎体型与纹路一致。
地面、树木、坡度、落叶等空间连续。
```

## 14.2 动作物理

老虎扑击必须写：

```text
从哪个方向扑来；
身体高度和速度；
前爪落点；
武松如何侧身闪避；
手抓住哪里；
脚步如何后撤或下沉；
接触瞬间尘土、落叶、衣服震动；
拳掌击打后老虎头部、肩背、身体重心如何反应。
```

## 14.3 不建议一镜头完成完整打虎

拆镜建议：

```text
SHOT-01：客栈警告 / 酒旗。
SHOT-02：武松上冈。
SHOT-03：老虎出现。
SHOT-04：第一次扑击，武松闪避。
SHOT-05：武松抓住虎颈或虎皮。
SHOT-06：近身搏斗。
SHOT-07：关键重击。
SHOT-08：战后收束。
```

## 14.4 老虎动作限制

```text
不要让老虎像宠物一样慢走。
不要让老虎表情拟人化过度。
不要让武松轻松挥手就打飞老虎。
不要忽略体重和惯性。
```

# 15. 传统故事音频规则

传统故事分镜表必须保留：

```text
旁白
对白
音乐
音效
```

原因：

```text
传统故事的信息量往往来自旁白；
情绪来自音乐；
动作可信度来自音效；
文化氛围来自乐器和环境声。
```

## 15.1 音乐方向

```text
年兽：低沉古风 → 紧张鼓点 → 民乐欢庆。
精卫：风暴低音 → 英雄弦乐 → 山海经神性合唱 / 鼓点。
嫦娥：危机鼓点 → 空灵仙乐 → 二胡 / 箫 / 冷月氛围。
武松：自然环境音 → 鼓点紧张 → 打击乐高潮 → 低沉收束。
```

## 15.2 音效方向

```text
年兽：风雪、嘶吼、木门、火焰、爆竹、烟火。
精卫：海浪、风暴、羽翼、石子撞击、雷声。
嫦娥：拉弓、箭啸、编钟、烛火、星辰、衣袂。
武松：酒旗、脚步、虎啸、落叶、重击、尘土、断枝。
```

# 17. 传统故事进入 DreaminaBatcher 的推荐流程

```text
1. story_outline.md：故事梗概、改编目的、主题、钩子。
2. visual_bible.md：美术风格、色彩、光影、文化符号。
3. assets.csv：角色、场景、道具、神兽、符号资产。
4. storyboard.xlsx：镜号、时长、画面、景别、运镜、旁白、音乐、音效。
5. keyframe_manifest.csv：每个 SHOT-KF 的 text2image / image2image 任务。
6. video_manifest.csv：每个 video 的 image2video / frames2video / multimodal2video 任务。
7. edit_timeline.csv：剪辑顺序、目标时长、旁白、音效、字幕。
8. review_log.md：生成结果、问题、复盘、规则更新。
```

## 17.1 task_type 建议

```text
A-CH / A-CREATURE / A-PROP / A-SC：text2image。
SHOT-KF 带角色/场景/道具参考：image2image。
普通单图动画：image2video。
变身 / 奔月前后 / 填海前后：frames2video。
角色 + 场景 + 动作参考 + 音频：multimodal2video。
多张连续分镜故事测试：multiframe2video，但正式成片仍建议剪辑合成。
```

## 17.2 剪辑目标时长与生成时长分离

```text
分镜表中的 2s / 3s 是剪辑目标。
Dreamina / Seedance 生成任务按模型支持时长提交。
如果需要 2s 镜头，可以生成 4s 或 5s，再裁剪最稳定的 2s。
```

# 18. 传统故事 Prompt 最小模板

## 18.1 角色资产模板

```text
A-CH-XXX_角色名。画面中不要出现任何文字、编号或标识。
纯白背景，国风角色设定图，角色为【身份】，外形为【外貌】，穿着【服装】，携带【道具】，性格表现为【表情/姿态】，整体风格为【国漫3D玄幻 / 新中式写实CGI / 山海经二次元】。
正面全身，身体比例清晰，服装结构完整，材质和颜色明确，适合作为后续视频参考图。
```

## 18.2 场景资产模板

```text
A-SC-XXX_场景名。画面中不要出现任何文字、编号或标识。
空场环境，画面中没有主要人物。
【时代 / 地点 / 时间 / 天气】的【场景】，包含【关键建筑/地形/道具/自然元素】，光线为【光源逻辑】，色彩为【色调】，整体风格为【风格】。
景别为【远景/全景/中景】，视角为【平视/俯视/仰视/斜侧】，空间结构清晰，适合作为后续角色进入的背景参考。
```

## 18.3 SHOT-KF 模板

```text
SHOT-XXX-KF。画面中不要出现任何文字、编号或标识。
【主体】位于【场景】中，处于【姿态/动作起始状态】，周围有【道具/环境元素】。
景别为【景别】，视角为【机位】，主体位于画面【位置】，前景是【前景】，中景是【中景】，背景是【背景】。
光线为【光源逻辑】，色彩为【色调】，风格为【国漫3D玄幻 / 新中式写实CGI / 山海经二次元】。
这是单一关键帧，不包含连续动作，不要强动态模糊，不要随机文字。
```

## 18.4 图生视频模板

```text
@首帧作为起始画面。保持首帧中的主体、服装、场景、构图、光影和关键道具不变。

0-1秒：【主体】保持起始状态，出现【微动作/呼吸/眼神/衣摆/环境变化】。
1-3秒：【主体】执行【核心动作】，方向为【方向】，速度为【速度】，终点为【终点】。
3-5秒：动作收束，环境余韵继续运动，【尘土/雪/火光/海浪/衣袖/粒子】按【方向】运动。

镜头为【固定/缓慢推近/低机位跟拍/轻微横移】。
限制：不要新增人物，不要改变主体身份，不要切换场景，不要改变关键道具位置，不要生成字幕。
```

# 19. 一句话总结

```text
传统故事 AI 视频不是把古代故事直接丢给模型，而是先提炼文化母题，再建立角色、场景、道具和符号资产；图片锁定单帧画面，视频执行动作和情绪，最终用剪辑把多个短镜头合成完整叙事。
```
