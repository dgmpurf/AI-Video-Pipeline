# 0. 本次小修订结论

本次小修订补充三类经验：

```text
1. 视角重构失败后的策略切换：
   当 image2image 过度保留旧斜侧构图时，不要继续对旧图做纠偏；
   应改用 text2image 或弱参考方式，直接重拍目标机位。

2. 短硬结果导向 prompt：
   当目标是“主殿正面中轴图”这类明确构图结果时，
   短、硬、结果导向的 prompt 往往比长篇“纠偏说明”更有效。

3. 地图切换与运镜控制：
   同一地图内保持画风稳定；切换新地图时允许整体视觉风格显著变化。
   普通运镜靠 prompt；复杂运镜建议建立 CTRL-CAM 参考视频库。
```

一句话总结：

```text
视角重构不是修图，地图切换不是乱换风格，复杂运镜不是靠一句“电影感”硬写。
```

# 1. A-SC-002 / 003 / 004 复盘结论

## 1.1 A-SC-002：方向接近，但仍是斜侧构图

A-SC-TEMPLE-COURTYARD-002 的目标是建立“主殿前战斗场景锚点”。它比 SHOT-01 的远景对峙背景更接近主殿前中场战斗需求，但仍保留明显斜侧视角。

问题表现：

```text
1. 主殿不是正面中轴。
2. 画面仍像从斜角看主殿。
3. 左右屋檐和边侧建筑仍然形成斜角框景。
4. 作为 SHOT-02 战斗背景，交锋中轴不够明确。
```

结论：

```text
A-SC-002 可作为中间失败/过渡版本，不宜作为 SHOT-02 最终主殿前战斗场景锚点。
```

---

## 1.2 A-SC-003：image2image 纠偏失败

A-SC-TEMPLE-COURTYARD-003 使用 A-SC-002 作为 primary image2image reference，希望纠偏成更正面的主殿中轴图。

结果问题：

```text
1. 仍然保留 A-SC-002 的 oblique composition。
2. 主殿虽然略有居中，但没有真正变成正面中轴图。
3. 侧边屋檐、边侧建筑和斜角透视仍然主导画面。
4. 输出像 A-SC-002 的轻微变体，不像新机位重拍。
```

结论：

```text
单图 image2image 对“斜侧图转正面图”纠偏力度不足。
旧斜侧参考图会强绑定模型的相机透视。
```

应记录状态：

```text
A-SC-TEMPLE-COURTYARD-003
status: historical_failed_viewpoint_correction_example
preferred_candidate: false
locked: false
reason: image2image over-preserved old oblique composition
```

---

## 1.3 A-SC-004：text2image 重拍正面中轴图有效

A-SC-TEMPLE-COURTYARD-004 改用 text2image，不再把 A-SC-002 / A-SC-003 作为 primary image2image reference。

核心 prompt 采用短硬结果导向：

```text
正面中轴视角，雨中古寺主殿正面，主殿正门正对镜头，主殿正门中线接近画面垂直中心线，左右石阶与石栏杆展开关系接近对称。
```

结果初步通过人工审核：

```text
1. 主殿正面中轴明显更清楚。
2. 主殿正门正对镜头。
3. 左右石阶 / 石栏杆近似对称。
4. 中景湿石板战斗舞台可用。
5. 空场无人，适合作为 SHOT-02 场景锚点。
```

结论：

```text
当目标是“换机位重拍”，text2image 的直接结果描述可能强于 image2image 的长篇纠偏。
```

# 2. 规则 A：区分“修图”与“换机位”

## 2.1 修图任务适合 image2image

以下属于修图 / 细化任务，可以使用 image2image：

```text
保留原构图，只调整天气、光影、材质。
保留原机位，只去掉人物或微调空场。
保留原场景，只增强雨雾、反光、细节。
基于已成功图做小范围优化。
```

稳定写法：

```text
保留图1的镜头位置、主体构图、透视关系和场景结构，只调整……
```

---

## 2.2 换机位任务不宜继续 image2image

以下属于换机位 / 视角重构任务，不应继续强依赖旧图：

```text
斜侧视角 → 正面中轴视角
角落透视 → 主殿正门正对镜头
远景对峙空场 → 主殿前中场战斗舞台
旧地图构图 → 新镜头功能构图
```

高风险写法：

```text
基于图1，把它调整成正面图。
把图1的斜侧视角改成正面中轴。
不要保留图1的 oblique composition。
```

问题：

```text
模型仍可能复制图1的原始透视，只做轻微修饰。
```

更稳策略：

```text
直接使用 text2image 或弱参考方式重拍目标机位。
不要让旧斜侧图成为 primary image2image input。
```

# 3. 规则 B：短硬结果导向 Prompt 优先

## 3.1 适用场景

当目标非常明确时，例如：

```text
正面中轴视角
主殿正门正对镜头
左右对称
中景战斗舞台
前景比例适中
空无一人
```

优先使用短硬结果导向 prompt，而不是长篇解释模型为什么要纠偏。

---

## 3.2 稳定核心句

```text
正面中轴视角，雨中古寺主殿正面，主殿正门正对镜头，主殿正门中线接近画面垂直中心线，左右石阶与石栏杆展开关系接近对称。
```

价值：

```text
1. 直接定义镜头位置。
2. 直接定义主殿和镜头的朝向关系。
3. 直接要求中线接近画面垂直中心线。
4. 比“更正面一点 / 纠偏 / 不要斜侧”更可执行。
```

---

## 3.3 战斗舞台锚定句

```text
前景地面比例适中，不要让前景湿石板占据画面过多比例；中景为清楚可用的战斗舞台区域，适合后续两名角色在中央近身交锋。
```

价值：

```text
1. 控制前景不要过大。
2. 把中景定义为后续角色落脚区。
3. 让场景资产服务镜头功能，而不是只追求空场美观。
```

---

## 3.4 反旧机位句

```text
不要斜侧机位，不要角落透视构图，不要继续保留 oblique composition，不要让屋檐和边侧建筑压过主殿主体。
```

价值：

```text
1. 防止回到 A-SC-002 / 003 的斜侧构图。
2. 防止侧边建筑抢走主殿中心地位。
3. 明确禁止旧失败模式。
```

# 5. 规则 C：什么时候 @ 旧图，什么时候不 @

## 5.1 不要 @ 失败斜侧图作为主参考

如果某张图的问题正是：

```text
斜侧机位
oblique composition
角落透视
前景过大
主殿不够正
```

则不要把它作为下一轮 primary image2image reference。

原因：

```text
模型会保留它的相机透视。
```

---

## 5.2 可以弱参考旧图的世界观

如果确实需要旧图保持世界连续性，只能弱参考：

```text
只参考雨中古寺世界观、美术风格、湿石板地、冷灰蓝雨雾、木质古寺、经幡和积水反光。
不要参考图1的斜侧机位，不要保留图1的 oblique composition，不要复制图1的角落透视构图。
```

但注意：

```text
弱参考仍然可能保留机位，因此换机位任务优先 text2image。
```

---

## 5.3 可以 @ 成功正面图做微调

如果已经得到正确的正面中轴图，可以将其作为主参考继续优化：

```text
@图1 作为主殿正面中轴构图参考，保持主殿正门正对镜头、左右近似对称、前方开阔湿石板广场、冷灰雨雾和空无一人的肃杀氛围。在此基础上微调画面，使中景战斗舞台更清楚，前景地面比例适中，主殿正门、石阶、石栏杆更清晰。不要新增人物，不要斜侧机位，不要角落透视构图，不要让屋檐和边侧建筑压过主殿主体。
```

# 6. 地图切换 = 视觉章节切换 / 画风切换规则

## 6.1 规则定位

这是项目级美术策略，不是单次 prompt 技巧。

本项目允许并鼓励：

```text
每切换到一个新地图 / 新场景体系时，整体画风也随之发生一次明显变化。
```

这意味着，整部作品不必从头到尾保持完全同一种视觉风格。

---

## 6.2 核心原则

```text
同一地图内：保持画风稳定。
切换新地图时：允许画风显著变化。
```

同一地图内需要保持：

```text
场景结构连续
色彩系统稳定
光影逻辑稳定
材质质感稳定
角色外观稳定
镜头语言相对统一
```

切换到新地图时，可以变化：

```text
主色调
光影风格
材质质感
环境气氛
镜头语言
美术流派倾向
画面颗粒 / 清晰度 / 饱和度
整体视觉情绪
```

---

## 6.3 必须保留的连续锚点

即使换地图、换画风，也必须保留：

```text
角色身份可辨认
角色核心服装 / 发型 / 武器 / 色彩符号可延续
世界观不崩
叙事气质仍属于同一部作品
不是随机换风格，而是地图级视觉章节切换
```

---

## 6.4 Prompt 模板

```text
这是新的地图 / 新场景体系，因此允许整体视觉风格从上一地图发生明显变化。
保持角色身份和世界观连续，但本地图应建立独立的色彩系统、光影逻辑、材质质感和气氛。
同一地图内后续镜头应保持该地图的视觉规则稳定。
```

---

## 6.5 禁止误用

不要把“地图切换画风变化”理解为：

```text
同一场景内每个镜头随机换画风
同一角色每次出场都换脸换服装
为了炫技乱换风格
地图还没切换就突然改变色彩和材质
完全抛弃前面建立的世界观
```

一句话总结：

```text
同一地图内要稳，换地图时可以变；地图切换就是视觉章节切换，但角色身份和世界观锚点必须保留。
```

# 7. CTRL-CAM 运镜参考库规则

## 7.1 规则定位

CTRL-CAM 是控制资源，不是项目画面资产。

它的用途是：

```text
为复杂运镜提供速度、方向、节奏和镜头惯性参考。
```

它不负责：

```text
角色身份
场景风格
服装
剧情内容
最终构图
```

---

## 7.2 什么时候不需要 CTRL-CAM

以下普通运镜可以直接用 prompt：

```text
固定镜头
极慢推近
极慢拉远
轻微横移
轻微镜头抬升
稳定空镜
稳定对峙慢推镜
```

例如 SHOT-01-V001 的 stable standoff slow push-in，只需要 image2video + 轻动效 prompt，不必引入运镜参考视频。

---

## 7.3 什么时候需要 CTRL-CAM

以下复杂运镜建议录制或制作参考视频：

```text
低机位贴地跟拍
近身手持微晃
快速推进到交锋点后停住
短促 hit-stop 镜头震动
横移揭示主体
从柱子后滑出
穿过门廊进入内院
45–90 度短距离环绕
速度渐变 speed ramp
俯冲跟随
坠落冲击镜头
```

原因：

```text
文字很难稳定描述复杂镜头的速度变化、停顿、惯性和空间路径。
```

---

## 7.4 CTRL-CAM 录制要求

推荐：

```text
4–6 秒
16:9
无人物脸
无品牌 logo
无字幕文字
无复杂剧情
无强风格
背景尽量简单
运动路径清楚
镜头意图单一
```

适合录制环境：

```text
空房间
空走廊
灰盒场景
Blender / Unreal 灰盒
手机稳定器拍摄的简单空间
桌面小物体
```

不适合：

```text
有人物表演的视频
有明显 IP / 品牌 / 水印的视频
有复杂剧情的视频
会让模型复制场景的强风格视频
```

---


因为 image2video 只支持单张 `--image`，不能传运镜视频。多图 / 视频 / 音频参考应走 multimodal2video。

示例：

```powershell
$prompt = Get-Content -Raw -Encoding UTF8 "G:\...\prompts\shot_prompt.txt"

C:\Users\msjpurf\bin\dreamina.exe multimodal2video `
  --model_version seedance2.0_vip `
  --ratio 16:9 `
  --duration 5 `
  --video_resolution 720p `
  --prompt "$prompt" `
  --image "G:\...\locked_refs\SHOT-02-KF-002.png" `
  --video "G:\...\controls\camera_refs\CTRL-CAM-007_speed_ramp_push_in_stop_5s.mp4"
```

## 7.9 CTRL-CAM 使用决策表

```text
普通慢推 / 慢拉 / 固定镜头
→ 不录，用 prompt。

角色轻动效 + 稳定关键帧
→ image2video，不用运镜参考。

战斗跟拍 / 手持晃动 / 快速推进停顿 / 短环绕 / 复杂揭示
→ 用 CTRL-CAM，走 multimodal2video。

一个镜头里又要复杂动作又要复杂运镜
→ 不硬做，优先拆镜头或用参考视频。
```
