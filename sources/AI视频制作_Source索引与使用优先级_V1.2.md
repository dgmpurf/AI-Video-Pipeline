# 1. 总优先级规则

## 1.1 内容创作与 Prompt 规则优先级

```text
本项目实测规则 V1.1 / V1.2 / V1.9 / V1.10 / V1.10.1
> 本项目 Source 增补 V1.3–V1.8
> Seedance V0.3 综合工作流手册
> 外部教程 / 外部案例 / GitHub prompt 案例
> 纯理论 / 未验证经验
```

核心原则：

```text
实测规则 > 外部教程 > 纯理论。
```

如果外部资料与本项目实测冲突，优先采用本项目实测。

针对固定场景中的双人对峙、远近人物、前景空地、脚下落点、左右分置等空间调度问题，优先采用 V1.9 / V1.9.2。针对场景锚点重构、斜侧机位转正面中轴、旧图 image2image 纠偏失败、地图画风切换、CTRL-CAM 运镜参考库问题，优先采用 V1.10 / V1.10.1。

## 1.2 Dreamina CLI / Batcher 执行层优先级

执行命令、参数、字段映射时，优先级不同于创作规则：

```text
dreamina_cli_help.md / 本地 dreamina <command> -h
> Dreamina_CLI工作流与执行规范_V1.1_官方Help校正版.md
> DreaminaBatcher_manifest_schema_V1.1_官方Help校正版.md
> 其他通用工作流 Source
```

如果命令参数冲突，以 `dreamina_cli_help.md` 或本地最新 `dreamina <command> -h` 为准。

# 3. 任务类型 → 应该调用的 Source

## 3.1 生成图片 prompt

优先顺序：

```text
V1.1 图片 prompt 实测规则
V1.9 空间调度、远近站位与脚下落点规则
V1.10 / V1.10.1 场景锚点重构、短硬结果导向 prompt 与地图画风规则
V1.3 图片 prompt 结构和构图具体化
V1.4 图片 prompt 结构统一、角色设定卡、参考图保护词
V1.5 美术设计与视觉统一
V1.6 风格词库使用规则
Seedance V0.3 图片 prompt 模板
```

适用问题：

```text
角色资产
场景资产
道具资产
关键帧图
风格板
角色设定卡
海报图
产品 hero shot
地图级场景锚点
主殿正面中轴图
跨地图视觉风格切换
```

基础判断：

```text
图片负责锁定“是什么”：主体、姿态、空间、构图、光影、材质、风格。
不要在关键帧里塞大量动态特效。
不要只堆画质词。
构图词必须翻译成可见空间关系。
当旧斜侧图 image2image 纠偏失败时，不要继续修旧图；优先用 text2image / 弱参考重建，并使用短硬结果导向 prompt。
```

## 3.3 动作、武打、变身、巨物、裂隙

优先顺序：

```text
V1.2 动作变身运镜增补
V1.1 战斗打击感实测规则
V1.3 关键帧特效和视频特效路径
V1.8 仙侠玄幻特效路径与复杂运镜风险
Seedance V0.3 拆镜头 / 参考视频 / 首尾帧规则
```

适用问题：

```text
打斗
冷兵器
轻武器挡重武器
爆气
变身
巨兽 / 机甲
山岳级尺度
空间裂隙
高速坠落
流光突进
御剑 / 飞升 / 雷暴 / 结界
```

核心规则：

```text
动作不能只写“猛”。要写发力链、受力反馈、命中位置、停顿、受击反应、环境反馈。
巨物不能只写“巨大”。要写比例尺、人类剪影、建筑、桥梁、塔吊、船坞、海浪。
裂隙不能只写“空间撕裂”。要写形状、边缘、内部颜色、空气扭曲、吸附碎石和禁止项。
变身如果是对峙关系，要锁定对视轴线，不能转成正面海报。
快节奏打斗不是整段慢动作，只在命中瞬间保留极短 hit-stop。
```

```text

```

## 3.10 空间调度 / 远近站位 / 双人对峙

优先顺序：

```text
V1.9 空间调度与远近站位控制
V1.1 机位与站位规则
Seedance V0.3 Composition Matrix
V1.2 动作对峙与变身对视轴线规则
```

适用问题：

```text
固定场景参考图中的双人对峙
女近男远 / 男远女近
远景角色降权
近景角色不要贴边
脚下落点与场地区块绑定
双远景同层左右分置
大面积前景空地保留
禁止两人被吸到同一个中心建筑锚点
```

核心规则：

```text
1. 场景图负责镜头构图和透视，角色图只负责外观，不负责站姿构图。
2. 远景角色必须主动降权：远景对峙目标，不是近景展示主体。
3. 近景角色内收时，不要只写“往里一点 / 后退一点”，要写“不要贴边框 + 脚下落点 + 场地区块”。
4. 双远景同层左右分置时，不要共享同一个中心锚点；要分别绑定左右两个远处区块。
5. 双远景同层必须写：两人都远、都小、左右明确分开、中间留出空场，禁止收拢到主殿前同一片区域。
6. 角色朝向必须单独写：身体朝向、头部朝向、视线目标、不要正面看镜头。
```

SHOT-01 当前已验证分支：

```text
Branch A：女近男远。霜玑近处中景主控，焚狩左侧远景小轮廓。
Branch B：双远景同层左右分置。焚狩左侧远处区块，霜玑右侧远处区块，两人都远都小，中间大面积空旷湿石板地。
```

当前《赤焰对天穹》SHOT keyframe image2image 手动测试 / 复跑约定：

```text
默认 model_version=4.7；
ratio=16:9；
resolution_type=2k；
使用三张 locked refs：场景图 + 焚狩角色图 + 霜玑角色图；
提交前检查 prompt 中不得混入已否定中心锚点词：主殿正门 / 石阶 / 石栏杆 / 围绕主殿正门。
```

注意：

```text
model_version=4.7 是当前项目 SHOT 关键帧测试约定，不是所有未来项目的全局默认。
```

## 3.11 场景锚点重构 / 视角纠偏 / 短硬结果导向 Prompt

优先顺序：

```text
V1.10 视角纠偏与场景锚点重构
V1.10.1 视角重构短硬 Prompt 补丁
V1.9 空间调度与远近站位
Seedance V0.3 Composition Matrix
```

适用问题：

```text
远景对峙场景切换到中场战斗场景
旧 A-SC 背景不适合新镜头功能
斜侧古寺庭院要转为主殿正面中轴图
image2image 纠偏保留旧 oblique composition
A-SC-002 / A-SC-003 类输出像旧图轻微变体
需要新建 A-SC-004 / A-SC-005 这类真前视场景锚点
```

核心规则：

```text
1. 镜头功能变化时，先判断背景锚点是否需要重做。
2. 远景对峙背景不一定适合近身交锋。
3. 如果旧图 image2image 纠偏失败，不要无限 rerun；应标记失败样本并改用 text2image / 弱参考重建。
4. 视角重构不是修图，而是换相机位置重拍。
5. 对模型更有效的往往是短硬结果导向描述：正面中轴视角、主殿正门正对镜头、左右近似对称、中景战斗舞台。
6. 旧斜侧图不要作为 primary reference；最多作为世界观弱参考或失败样本。
```

高价值短硬 prompt 关键词：

```text
正面中轴视角
主殿正门正对镜头
主殿正门中线接近画面垂直中心线
左右石阶与石栏杆展开关系接近对称
前景地面比例适中
中景为清楚可用的战斗舞台区域
不要斜侧机位
不要角落透视构图
不要继续保留 oblique composition
```

## 3.12 地图切换 / 视觉章节切换 / 画风切换

优先顺序：

```text
V1.10.1 地图切换 = 视觉章节切换规则
V1.5 美术圣经与视觉统一
V1.6 风格词库使用边界
Seedance V0.3 工作流
```

适用问题：

```text
新地图 / 新场景体系
不同地图是否要保持同一画风
地图级色彩系统和光影系统
跨地图角色一致性
视觉章节设计
```

核心规则：

```text
1. 同一地图内：画风稳定，保证镜头连续性、空间连续性和角色一致性。
2. 切换新地图时：允许并鼓励整体画风显著变化一次。
3. 地图切换可以改变主色调、光影逻辑、材质质感、环境气氛、镜头语言和美术流派倾向。
4. 即使换画风，也必须保留角色身份、核心服装/发型/武器/色彩符号和世界观锚点。
5. 画风变化应是地图级视觉章节切换，不是同一场景内随机乱跳。
```

# 4. 冲突处理规则

## 4.1 外部教程 vs 实测规则

如果外部教程说：

```text
每一拍都要有运镜。
15 秒可以塞 6–10 个镜头。
复杂动作 + 复杂运镜可以直接写长 prompt。
```

而本项目 Source 说：

```text
一个镜头优先只有一个主要动作、一个主要运镜、一个情绪目标。
复杂动作、复杂运镜、复杂转场优先拆镜头或用参考视频。
15 秒多镜头模板适合做分镜规划，不建议全部塞进单次生成任务。
```

则采用本项目 Source。

---

## 4.2 Prompt 美学词 vs 空间关系

如果 prompt 里只有：

```text
电影感，大师构图，黄金螺旋，视觉冲击，史诗级，震撼。
```

必须补成：

```text
主体在画面哪个区域？
占画面多大？
前景 / 中景 / 背景分别是什么？
引导线从哪里到哪里？
光源来自哪里？
主体朝向哪里？
视线看向哪里？
```

---

## 4.3 视频动作 vs 图片关键帧

如果关键帧里已经出现：

```text
爆炸正在爆开
冲击波已经扩散
碎石已经飞满画面
角色已经半变身
刀光已经划满画面
```

应判断为高风险关键帧。推荐改成：

```text
关键帧只保留主体、姿态、空间、构图、光影基调和弱环境状态。
爆发、扩散、飞散、流动、变身过程交给视频 prompt。
```

---

## 4.4 网页端 @素材 vs CLI 文件路径

网页端可以写：

```text
@图片1 作为角色参考。
@视频1 作为动作参考。
```

CLI / Batcher 应使用：

```text
image / video / audio 字段 + 真实本地文件路径映射。
```

prompt 中可以写：

```text
参考图1作为角色参考。
参考图2作为场景参考。
参考视频1只参考动作节奏。
参考视频2只参考运镜速度、方向和节奏。
```

不要默认把逻辑 ID 当作平台真实 @token。

---

## 4.5 image2image 纠偏 vs text2image 重建

如果目标是修细节、保留构图、保持同一机位，可以使用 image2image。

如果目标是从斜侧机位变成正面中轴机位，或从旧地图构图变成新地图构图，则不能默认继续 image2image。

判断规则：

```text
保留旧机位，只改局部 → image2image
换相机位置，重建构图 → text2image / 弱参考重建
旧图会把模型锁回错误构图 → 不作为 primary reference
```

---

## 4.6 地图内一致性 vs 跨地图画风变化

同一地图内，连续镜头应保持统一的色彩、光影、材质和空间规则。

跨地图时，可以主动改变画风，但必须保留角色身份和世界观锚点。

判断规则：

```text
同一地图 / 同一场景体系 → 稳定画风，不乱跳。
新地图 / 新场景体系 → 允许视觉章节切换。
```

# 5. 标准工作流索引

## 5.1 单个正式项目工作流

```text
1. 确定项目类型：短片 / 广告 / 神话 / 动作 / 产品 / 测试。
2. 写一句话主题和受众。
3. 写故事结构 / 剧本 / 分镜表。
4. 制作美术圣经：风格、色彩、光影、角色、场景、道具。
5. 生成 A 原子资产：角色、道具、场景。
6. 生成 L / C / HC 复合资产和关键帧。
7. 对复杂镜头写 Composition Matrix。
8. 写 SHOT-KF 图片 prompt。
9. 写 video prompt：机位、站位、视线、运动路径、锁定元素、时间线。
10. 写 manifest / CSV / JSON。
11. 用 DreaminaBatcher / CLI 执行。
12. 下载、重命名、整理版本。
13. 轻记录所有结果，深分析 MIXED / FAIL / LOCK。
14. 剪辑成片。
15. 复盘并更新规则库。
```

---

## 5.2 快速测试工作流

```text
1. 明确测试目标。
2. 选择 1 个镜头类型。
3. 每次只改 1–2 个变量。
4. 同 prompt 跑 3 条。
5. 记录平台、模块、模型、参考素材、参数、结果。
6. 判断成功点、失败点、触发误解词。
7. 如果问题反复出现，写成风险规则。
8. 只有本项目复测成功后，升级为 B/A 级规则。
```

# 6. Source 更新机制

## 6.1 什么时候可以新增 V1.11 或后续修订

只有出现以下情况才新增 V1.11、V1.10.x 或后续新版本：

```text
1. 某个真实项目完成了一轮生成和复盘。
2. 出现新的稳定失败模式。
3. 出现新的稳定成功写法。
4. Dreamina CLI / 即梦 / 可灵 / 其他平台能力明显变化。
5. Batcher schema 或执行链路发生重大变化。
6. 多平台对比测试形成明确结论。
```

不因为以下原因新增新版本：

```text
又找到一个外部 prompt 包。
又看到一个教程。
又看到别人好看的图。
又收集了一批风格词。
还没有复测的猜测。
```

---

## 6.2 新规则进入 Source 的等级

```text
A 级：本项目重复验证 2 次以上，可作为稳定规则。
B 级：本项目单次验证，可作为暂行规则。
C 级：外部公开案例观察，只能作为假设规则。
D 级：网上教程 / 经验说法，仅供参考。
```

新增规则格式：

```text
规则名称：
证据等级：A / B / C / D
来源：本项目实测 / 外部案例 / 教程经验
适用范围：图片 / 视频 / 动作 / 构图 / 特效 / 光影 / 角色一致性
问题表现：
可能原因：
稳定写法：
风险写法：
禁止写法：
是否需要复测：是 / 否
```
