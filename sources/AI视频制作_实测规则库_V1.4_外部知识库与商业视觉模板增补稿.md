# 0. 本次更新结论

本次 V1.4 的核心不是新增某个项目 prompt，而是建立一套 **外部资料吸收规则 + 商业视觉模板 + 视频提示词结构校正规则**。

```text

V1.4：外部知识库、GitHub 图片案例、Seedance 教程、AI提示词文档的吸收和改写规则。
```

新增核心规则：

```text
1. AI视频知识库类文档只作为资料索引，不直接成为高优先级 Source。
2. Seedance 教程里的多镜头模板适合做“分镜规划”，不建议整段直接塞进一个视频任务。
3. AI生图提示词结构可统一为：景别 + 视角 + 主体行为环境 + 美学短词 + 限制。
4. AI生视频提示词结构可统一为：动作 + 镜头 + 情绪/照明 + 环境互动 + 时间线。
5. 复杂视频 prompt 必须区分“分镜脚本语言”和“单次生成任务语言”。
6. GitHub 图片 prompt 案例适合提炼角色设定卡、商业海报、产品 hero shot、9宫格 storyboard，不适合直接作为视频执行规则。
7. 参考图保护词必须写清：保持身份、结构、比例、材质、纹理、核心识别特征，不要重设计。
8. 商业广告 / 产品展示镜头要优先锁产品身份、产品层级、卖点顺序和 closing frame。
9. 镜中人 / 反射 / 分身类镜头必须区分“现实主体”和“镜中主体”。
10. 所有外部教程建议先标 C/D 级，只有本项目复测后才能升级为 B/A 级。
```

# 2. AI视频知识库类资料的使用规则

## 2.1 定位

`AI视频知识库.docx` 本质上是一个资料导航：包含平台工具、Seedance 手册、GPT-Image GitHub、案例文档、脚本分镜模板、电影风格提示词、思维导图、LibTV 指南等。

它不应该直接作为高优先级 source，而应该作为：

```text
资料目录
后续上传顺序规划
案例来源清单
待提炼材料索引
```

## 2.2 不直接吸收的内容

以下内容不要直接进入主 Source：

```text
外部完整 prompt 原文
别人打包的素材链接
现成 IP 案例设定
平台广告语
未验证的工具推荐
大批量风格词表原文
大批量分镜提示词原文
```

## 2.3 可以吸收的内容

可以吸收：

```text
工具链分类
视频生产阶段划分
脚本分镜 → 静帧画面 → 视频动画 → 配音剪辑的流程意识
案例文件的分类方法
后续资料上传优先级
```

## 2.4 后续处理顺序

根据知识库目录，后续资料建议按以下顺序处理：

```text
第 1 批：影片的美术设计.docx、优秀的脚本是什么样的.docx
第 2 批：AI视频脚本分镜模板_共300条.xlsx、5000+分镜画面提示词.xlsx、300+电影风格提示词.csv
第 3 批：中国传统故事案例：武松打虎、精卫填海、年兽、嫦娥奔月
第 4 批：高风险 IP / 网文类案例：剑来、凡人逆天伐仙，只提炼通用结构，不吸收专有设定
```

# 3. 图片 Prompt 结构统一规则

## 3.1 核心结构

从 `AI提示词.docx` 和 GitHub 图片案例中可提炼出图片 prompt 的统一结构：

```text
景别
+ 视角 / 机位
+ 画面内容：主体 + 行为 + 环境
+ 画面美学：风格 + 色彩 + 光影 + 构图
+ 画质 / 材质 / 限制
```

该结构与 V1.3 的“自然语言画面描述 + 美学短词描述”一致，V1.4 只做进一步标准化。

## 3.2 图片 prompt 推荐模板

```text
资源ID。画面中不要出现任何文字、编号或标识。

【景别与视角】
远景 / 全景 / 中景 / 近景 / 特写 / 大特写 / 极端特写；
正面平视 / 侧面 / 背面 / 斜侧 / 俯视 / 仰视 / 四分之三侧。

【画面内容】
主体是谁，正在做什么，位于什么环境；
主体与环境、道具、其他人物的空间关系。

【构图】
主体位于画面哪个区域，占画面多大；
前景 / 中景 / 背景分别是什么；
是否有引导线、负空间、对称、非对称、对角线、中心构图、重复韵律。

【主体细节】
身份、年龄段、服装、发型、道具、姿态、朝向、视线、材质。

【美学短词】
风格、色彩、光影、材质、画面质感。

【限制】
不要文字、水印、logo、随机字幕、额外人物、主体变形。
关键帧阶段避免强动态特效。
```

## 3.3 图片 prompt 不应只堆画质词

弱写法：

```text
8K、超清、史诗级画质、电影质感、大师作品、高级感、震撼。
```

问题：

```text
这些词可以提升质感倾向，但不能稳定解决主体、构图、空间、动作、尺度问题。
```

更稳：

```text
先锁：主体、空间、比例、机位、光源。
再补：画质、材质、风格。
```

## 4.1 适用场景

适用于：


## 4.2 标准角色设定卡内容

参考 GitHub 角色设计案例，角色资料卡可以从 V1.1 的“三视图 + 正面头像四联图”扩展为：

```text
正面全身

色板
材质说明
世界观摘要
白色背景

```

## 4.3 角色设定卡模板

```text
A-CH-XXX_角色设定卡。画面中不要出现无关文字、水印或随机编号。

白色背景，专业概念艺术设定资料卡布局。
画面分区清晰，包含同一角色的正面全身、侧面全身、背面全身、正面头像特写。

整体为高分辨率专业角色概念设定图，干净、可读、适合后续作为视频参考图。
```

## 4.4 文字风险

如果模型文字不稳定：

```text

角色名称、部件名称、说明文字后期添加。
```

# 6. 参考图保护词规则

## 6.1 适用范围

适用于：

```text
人物脸
产品
武器
神兵
铠甲
饰品
符号
道具
建筑局部
怪物轮廓
```

## 6.2 保护词模板

```text
@参考图1 作为唯一的【主体/产品/道具/角色身份】参考。
保持原始设计、轮廓、比例、材质、纹理、颜色、装饰结构和核心识别特征。
不要重新设计，不要简化，不要替换成类似物，不要改变主体身份。
可以改变背景、光影、构图和拍摄角度，但不能改变主体的关键结构。
```

## 6.3 人物身份保护

```text
保持同一个人的脸型、五官比例、发型、肤色、身形比例和整体气质。
不要换脸，不要年龄漂移，不要变成另一个人，不要让多帧中的角色身份不一致。
```

## 6.4 产品 / 道具保护

```text
产品必须成为画面主视觉，不被模特、手、文字、道具或背景遮挡。
产品标签、轮廓、材质、核心形状和识别特征保持清晰。
不要把产品变成类似产品，不要重新设计包装，不要改变比例。
```

# 7. 商业海报与负空间规则

## 7.1 负空间不是空洞

海报类 prompt 可以使用大量负空间，但必须说明：

```text
负空间在哪里
保留给什么用途
主体如何引导视线
文字区是否后期添加
```

更稳写法：

```text
画面上半部分保留干净的暗色负空间，用于后期添加标题。
主角位于画面右下角，运动轨迹从右下向左上形成弧形引导线。
背景城市沿这条轨迹逐渐展开，形成从小主体到大世界的视觉转化。
```

## 7.2 海报构图模板

```text
【海报主题】。
干净背景，大量负空间位于【上方/左侧/右侧】。
主视觉主体位于画面【区域】，占画面【比例】。
一条【水流/光带/烟雾/布料/道路/笔触】从主体出发，沿【方向】形成引导线，并逐渐转化为【城市/山川/战场/记忆场景】。
前景、中景、背景层级清晰，画面不拥挤。
文字区保持干净，如需文字建议后期添加。
```

# 8. 9宫格 TVC Storyboard 模板

## 8.1 定位

9宫格 storyboard 不是最终视频生成任务，而是：

```text
广告分镜规划图
产品短片预览板
关键镜头清单
后续生成 SHOT-KF 的母版
```

## 8.2 9宫格通用结构

适用于产品广告、道具展示、角色变装、世界观资产展示。

```text
1. 环境建立镜头
2. 主体 hero shot
3. 极近细节特写
4. 使用动作 / 交互动作
5. 功能 / 容量 / 能力展示
6. 清洁 / 收纳 / 变化 / 处理过程
7. 底部 / 隐藏 / 工艺细节
8. 情绪氛围镜头
9. 品牌 / 主题 closing frame
```

## 8.3 产品 9宫格模板

```text
生成一个 9 宫格产品 TVC storyboard，竖屏 9:16，专业商业分镜板布局。
@参考图1 作为唯一产品参考，保持产品原始设计、比例、材质、标签和识别特征不变。

第1格：环境建立，全景，产品位于真实使用环境中。
第2格：产品 hero shot，中景，产品成为画面主视觉。
第3格：极端特写，展示核心材质 / 工艺 / 图案。
第4格：使用动作，手或人物与产品自然互动，不遮挡主体标签。
第5格：功能展示，用俯视或剖面感展示容量 / 结构 / 效果。
第6格：处理过程，展示清洗 / 收纳 / 打开 / 变形 / 启动。
第7格：底部或隐藏细节特写，展示防滑垫、接口、纹理、铭文等。
第8格：夜间或情绪氛围镜头，产品与环境光互动。
第9格：closing frame，产品居中定格，留出后期标题和 logo 区域。

整体保持一致的产品身份、光影调性、色彩系统和商业摄影质感。
不要生成随机文字，不要改变产品设计，不要让每格产品变成不同物体。
```

# 9. 视频 Prompt 结构统一规则

## 9.1 核心公式

从 `AI提示词.docx` 和 `Seedance 2.0 使用教程.pdf` 可统一为：

```text
动作
+ 镜头
+ 情绪 / 照明
+ 环境互动
+ 时间线
+ 锁定元素
```

## 9.2 单动作模板

```text
@首帧作为起始画面。保持主体、服装、场景、构图和光影不变。
【主体】在【场景】中执行一个清晰动作：【动作】。
动作速度为【快速/缓慢/平稳/用力】，方向为【向前/向后/向左/向右/向上/向下】，幅度为【小幅/中等/大幅】。
镜头为【固定/缓慢推近/跟拍/轻微横移】。
环境中【头发/衣摆/尘埃/水面/光影】随动作产生轻微变化。
不要新增人物，不要改变主体身份，不要切换场景。
```

## 9.3 时序组合动作模板

复杂动作必须按顺序写，不要把多个动作堆成一句。

```text
0-1秒：主体保持起始姿态，出现微表情和呼吸变化。
1-2秒：主体开始执行第一个动作，运动方向为【方向】。
2-3秒：动作衔接到第二个动作，身体重心从【位置】转移到【位置】。
3-4秒：动作达到高潮，镜头只做【一种主要运镜】。
4-5秒：动作收束，环境余韵继续运动，主体回到稳定姿态。
```

## 9.4 动作修饰词

动作不只写动词，要补：

```text
速度：快速、缓慢、突然、平稳、逐渐。
方向：向前、向后、向画面左侧、向画面右上方、沿道路深处。
幅度：轻微、小幅、中等、大幅、猛然。
力度：用力、沉重、轻盈、拖拽、被重力拉下。
终点：停在哪里、看向哪里、手碰到哪里、最后姿态是什么。
```

# 10. 运镜模板吸收与限制

## 10.1 基础运镜可用

外部教程中常见的基础运镜可以吸收：

```text
Slow Dolly In / 缓慢推近
Dolly Out / 拉远
Orbit Shot / 环绕
Top View Pan / 俯视平移
Low Angle Tracking / 低角度跟拍
Macro Fast Zoom / 微距快速变焦
Handheld Shake / 手持轻微晃动
Cinematic Zoom Out / 电影感拉远
```

但进入本项目时必须改写为中文自然语言：

```text
摄影机从产品正前方缓慢推近。
镜头围绕主体右侧做小角度环绕，不超过 90 度。
低机位贴近地面跟随主体向前移动。
镜头快速推近到道具表面细节，然后停住。
```

## 10.2 每个生成任务只保留一种主要运镜

外部教程有时要求每一拍都包含运镜，但本项目仍保持 V1.1/V0.3 原则：

```text
一个镜头优先只有一种主要运镜。
```

可以有轻微辅助运动：

```text
主运镜：缓慢推近。
辅助：人物呼吸、衣摆、尘埃、光影轻微变化。
```

不要写：

```text
同时快速推进 + 360环绕 + 上升 + 下落 + 镜头旋转 + 变焦。
```

## 10.3 特殊机位谨慎

外部提示词文档提到，特殊机位如严格垂直俯拍可能不稳定。

本项目规则：

```text
如果必须垂直俯拍，先用图片关键帧锁定视角，再图生视频。
如果文生视频无法稳定特殊机位，改用首帧 / 首尾帧 / 参考视频。
```

## 10.4 复杂动作 + 复杂运镜不要叠加

高风险组合：

```text
复杂打斗 + 复杂环绕
多人动作 + 快速切换
变身 + 大幅移动 + 镜头旋转
从地面运镜到宇宙
同一镜头中升起、转圈、推进、下落全部发生
```

处理：

```text
拆镜头。
使用首尾帧。
使用动作参考视频。
把复杂运镜改为简单推近 / 跟拍 / 固定镜头。
```

# 11. 多镜头模板的使用边界

## 11.1 外部教程中的多镜头结构

Seedance 教程中提供了 5秒、10秒、15秒的多镜头模板，适用于漫剧、微短剧、电商产品展示、TVC广告、科普短片等。

这些模板适合做：

```text
分镜规划
shot list 设计
广告结构设计
TVC 段落拆分
剪辑方案
```

不建议直接做：

```text
把 6-10 个镜头全部塞进一个 15 秒视频生成 prompt。
```

原因：

```text
模型不一定能稳定执行真实剪辑切换。
多镜头会增加角色漂移、场景跳变、动作丢失和时间线混乱。
```

## 11.2 本项目改写规则

外部模板：

```text
镜头1 → 镜头2 → 镜头3 → 镜头4
```

进入本项目后应改成：

```text
SHOT-01-KF + SHOT-01 video
SHOT-02-KF + SHOT-02 video
SHOT-03-KF + SHOT-03 video
最后用剪辑合成
```

或者：

```text
如果首尾状态明确，用 frames2video。
如果多张分镜需要连续过渡，用 multiframe2video 测试。
如果需要角色、场景、音频、动作多参考，用 multimodal2video。
```

## 11.3 5秒 / 10秒 / 15秒建议

```text
5秒：1 个核心动作 + 1 个主要运镜 + 1 个情绪目标。
10秒：2-3 个动作阶段，可拆成两个 5秒。
15秒：只适合广告展示、稳定环境镜头、产品展示；剧情和打斗优先拆镜头。
```

# 12. 时间标记提示词模板

## 12.1 可用格式

外部教程中的时间标记结构可以吸收，但要减少冗余。

标准格式：

```text
(00-01.5s)：【运镜】+【环境光影】+【主体起势动作与微表情】。
(01.5-03.0s)：【运镜变化】+【主体位移 / 动作发展】+【环境互动】。
(03.0-04.0s)：【动作高潮】+【关键细节特写】+【短暂停顿 / 冲击反馈】。
(04.0-05.0s)：【收尾】+【动作惯性】+【环境余韵】。
```

## 12.2 本项目推荐 5 秒模板

```text
@首帧作为起始画面。保持首帧中的主体、服装、场景、构图和光影不变。

(00-01.5s)：摄影机【固定/缓慢推近/低机位跟拍】，主体保持起始姿态，出现【呼吸/眼神/衣摆】轻微变化，环境中【雾气/尘埃/光影】缓慢流动。
(01.5-03.0s)：主体开始执行【核心动作】，身体从【起点】移动到【中间位置】，目光始终看向【目标】。
(03.0-04.0s)：动作达到高潮，【接触/转身/抬手/爆发】发生在画面【区域】，镜头不乱转。
(04.0-05.0s)：主体停在【终点位置】，环境余韵继续运动，保持角色身份和场景结构不变。

限制：不要新增人物，不要切换场景，不要随机字幕，不要改变主体身份，不要让关键道具变形。
```

# 13. 情绪提示词规则

## 13.1 情绪不能只写一个词

弱写法：

```text
恐惧。
悲伤。
愤怒。
害羞。
```

更稳：

```text
一个躲在衣柜中的小男孩，面露恐惧，眉头紧皱，眼睛紧张地左右张望，蹲着用双臂慢慢抱紧身体，身体微微发抖。
```

## 13.2 情绪结构

```text
环境
+ 基础情绪词
+ 面部表情动作
+ 身体动作
+ 呼吸 / 眼神 / 手部细节
+ 镜头距离
+ 光影配合
```

## 13.3 多情绪递进

多情绪可以写，但必须有逻辑和上限：

```text
胆怯 → 放松 → 欣慰
愤怒 → 停顿 → 压抑
惊讶 → 恐惧 → 后退
```

不要写：

```text
恐惧、快乐、愤怒、悲伤、温柔、疯狂同时出现。
```

# 14. 镜中人 / 反射 / 分身类镜头规则

## 14.1 问题

镜子类镜头容易混乱：

```text
现实人物和镜中人物身份混合。
镜中人跑到现实中但没有过程。
镜中反射和真实人物动作完全同步，无法产生恐怖感。
人物位置、镜子位置、浴室空间漂移。
```

## 14.2 稳定规则

必须区分：

```text
现实主体 = A
镜中主体 = A_reflection
```

并写清：

```text
A 位于镜子外，站在洗手台前。
A_reflection 只存在于镜面内部。
A_reflection 可以表情不同、动作不同，但脸部身份仍来自 A。
镜子边框、洗手台、墙面瓷砖从第一帧开始固定存在。
```

## 14.3 镜中人模板

```text
@图片1 作为现实女孩 A 的身份参考，保持脸型、发型、服装一致。
@图片2 作为浴室环境参考，保持镜子、洗手台、瓷砖墙和灯光位置一致。

现实女孩 A 站在镜子外的洗手台前，属于画面中景。
镜中女孩 A_reflection 只存在于镜面内部，与现实女孩拥有相同脸型和服装，但表情变得冷淡、诡异。
镜子外的 A 正在擦脸，镜子里的 A_reflection 没有同步擦脸，而是缓慢抬头微笑。
镜子边框、洗手台和浴室墙面从第一帧开始固定，不要改变空间结构。
不要让镜中女孩一开始就站在现实地面上；如果她要爬出镜面，必须写清从镜面内部伸手、破裂、上半身探出、落到地面的过程。
```

# 15. 广告 / 产品展示视频结构

## 15.1 产品广告常用镜头顺序

```text
1. 环境氛围建立
2. 产品整体 hero shot
3. 材质 / 标签 / 工艺微距
4. 产品与光影互动
5. 使用场景 / 功能展示
6. closing frame：产品定格 + 主题 / 品牌区
```

## 15.2 产品视频单镜头模板

```text
@参考图1 作为唯一产品参考，保持产品外形、比例、标签、材质、颜色和核心识别特征不变。

(00-01.5s)：微距镜头从产品表面缓慢推近，侧光掠过材质纹理，水珠 / 尘埃 / 光斑轻微移动。
(01.5-03.0s)：镜头小角度环绕产品，产品始终位于画面中心，不被遮挡，标签清晰。
(03.0-04.0s)：快速推近到核心细节，如瓶盖、纹理、接口、镜头、宝石、刀柄纹路。
(04.0-05.0s)：镜头平稳拉远，产品成为 hero object，背景保持简洁，留出后期标题区域。

限制：不要改变产品设计，不要生成随机 logo，不要让产品变形，不要让手或人物遮挡产品核心标识。
```

## 15.3 多产品 / 全系列展示

```text
多个产品必须保持同一品牌视觉系统。
每个产品的位置、大小、朝向要明确。
不要让多个产品互相融合。
closing frame 中全系列产品必须完整可见，并留出后期文案区域。
```

# 16. 配音 / 字幕 / 音效规则

## 16.1 配音和画面分开写

外部教程中常把旁白和画面放在同一段，进入本项目后建议拆成：

```text
旁白轨：时间码 + 文案 + 语气
画面轨：时间码 + 镜头 + 动作 + 光影
音效轨：环境音 / 动作音 / 氛围音
```

## 16.2 不要字幕规则

如果不想让模型生成字幕，要明确写：

```text
画面中不要出现字幕、文字、logo、水印或随机字符。
旁白只作为音频 / 文案参考，不生成在画面里。
```

## 16.3 音效写法

音效可以帮助动作理解，但不要过多：

```text
低频撞击声
碎裂声
急促呼吸声
布料摩擦声
水滴声
远处风声
镜面细微开裂声
```

音效不是画面主体，不要让模型转去生成文字或声波图。

# 19. 冲突处理规则

## 19.1 外部教程：每一拍都必须包含运镜

本项目修正：

```text
每一拍可以有镜头状态，但每个生成任务只保留一种主要运镜。
环境、微表情、衣摆、尘埃可以作为动态补充。
```

## 19.2 外部教程：一个 15 秒 prompt 可含 6-10 个镜头

本项目修正：

```text
这种写法适合做分镜脚本，不适合直接作为一个稳定视频任务。
正式生产中拆成多个 SHOT 生成，再剪辑合成。
```

## 19.3 外部 prompt：大量品牌/IP/人物名

本项目修正：

```text
只提炼结构，不复制专有设定。
商业项目中避免直接使用现成 IP 名、明星名、真实品牌名作为默认规则。
```

## 19.4 外部 prompt：画质词过多

本项目修正：

```text
画质词放最后，不替代主体、空间、动作、构图、光源。
```
