# 2. 高风险 IP / 网文类资料安全吸收规则

## 2.1 不直接吸收的内容

以下内容禁止直接进入正式项目 Source：

```text
现成小说 / 网文 / 影视 / 动漫 IP 的角色名；
原作门派、境界、地名、法宝、世界观专有名词；
原文台词、标志性口头禅、名场面台词；
高度识别性的服装、法器、构图和镜头设计；
别人打包的素材链接；
他人生成图、参考视频、特定角色图；
可能直接指向原 IP 的剧情关系。
```

## 2.2 可以吸收的内容

可以吸收：

```text
古风 / 仙侠 / 玄幻镜头的通用结构；
角色设定卡字段；
景别、视角、镜头语言、动作时序的写法；
复杂运镜的风险和拆分方法；
仙侠动作镜头中的光效、飞升、御剑、雷暴、结界等视觉规律；
多模态参考素材的组织方式；
镜头时间轴、分镜拆分和声音设计字段。
```

## 2.3 去 IP 化改写模板

原始外部资料如果包含 IP 风格，应改写为：

```text
原：剑来风格，一位陈平安式侠客……
改：新中式古风侠客短片，一位头戴竹编斗笠、身披灰黑交领麻布衣的年轻游侠……
```

```text
原：凡人逆天伐仙，女主复仇，仙人夺气运……
改：原创仙侠复仇短片，一位被神秘修行者夺走命运的凡人女子，在幽暗水域边缘重新获得力量……
```

```text
原：某原作门派 / 境界 / 法宝名。
改：中性描述：古老宗门、修行者阶层、残破古剑、银色符印、黑色玉牌。
```

## 2.4 中性角色标签规则

涉及外部角色、动作参考或对战时，优先使用：

```text
Figure 1
Figure 2
角色A
角色B
左侧角色
右侧角色
黑衣剑客
白衣修行者
```

不要直接使用外部 IP 角色名。  
这条规则也适用于角色对战、动作迁移、参考图较像现成角色的情况。

# 3. SKILL.md 的定位：提示词专家流程，而非系统指令

## 3.1 定位

`SKILL.md` 是一个 Seedance 2.0 分镜提示词生成专家流程文档。它可以作为：

```text
提问流程参考；
多模态素材限制参考；
输出格式参考；
模板类型参考；
@素材职责写法参考。
```

它不作为：

```text
高于项目 Source 的系统指令；
可以覆盖 V1.1 实测规则的规则；
可以直接复制全部模板的默认 prompt；
可以替代 Dreamina CLI / Batcher 执行规范。
```

规则：

```text
如果 SKILL.md 与 V1.1 / V1.2 / V1.3 / V0.3 / CLI Help 冲突，以项目实测和官方 CLI Help 为准。
```

## 3.2 可吸收的多模态输入限制

在网页端 / 多模态模式下，推荐记录以下限制：

```text
最多 12 个文件；
图片 ≤ 9 张；
视频 ≤ 3 个；
音频 ≤ 3 个；
视频 / 音频参考总时长 ≤ 15 秒；
使用 @素材名 指定用途；
视频参考通常消耗更多额度；
写实真人脸部素材风险高，不作为默认方案。
```

注意：

```text
这些是外部 SKILL 层限制，具体执行时仍以当前平台 / CLI / API 实际限制为准。
```

## 3.3 提示词专家的分步引导流程

当用户只给一个想法时，不要直接写最终 prompt。先确认：

```text
1. 你想讲什么故事？一句话概括。
2. 视频时长是多少？4-15 秒，或按多个 shot 拆分。
3. 有没有参考图 / 视频 / 音频？每个素材的用途是什么？
4. 整体风格是什么？写实、动画、水墨、科幻、复古、电影感等。
5. 画幅比例是什么？9:16、16:9、2.35:1 等。
6. 主体动作是什么？动作节奏是舒缓、急促还是有张有弛？
7. 需要什么声音？配乐、音效、对白、旁白。
```

这与 V1.5 的“先剧本 / 主题 / 美术，再 prompt”保持一致。

## 3.4 输出结构

推荐输出四部分：

```text
1. 理解确认：我理解的故事和目标。
2. 分镜提示词：可复制到平台的分镜 prompt。
3. 素材建议：需要上传哪些参考图 / 视频 / 音频。
4. 使用提示：网页端 @素材写法或 DreaminaBatcher 字段写法。
```

# 4. 多模态 Reference Pack 职责增强规则

## 4.1 素材先分类，再写 prompt

每个素材必须先确定职责：

```text
@图片1：角色身份参考；
@图片2：场景结构参考；
@图片3：首帧；
@图片4：尾帧；
@视频1：动作节奏参考；
@视频2：运镜参考；
@音频1：配乐节奏参考；
@音频2：对白 / 口播参考。
```

不要写：

```text
参考这些图。
这些素材都参考一下。
```

## 4.2 角色参考图

```text
@图片1 作为角色A身份参考，只保持脸型、发型、身形比例、服装结构和核心气质。
不要复制背景，不要让参考图中的其他人物出镜。
```

## 4.3 场景参考图

```text
@图片2 作为场景结构参考，只保持建筑布局、道路方向、光源方向和空间层级。
不让图片中的人物出镜，不复制具体构图。
```

## 4.4 动作参考视频

```text
@视频1 只参考动作节奏、身体发力路径和关键姿态。
不复制视频中的人物身份、服装、场景和镜头背景。
```

## 4.5 运镜参考视频

```text
@视频2 只参考运镜速度、镜头方向和节奏。
不复制视频中的人物、场景和剧情。
```

## 4.6 音频参考

```text
@音频1 作为背景音乐节奏参考，只参考节拍、情绪和强弱变化。
@音频2 作为对白/口播内容参考，需明确语气和情绪，不要让画面自动生成随机字幕。
```

## 4.7 网页端 @ 与 CLI 字段区别

```text
网页端：可以写 @图片1、@视频1、@音频1。
CLI / DreaminaBatcher：应使用真实文件路径，通过 image / video / audio 字段传入。
```

不要把网页端 @token 当成 CLI 的真实引用路径。

# 5. 图片 Prompt 结构复核

`凡人逆天伐仙.docx` 和 `剑来.docx` 都重复出现了图片 prompt 的基本结构：

```text
景别
视角
画面内容：人物 + 行为 + 环境
画面美学：风格、色彩、光影、构图
画质
```

这与 V1.3 / V1.4 已经一致。V1.8 不新增新结构，只强化执行顺序。

## 5.1 图片 prompt 推荐顺序

```text
资源ID。画面中不要出现任何文字、编号或标识。
景别与视角：远景 / 全景 / 中景 / 近景 / 特写；平视 / 仰视 / 俯视 / 侧面 / 背面 / 四分之三侧。
主体行为环境：主体是谁，在哪里，做什么，和场景 / 道具的空间关系。
构图：主体位置、画面占比、前景 / 中景 / 背景、引导线、留白、尺度对比。
主体细节：服装、发型、道具、姿态、朝向、视线、材质。
美学短词：风格、色彩、光影、质感。
限制：不要文字、不要水印、不要多余人物、不要强动态特效。
```

## 5.2 仙侠角色图片 prompt 风险

高风险词：

```text
绝美仙侠；
不朽的女士；
史诗般氛围；
大师作；
超现实主义；
极致华丽；
玄幻大招；
```

问题：

```text
容易生成页游封面、过度美颜、性别漂移、服装泛化、光效压过主体。
```

更稳：

```text
明确脸型、发型、服装层级、材质、道具位置、站姿、背景纯白或具体场景；
风格词放在后面，不要替代角色结构描述。
```

# 6. 视频 Prompt 结构复核

`剑来.docx` 明确采用“动作 + 镜头 + 情绪/照明”的视频结构，并强调动作可以添加速度、方向、幅度；复杂动作要按时序写。

V1.8 推荐统一为：

```text
起始状态
+ 主体动作
+ 镜头运动
+ 情绪 / 表情 / 呼吸
+ 光线 / 环境互动
+ 特效运动路径
+ 结束状态
+ 锁定元素 / 禁止变化
```

## 6.1 单动作模板

```text
@首帧作为起始画面，保持主体、服装、场景、构图和光影不变。
角色A在【场景】中执行一个清晰动作：【动作】。
动作速度为【快速/缓慢/突然/平稳】，方向为【向前/向左/向右/向上/向下/向画面深处】，幅度为【小幅/中等/大幅】。
镜头为【固定/缓慢推近/平稳跟拍/轻微横移】。
角色的眼神、呼吸、衣摆、发丝和环境尘埃产生轻微变化。
结束时角色停在【终点】，看向【目标】。
限制：不要新增人物，不要切换场景，不要改变身份和道具形状。
```

## 6.2 时序组合动作模板

```text
0-1秒：角色保持起始姿态，出现呼吸、眼神和衣摆轻微变化。
1-2秒：角色开始第一个动作，身体从【起点】向【方向】移动。
2-3秒：动作衔接到第二个动作，重心从【位置】转移到【位置】。
3-4秒：动作达到高潮，镜头只做一种主要运镜。
4-5秒：角色停在终点，环境余韵继续运动，锁定角色身份和场景结构。
```

## 6.3 多人物动作模板

```text
角色A位于画面左侧，角色B位于画面右侧。
A 的身体朝向 B，B 的身体朝向 A。
两人的视线始终互相锁定，不要转向镜头。
动作按顺序发生：A 先起势 → B 反应 → 两者接触 / 交错 → 双方分开或定格。
不要让两人贴在同一平面，不要变成横版游戏式侧拍。
```

# 7. 镜头语言速查表：镜头名必须转成可执行动作

## 7.1 基础镜头语义

| 镜头 | 作用 | 稳定写法 |
|---|---|---|
| 推镜头 | 强调主体、细节、心理压力 | 摄影机从中景缓慢向角色脸部推进，最后停在近景，不改变视角 |
| 拉镜头 | 揭示环境、表达孤独 / 宏大 | 摄影机从角色背部逐渐向后上方拉远，展示周围环境和空间关系 |
| 摇镜头 | 固定支点转向，引导视线 | 摄影机位置不动，镜头从角色脸部向右摇，最后落在远处目标上 |
| 移镜头 | 摄影机本体平移，展示连续空间 | 摄影机沿画面右侧平稳横移，平行经过人物和建筑 |
| 跟镜头 | 跟随主体移动 | 摄影机在角色身后低机位跟拍，角色始终位于画面中央偏下 |
| 航拍 | 展示宏大尺度 / 个体渺小 | 摄影机从高空斜向下俯拍，先展示环境全貌，再向目标区域推进 |
| 环绕 | 突出主体成为中心 | 摄影机围绕主体右侧小角度环绕，不超过 90 度，主体始终在画面中央 |
| 变焦 | 焦距变化，产生戏剧性强调 | 机位不动，镜头焦距快速拉近到角色眼睛或道具细节 |
| 手持 | 紧张、纪实、不稳定感 | 摄影机轻微手持晃动，幅度很小，不影响主体清晰度 |

## 7.2 不要只写镜头名

弱写法：

```text
推镜头，角色很震撼。
环绕镜头，打斗很激烈。
航拍镜头，展现宏大。
```

更稳：

```text
摄影机从角色半身中景缓慢向脸部推进，最后停在眼睛近景，角色眼神逐渐坚定。
摄影机围绕两名对峙者右侧做 60 度小角度环绕，两人始终互相对视，背景山门保持在画面后方。
摄影机位于山谷上方百米高度，斜向下俯拍，画面下方是渺小人群，中景是巨大石门，远处是山脉。
```

# 8. 复杂运镜风险规则

## 8.1 每次组合建议 2–3 个镜头动作以内

外部资料提到：复杂镜头建议逐句拆写，且每次组合 2–3 个镜头动作最稳。进入本项目后规则更严格：

```text
单个生成任务优先只保留 1 个主要运镜。
最多允许 1 个主要运镜 + 1 个轻微辅助运动。
超过 2 个镜头动作，优先拆镜头。
```

## 8.2 特殊机位默认高风险

高风险：

```text
严格垂直俯拍；
360 度高速环绕；
希区柯克变焦；
鱼眼镜头；
镜头时间停滞；
地面直接运镜到宇宙；
同一镜头中同时升起、转圈、推进、下落。
```

处理：

```text
先生成关键帧锁视角；
图生视频时只让画面轻微运动；
或使用参考视频；
或拆成多个镜头；
或使用 frames2video 连接首尾状态。
```

## 8.3 复杂动作 + 复杂运镜不要叠加

高风险组合：

```text
人物飞升 + 360 环绕 + 大范围爆炸；
两人打斗 + 快速摇镜 + 快速推进 + 特写切换；
御剑飞行 + 镜头从地面升到宇宙；
变身 + 飞到空中 + 雷暴 + 巨型结界 + 镜头旋转。
```

推荐拆法：

```text
SHOT-A：角色起势，固定或慢推。
SHOT-B：角色飞升，低机位跟拍或静帧转视频。
SHOT-C：高空能量爆发，远景固定镜头。
SHOT-D：结界 / 雷暴反馈，环境镜头。
```

# 9. 仙侠玄幻镜头专项规则

## 9.1 仙侠玄幻资产先行

不要直接把完整仙侠片段写成一个长 prompt。先建立资产：

```text
A-CH：凡人女子 / 白衣修行者 / 黑衣剑客 / 掌灯人 / 反派修士；
A-PROP：古剑 / 竹编斗笠 / 玉牌 / 符纸 / 药瓶 / 法器；
A-SC：忘川水岸 / 破败山门 / 青石步道 / 雨夜古镇 / 高空结界；
A-FX：蓝白雷光 / 金色法力波动 / 黑色虚无瞳孔 / 半球形结界 / 能量光带；
CTRL：动作参考 / 运镜参考 / 音乐卡点参考。
```

## 9.2 仙侠特效必须写路径

弱写法：

```text
爆发雷电，画面震撼，能量很强。
```

更稳：

```text
蓝白色雷光从角色右手掌心聚集，沿手臂向肩部蔓延，再向角色背后扩散成十几条细长光带。
光带从画面中央向上方和左右两侧扩张，不遮挡角色脸部。
最后光带逐渐散开，只留下角色周围微弱轮廓光。
```

## 9.3 飞升 / 御剑 / 流光突进

必须写：

```text
起点：角色原本站在哪里；
方向：向画面上方 / 深处 / 左上 / 右上；
速度：缓慢上升 / 瞬间突进 / 稳定飞行；
身体姿态：背对镜头 / 侧身 / 俯冲；
镜头位置：低机位仰拍 / 侧后跟拍 / 固定远景；
结束：停在空中 / 飞出画面 / 撞向目标。
```

## 9.4 结界 / 护城大阵

必须写：

```text
结界形状：半球形透明穹顶 / 圆形屏障 / 网状符文罩；
位置：覆盖画面上方天空，底部边缘与远处城墙相接；
材质：半透明蓝白光膜，有水波纹状能量涟漪；
比例尺：下方树林、城墙、人群显得渺小；
运动：雷电击中结界表面后，涟漪从接触点向四周扩散。
```

## 9.5 仙侠爆炸 / 大招

关键帧不要直接放完整爆炸。  
视频阶段按时间写：

```text
0-1秒：能量聚集在角色身体或武器上。
1-2秒：核心光团变亮，周围空气轻微扭曲。
2-3秒：冲击波从中心向四周扩散，碎石和尘埃沿半径方向飞出。
3-5秒：光效退去，角色和场景结构保持稳定，只有余尘飘动。
```

限制：

```text
不要让强光吞掉主体脸部。
不要让光效遮住武器和身体姿态。
不要从第一帧就已经爆炸完成。
```

# 10. 情绪演绎规则增强

外部资料对情绪写法的价值在于：情绪不能只写一个词，要用面部、身体、呼吸、眼神、手部细节表现。

## 10.1 单情绪模板

```text
主体位于【环境】中，基础情绪为【恐惧/悲伤/愤怒/警觉/放松】。
脸部表现：【皱眉/瞳孔放大/嘴唇颤抖/眼神躲闪/眼神坚定】。
身体表现：【肩膀收紧/呼吸急促/双手握拳/身体微微后退/胸口起伏】。
手部表现：【手指抓住衣角/握住剑柄/按住桌面/缓慢松开】。
镜头：【近景/特写/中景】。
光影：【冷色侧光/暖色窗光/弱轮廓光】。
```

## 10.2 多情绪递进

可以写：

```text
胆怯 → 惊觉 → 坚定；
疑惑 → 愤怒 → 决绝；
悲伤 → 沉默 → 爆发；
压抑 → 冷笑 → 出手。
```

不要写：

```text
恐惧、快乐、愤怒、悲伤、疯狂、温柔同时出现。
```

## 10.3 仙侠复仇情绪模板

```text
角色低头跪在雨中，肩膀微微颤抖，呼吸急促。
听到真相后，她慢慢抬头，眉头收紧，眼神从茫然转为愤怒。
双手握紧，指节发白，衣袖被雨水贴在手臂上。
最后她抬眼看向远处敌人，眼神变得冷静而决绝。
```

# 11. 模板库使用边界

`SKILL.md` 包含多个模板：叙事故事、产品展示、角色动作、风景旅拍、视频延长、视频编辑、情感冲突、产品动效、空间漫游、角色对战、口播、音乐卡点、战争场景、长镜头追踪、伪纪录片等。

## 11.1 模板不是默认 prompt

每个模板只能作为：

```text
结构参考；
字段参考；
提问框架；
分镜草案。
```

不能直接作为：

```text
项目固定 prompt；
所有镜头通用模板；
无复测的稳定规则；
替代 Composition Matrix 的方法。
```

## 11.2 选择模板前先选用途

```text
叙事故事类：适合剧情短片；
产品展示类：适合广告和产品功能；
角色动作类：适合角色亮相和单动作展示；
空间漫游类：适合建筑 / 室内 / 展馆；
角色对战类：适合动作参考视频辅助；
口播类：适合数字人 / 对口型；
音乐卡点类：适合多图卡点，但需音频参考。
```

## 11.3 模板进入本项目后的改写步骤

```text
模板原文
→ 去掉外部角色名 / IP 名 / 过度泛化描述
→ 补项目角色和资产 ID
→ 补 Composition Matrix
→ 补参考图职责
→ 补起始状态和锁定元素
→ 拆成 SHOT-KF + 视频 prompt
→ 进入 DreaminaBatcher manifest
```

# 12. 多模态任务与 DreaminaBatcher 映射建议

## 12.1 网页端素材规划

```text
角色图 ≤ 3：主角 / 对手 / 群体角色；
场景图 ≤ 2：主场景 / 远景结构；
关键帧图 ≤ 2：首帧 / 尾帧；
视频参考 ≤ 1-2：动作 / 运镜；
音频 ≤ 1：配乐 / 口播 / 节奏。
```

避免：

```text
上传太多参考图却不写用途；
同一素材同时承担角色、场景、动作、风格；
参考视频既要动作又要场景又要人物；
音频和视频不对齐却要求卡点。
```

## 12.2 DreaminaBatcher 推荐 task_type

```text
角色 / 道具 / 场景原子资产：text2image
带参考图重建关键帧：image2image
单张关键帧转视频：image2video
首尾状态明确：frames2video
多帧连续故事：multiframe2video，可测试
角色 + 场景 + 动作视频 + 音频：multimodal2video
```

## 12.3 多模态 prompt 写法

```text
参考图1作为角色A身份参考，只保持脸型、服装和身形。
参考图2作为场景结构参考，只保持建筑布局和光源方向。
参考视频1只参考动作节奏，不复制人物和场景。
参考音频1只参考音乐节拍和情绪。
```

不要写：

```text
所有素材都参考。
按这些素材生成。
完全照着视频做。
```

# 13. 生成前检查清单 V1.8

## 13.1 IP / 资料安全检查

```text
是否包含现成 IP 名？
是否包含原作角色名？
是否复制了专有台词？
是否复制了独特设定 / 地名 / 法宝名？
是否使用别人素材链接作为项目资产？
是否已经把角色改成原创中性描述？
```

## 13.2 多模态素材检查

```text
图片数量是否 ≤ 9？
视频数量是否 ≤ 3？
音频数量是否 ≤ 3？
视频 / 音频参考是否过长？
每个素材是否写明职责？
是否有素材职责冲突？
是否需要视频参考，还是关键帧足够？
```

## 13.3 镜头语言检查

```text
是否只写了镜头名，没有写起点/终点/方向？
是否同时堆了太多运镜？
是否把复杂动作和复杂运镜放进同一镜头？
是否需要拆成多个 5 秒镜头？
是否需要首尾帧？
是否有特殊机位风险？
```

## 13.4 仙侠特效检查

```text
法术从哪里出现？
向哪里扩散？
经过前景 / 中景 / 背景哪一层？
是否遮挡角色脸部和武器？
结束后画面留下什么余韵？
有没有比例尺？
是否强光吞掉主体？
```
