# AI视频制作_实测规则库 V1.1

> 用途：记录真实项目中已经验证的稳定规则、风险规则和禁止规则。  
> 适用：所有后续 AI 视频项目。  
> 优先级：高于外部教程和通用手册。  
> 来源：当前《修月人》项目制作实测 + 用户反馈 + 足球进球镜头/动作打斗镜头实测。

---

# 0. 核心原则

```text
实测规则 > 外部教程 > 纯理论。
```

如果外部资料建议与本规则冲突，优先采用本规则。

# 1. 资产阶段规则

## 1.1 独立人物与物品使用纯白背景

```text
A-CH 独立人物资产：纯白背景
A-PROP 独立物品资产：纯白背景
A-SC 独立环境资产：真实环境空场，不用纯白背景
```

原因：

```text
纯白背景方便后续抠图、放入场景、作为参考图复用。
```

## 1.2 基础环境必须空场化

错误：

```text
SC-NODE-01 月影节点空地里出现白衣人或黑石。
SC-NODE-03 草叶低伏细节里出现人物。
```

正确：

```text
A-SC 环境只负责空间、光线、材质、气氛。
人物、关键道具、剧情关系必须放入 L / C / ANCHOR。
```

## 1.3 重要细节主体要独立建模

包括：

```text
匠人
牵光天女
兔形工役
寒凿
宽平刷
巨手
玉屑饭
黑石
行囊
书札
残卷
```

不要等到复合镜头里再临时生成。

## 1.4 三视图 + 正面头像四联图

普通重要角色建议：

```text
左一：正面全身
左二：侧面全身
左三：背面全身
右一：正面头像特写
```

神秘角色例外：

```text
白衣修月人不做完整三视图；
只做限制视角基准：侧背、半遮面、手部、衣袖、背影。
```

# 2. 命名与文件管理规则

## 2.1 Prompt 第一行加入 ID

示例：

```text
A-PROP-BS-01。画面中不要出现任何文字、编号或标识。
SHOT-08_月影节点初见关键帧。画面中不要出现任何文字、编号或标识。
```

好处：

```text
即梦下载文件名会带 ID，方便批量重命名。
```

## 2.2 多版本先保留

不要急着删：

```text
v01 / v02 / v03
```

最后再复制主版本到：

```text
00_locked_refs
```

## 2.3 参考图异常处理

实测有效：

```text
WPS 轻微压缩画质后重新上传，可解决部分“版权受限 / 参考图异常”误判。
```

处理顺序：

```text
WPS 轻微压缩
→ PNG 转 JPG / 改尺寸
→ 裁切去水印 / 去印章 / 去边框
→ 换图
```

# 3. 即梦图片 Prompt 规则

## 3.1 物理词优先于意象词

即梦图片生成更像“视觉关键词 + 材质 + 形状”的重组。

更稳：

```text
苍白冷玉峡谷
银白断壁
青灰岩脉
短小椭圆米粒轮廓
乳白玉
瓷白石英
```

高风险：

```text
月球大峡谷
玉屑饭
仙子
神秘仙境
山一样巨大
```

## 3.2 高训练频概念会触发强联想

例子：

```text
写“月球大峡谷”容易生成完整月亮。
写“月宫”容易生成宫殿。
写“山一样的工具”容易真的生成山。
写“仙子”容易变西方 fairy 或仙侠手游女角色。
```

修正：

```text
月球大峡谷 → 苍白冷玉裂口 / 银白冷玉断壁
仙子 → 牵光天女 / 敦煌飞天式东方天女
山一样巨大 → 超大尺度 / 占据画面大部分
玉屑饭 → 短小椭圆米粒轮廓的乳白玉与瓷白石英颗粒
```

## 3.3 Negative Prompt 少量纠偏

默认不把长串避免项放进 prompt。  
如果跑偏，再加 2–4 个关键限制。

例如：

```text
不要石碑，不要石柱，不要方形人造石台。
不要西方 fairy，不要西式天使。
不要水潭，不要新月。
```

# 4. 视频阶段总规则

## 4.1 进入视频阶段后，重点从“画面”转为“调度”

资产图和环境图已经解决：

```text
人物是谁
道具长什么样
场景是什么风格
```

视频 prompt 需要解决：

```text
摄影机在哪里
人物从哪里来
停在哪里
看向哪里
谁在前景 / 中景 / 背景
哪些主体从第一帧就已经存在
```

## 4.2 每个视频镜头都要写六项

```text
Camera position：摄影机位置
Framing：前景 / 中景 / 背景
Blocking：人物站位
Eyeline：人物视线方向
Movement path：运动路径
Locked elements：锁定元素
```

## 4.3 “起始状态”必须锁死

如果不写清，模型会让主体突然出现。

必须写：

```text
黑石和白衣人从第一帧开始就已经存在。
不要让黑石或白衣人中途突然出现。
```

## 4.4 “视线”不能省略

如果想让角色看见某物，要写：

```text
头部、胸口、脚尖都朝向目标。
目光集中在目标上。
不要继续沿原方向向前看。
```

仅写“看见”不够。

# 5. 机位与站位规则

## 5.1 双人镜头避免纯正面近距离

风险：

```text
两个人脸融合
两人长成同一张脸
人物朝向混乱
```

更稳：

```text
背后跟拍
侧后方
三分之二侧背
拉开距离
一前一后
```

## 5.2 人物进入场景要写入口和终点

模板：

```text
郑洵和王晦从画面左下方同一条草径进入，郑洵在前，王晦在后。
二人在空地边缘停下。
身体转向画面中央的黑石和白衣人。
```

## 5.3 避免“平行路过”

如果目标在中景，人物不能只是沿画面边缘走过。  
要写：

```text
人物进入后转身面对目标。
人物视线落在目标上。
人物停在目标前方若干步。
```

# 6. 光线实测规则

## 6.1 “月光照脸”容易变手电筒

高风险：

```text
月光照出半张脸
一道月光落在人物身上
光束照向修月人
```

更稳：

```text
柔和弥散的夜间环境光
淡淡冷白轮廓光
衣袖和黑石边缘有弱冷光
没有可见光束
没有丁达尔效应
没有手电筒式聚光
没有舞台灯打脸
```

## 6.2 避免丁达尔效应

如果不想要光柱，必须明确写：

```text
没有可见光束，没有丁达尔效应。
```

# 7. 特定高风险画面规则

## 7.1 近圆缺口天体

不要写：

```text
裂痕
裂缝
月亮裂开
```

容易生成：

```text
沿边缘发光白缝
表面大裂纹
缺口移动
新月
```

更稳：

```text
近乎完整的冷白圆形天体。
右上边缘只有一个极小暗色缺口。
缺口是黑暗空缺，不是白色裂缝，不发光，不沿边缘延伸。
圆形天体保持静止。
```

如果仍不稳：

```text
不用视频生成，改成静帧 + 剪辑慢推 + 云雾叠加。
```

## 7.2 黑石与白衣人位置

不要只写：

```text
黑石一侧
```

容易被理解成：

```text
人物在黑石顶部
人物从黑石里出现
人物躺在镜面上
```

更稳：

```text
白衣人位于黑石旁边的地面上，身体靠近黑石侧面，但不在黑石顶部，不在光滑镜面区域。
黑石顶部镜面和月痕区域保持空出。
```

如果要坐在黑石上：

```text
坐在黑石粗粝侧缘，不坐在顶部光滑镜面区域。
```

## 7.3 主观五视图 / 空间控制图

不要写：

```text
我站在峡谷底部向左看。
```

模型会生成：

```text
画面里出现一个人。
```

更稳：

```text
空镜头。
画面只拍一面高耸苍白冷玉断壁。
镜头几乎垂直朝上，只拍天顶空间。
镜头方向是脚下地面，只拍银白冷玉峡谷底部地面。
```

左右视角不要写“两侧崖壁”，只写：

```text
单面崖壁近景。
```

# 8. 首尾帧规则

## 8.1 大幅动作需要首尾帧

以下动作不要只靠单张图生视频：

```text
卧姿 → 坐起
远处进入 → 停下观察
空场 → 发现主体
手未触物 → 手触物
现实 → 抽象转场
```

需要：

```text
首帧：起始状态
尾帧：结束状态
```

## 8.2 首尾帧差距不要过大

错误：

```text
首帧空场，尾帧突然有黑石、白衣人、二人。
```

更稳：

```text
首帧中关键主体已经存在，只让人物做小幅动作。
```

# 9. 镜头拆分规则

## 9.1 不稳定时先拆，不要硬写

如果一个镜头包含：

```text
人物进入
发现目标
停下
转身
对话
目标坐起
```

应拆成多个镜头。

## 9.2 典型拆分

```text
SHOT-08A：二人进入空地，黑石与白衣人已在中景
SHOT-08B：二人停下并看向白衣人
SHOT-12A：白衣人卧在黑石旁
SHOT-12B：白衣人已半坐，抬眼
```

# 10. 《修月人》已验证问题库

## 10.1 SHOT-08

问题：

```text
二人从画面两边出来；
走得太快；
黑石和白衣人突然出现；
二人没有看向白衣人，而像平行路过。
```

原因：

```text
缺入口、运动路径、站位、视线、锁定元素。
```

修正：

```text
摄影机在二人身后偏左；
二人从画面左下方同一条草径进入；
郑洵在前，王晦在后；
中景黑石和白衣人从第一帧已存在；
二人停下后头部、胸口、脚尖都朝向黑石和白衣人。
```

## 10.2 SHOT-12

问题：

```text
白衣人躺在黑石上坐起来；
月光像手电打脸；
后来虽无强光，但仍出现丁达尔光柱。
```

原因：

```text
位置没锁死；
“月光照脸”暗示聚光；
单关键帧无法稳定卧姿到坐姿。
```

修正：

```text
使用首尾帧；
首帧：白衣人卧在黑石旁边地面，不在顶部镜面；
尾帧：白衣人半坐，仍不遮挡顶部镜面；
光线：柔和弥散环境光，无可见光束，无丁达尔效应。
```

## 10.3 SHOT-04

问题：

```text
缺口变成沿月球边缘的白色裂缝；
缺口移动。
```

原因：

```text
“裂痕 / 裂缝”词触发表面纹理和动态裂开。
```

修正：

```text
用“右上边缘极小暗色缺口”；
缺口是空缺，不是白线；
圆形天体静止；
必要时用静帧剪辑，不生成视频。
```

# 12. 体育 / 足球镜头实测规则

## 12.1 足球镜头不能只锁“球门存在”，还要锁“球门位置”

失败表现：

```text
球门虽然出现了，但不在它应该在的球场位置；
球门漂浮在草坪中段；
出现两个平行球门；
球门不是完整的；
球员像在一个球门前射向另一个球门。
```

原因：

```text
Prompt 只写了“完整球门”“射门入网”，但没有锁定球门与球场线、底线、禁区线、观众席的关系。
模型会把球门当成一个独立物体，而不是球场结构的一部分。
```

稳定写法：

```text
画面中只有一个完整足球球门。
这个唯一球门固定在球场远端的底线尽头，位于草坪最远处。
球门前方清楚可见禁区线和小禁区线。
球门后方是广告牌和观众席。
球门不能漂浮在草坪中段，不能离开底线位置。
```

## 12.2 球门完整可见规则

如果球门是关键主体，必须写：

```text
左门柱、右门柱、横梁、球网全部完整可见。
球门不被裁切，不被前景人物遮挡。
```

仅写“完整球门”不够。

## 12.3 进攻方向规则

足球射门必须写清运动轴线：

```text
进攻方向从画面左侧到画面右侧。
中国队前锋位于画面左中部，朝画面右侧远端唯一球门射门。
足球从画面左中部飞向画面右上远端唯一球门。
足球进入同一个唯一球门的右上角球网。
```

不要只写：

```text
球员射门，足球进球。
```

## 12.4 女球迷 / 前景人物规则

失败表现：

```text
女球迷像贴图一样叠在球门上；
女球迷站到了草坪里；
女球迷朝向不对，正面对镜头而不是看球门；
女球迷遮挡球门。
```

稳定写法：

```text
女球迷站在看台第一排护栏外的水泥地面上，不在草坪上，不在球门线上。
她位于画面左下角前景，只占画面高度约三分之一。
她是三分之二侧背面向球场。
肩膀、胸口、鼻尖和目光都朝向远端右侧球门。
她不看镜头，不正面对镜头，不遮挡球门。
```

## 12.5 前景 / 中景 / 背景分层规则

足球进球镜头建议固定三层：

```text
前景：球迷，站在护栏外。
中景：草坪、禁区线、射门球员、唯一球门。
背景：观众席、大屏幕、旗帜、烟花、彩带。
```

不要让大屏幕、女球迷、球门抢同一个空间层。

## 12.6 大屏幕规则

失败风险：

```text
大屏幕可能显示另一个足球场、另一个球门、另一个比赛画面，造成空间混乱。
```

稳定写法：

```text
大屏幕只显示红金色冠军庆祝图案、奖杯剪影、抽象庆祝画面。
大屏幕不显示另一个足球场，不显示另一个球门，不显示比赛直播画面。
```

## 12.7 足球镜头生成策略

如果文生视频反复出现球门位置错误，直接切换为：

```text
关键帧图片 prompt → 图生视频
```

原因：

```text
足球场几何关系属于构图和空间结构，图片更适合锁定；
视频 prompt 负责射门、入网、球网震动、球迷欢呼和庆祝。
```

# 13. 场地结构物通用规则

适用于：

```text
足球球门
篮球篮筐
排球网
网球网
拳台边绳
擂台角柱
赛道终点线
舞台主屏幕
门、桥、祭坛、黑石、月亮等固定结构物
```

如果关键对象是“场地结构物”，不能只写它出现，必须写：

```text
1. 它位于场地的哪一端；
2. 它和场地线 / 地面结构的关系；
3. 它前方有什么；
4. 它后方有什么；
5. 它位于画面的哪个区域；
6. 它从第一帧开始就已经存在；
7. 它不能漂浮、不能换位置、不能复制成多个。
```

通用模板：

```text
【结构物】从第一帧开始固定在【场地具体位置】。
它前方是【前方空间 / 地面线索】。
它后方是【背景结构】。
它位于画面【区域】。
它完整可见，不被裁切，不被遮挡。
不要复制出第二个【结构物】，不要让它漂浮到场地中段，不要改变它的位置。
```

这条是对“锁定元素”规则的细化：固定结构物还要绑定场地几何关系。

# 14. 战斗打击感实测规则

## 14.1 不要只写“拳拳到肉”

弱提示：

```text
打得很重，拳拳到肉，很有冲击力。
```

问题：

```text
模型可能生成软绵绵挥手、慢速碰撞、拳头滑过身体，没有真实接触感。
```

稳定写法：

```text
发力准备 → 身体发力路径 → 命中位置 → 接触瞬间停顿 → 被击中者反应 → 环境反馈 → 镜头/声音反馈
```

## 14.2 身体发力链规则

打击动作要写：

```text
攻击者压低重心；
后脚稳定踩地；
后脚蹬地发力；
腰胯转动；
肩膀带动手臂；
拳头短距离快速打出；
拳头命中后不要穿过身体；
接触瞬间有短暂停顿。
```

## 14.3 被击中反应链规则

被击中者不能只是“被打到”，要写：

```text
上半身猛然后仰；
脚步后退半步；
肩膀和衣服产生冲击抖动；
头发或衣角有轻微震动；
随后稳住身体，恢复对峙。
```

## 14.4 打击反馈规则

增强打击感可以写：

```text
接触瞬间低频撞击声；
鞋底摩擦地面声；
短促呼吸声；
镜头只有轻微震动；
空气中灰尘或汗水飞散。
```

不要写太多夸张特效，否则容易变成超能力或飞天。

## 14.5 慢动作打击规则

本次实测结果：

```text
48fps动作清晰感 + 低动态模糊 + 接触瞬间短暂停顿
容易生成慢动作重击感。
```

归类：

```text
稳定规则：重击慢动作可用。
```

如果想保留慢动作：

```text
升格慢动作，接触瞬间动作放慢，清楚看到身体发力和冲击反应。
```

如果不想要慢动作：

```text
真实速度动作，不是慢动作。
接触瞬间只有0.2秒顿挫，不要升格慢动作，不要拖慢挥拳过程。
```

## 14.6 战斗镜头安全与稳定建议

建议写成：

```text
电影化舞台格斗 / 动作片打斗 / 训练馆对打 / 安全护具
```

避免写成过度真实伤害。

打击位置优先选择：

```text
胸口护具
肩膀
腹部护具
手臂格挡处
```

高风险位置：

```text
脸部重击
喉咙
眼睛
关节折断
血腥伤害
```

## 14.7 战斗镜头推荐 3–5 秒

一个镜头只做一次清晰打击：

```text
0-1秒：准备；
1-2.5秒：出拳；
2.5-3秒：命中停顿；
3-4秒：被击中反应；
4-5秒：恢复对峙。
```

复杂连招不要一镜硬写，应该拆成多个 3–5 秒镜头，或者上传动作参考视频。
