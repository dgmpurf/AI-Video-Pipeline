# 0. 本次更新结论

V1.5 的核心不是新增某个具体项目 prompt，而是补上 **剧本与美术设计在 AI 视频生产中的前置规范**。

```text
V1.1：主实测规则库，优先级最高。
V1.2：动作、变身、巨物、裂隙、运镜增补。
V1.3：外部案例如何进入 Source 的方法论。
V1.4：外部知识库、GitHub 图片案例、Seedance 教程、AI提示词文档的吸收和改写规则。
V1.5：剧本、美术设计、分镜表、视觉统一、角色动机和可执行性规则。
```

新增核心规则：

```text
1. AI 视频项目不能从 prompt 开始，必须先确定剧情目标、受众、主题和钩子。
2. 剧本必须能拆成镜头，不只是文学文本。
3. 每个镜头必须服务一个叙事功能：交代、推进、转折、反应、情绪、信息、卖点或收束。
4. 角色行为必须由动机驱动，不能“为了镜头而行动”。
5. 情绪要用表情、动作、呼吸、手部、停顿、环境反应来表现，不要只写情绪词。
6. 美术设计必须服务故事，不为炫技而存在。
7. 场景、色彩、服装、道具、光影、构图要形成统一视觉系统。
8. 关键场景要有“场景美术圣经”，保证多个镜头之间空间、道具、色调一致。
9. 分镜表字段必须包含：场景、镜号、画面内容、景别、运镜、音乐/音效、台词、时长、文生图 prompt、图生视频 prompt。
10. IP / 名场面还原类资料只提炼结构，不吸收专有角色、台词和设定。
```

| 资料 | 定位 | 证据等级 | 是否直接进入高优先级 Source |
|---|---|---:|---|

```text

本项目复测成功 → 升级到 B 级。
多次复测成功 → 升级到 A 级稳定规则。
```

# 2. AI 视频项目的前置顺序

不要从一句 prompt 直接开始。

推荐顺序：

```text
确定剧情 / 主题 / 目的
→ 确定主角、反派、关系和动机
→ 确定美术风格和视觉基调
→ 确定场景、服装、道具、色彩、光影
→ 生成角色 / 场景 / 道具资产
→ 写分镜表
→ 写每个镜头的文生图 prompt
→ 生成关键帧
→ 写每个镜头的图生视频 prompt
→ 生成视频片段
→ 剪映 / PR / DaVinci 粗剪、精剪、配音、音效、导出
→ 复盘并更新规则库
```

这与当前项目工作流一致：

```text
故事 -> 风格板 -> 角色/场景/道具板 -> 分镜 -> 镜头清单 -> 构图矩阵 -> 关键帧 -> 多模态参考 -> 视频片段生成 -> 剪辑合成 -> 反馈复盘 -> 更新规则库
```

# 3. 剧本核心规则

## 3.1 脚本必须服务一个核心目的

每个脚本先回答：

```text
这个视频要让观众看到什么？
感受到什么？
记住什么？
产生什么行动？
```

常见目的：

```text
讲故事
塑造人物
展示世界观
传递情绪
科普知识
展示产品
制造反转
引流转化
建立 IP 气质
```

禁止：

```text
一个短片同时讲太多主题。
一个镜头同时承担太多信息。
画面很好看但不服务故事。
```

---

## 3.2 开场必须有钩子

短视频尤其需要前 3–10 秒建立钩子。

钩子类型：

```text
冲突：有人做出反常行为。
悬念：出现不该出现的物体 / 人 / 声音。
共鸣：观众熟悉的痛点或情绪。
视觉奇观：罕见尺度、强构图、强动作。
问题：提出观众想知道答案的问题。
反差：身份、环境、行为之间形成冲突。
```

AI 视频写法：

```text
第一镜头不要只写“美丽场景”。
第一镜头要有问题、冲突、异常、行动或强视觉信息。
```

---

## 3.3 剧情推进链：欲望 → 阻碍 → 行动 → 结果

故事型镜头建议检查：

```text
角色想要什么？
阻碍是什么？
角色采取什么行动？
行动带来什么结果？
结果如何推动下一镜头？
```

如果答不上来，这个镜头可能只是“漂亮废镜”。

---

## 3.4 三幕式与问题解决结构

故事型短片可用三幕式：

```text
建置：人物、场景、目标、关系。
对抗：阻碍、冲突、误解、风险升级。
解决：选择、代价、反转、情感收束。
```

广告 / 科普 / 产品类可用问题解决结构：

```text
痛点 / 问题
→ 产品 / 方法 / 角色介入
→ 过程 / 功能 / 对比
→ 效果 / 结果
→ 情感或品牌收束
```

进入分镜时，不要保留抽象结构名，要翻译成镜头功能。

示例：

```text
SHOT-01：痛点出现。
SHOT-02：主角发现问题。
SHOT-03：道具 / 产品 / 方法出现。
SHOT-04：使用或行动。
SHOT-05：结果展示。
SHOT-06：情绪或品牌 closing frame。
```

# 4. 角色与情感规则

## 4.1 角色必须有差异化标签

角色不应只写：

```text
男主
女主
皇帝
少女
老人
反派
```

要写：

```text
外貌标签
行为习惯
语言方式
价值观
内心缺口
视觉符号
道具关联
```

例：

```text
总把袖口攥皱的年轻修士。
说话前会先低头整理玉佩的女主。
从不直视别人眼睛的老匠人。
背着破旧竹箱、步伐很轻的少年。
```

AI 视频中，差异化标签要能变成画面：

```text
服装
道具
姿态
手部动作
视线习惯
走路方式
常出现的颜色
```

---

## 4.2 角色行为必须由动机驱动

不要让角色“为了剧情需要”突然行动。

每个关键动作前最好有动机或触发点：

```text
看到某物
听到某声
想隐藏真相
害怕失去
想保护别人
被逼到绝境
突然意识到危险
```

AI 视频 prompt 中可写成：

```text
他先听到身后细微响声，肩膀僵住，然后缓慢转头。
她看到镜中人的笑容没有同步，呼吸停顿，手中的毛巾滑落。
他盯着地上的碎玉，眼神从震惊转为愤怒，右手慢慢握紧剑柄。
```

---

## 4.3 情绪要具象化

弱写法：

```text
她很悲伤。
他很愤怒。
少年恐惧。
老人很失望。
```

稳定写法：

```text
她眼眶泛红，嘴角努力压住颤抖，手指紧紧攥住衣袖，却始终没有抬头。
他下颌绷紧，鼻翼轻微张开，右手握拳抵在桌面上，指节发白。
少年缩在衣柜阴影里，眼睛紧张地左右张望，双臂抱紧膝盖，身体轻微发抖。
```

情绪结构：

```text
环境
+ 基础情绪词
+ 面部表情动作
+ 身体动作
+ 呼吸 / 眼神 / 手部细节
+ 镜头距离
+ 光影配合
```

---

## 4.4 台词少而精，动作优先

原则：

```text
能用表情、动作、道具、沉默表达的，不要全靠台词。
关键台词只留最有冲击力的句子。
台词必须符合角色身份、时代、关系和情绪。
```

AI 视频中，台词建议和画面分开处理：

```text
画面 prompt：动作、表情、环境、镜头。
配音台词：后期配音或字幕中实现。
```

不要把大量台词塞进视频生成 prompt，尤其不要指望模型稳定生成口型和可读字幕。

# 5. 节奏与镜头长度规则

## 5.1 短片节奏要张弛有度

情感戏：

```text
慢动作、长停顿、近景、特写、低信息密度。
```

高潮戏：

```text
短镜头、快切、冲突动作、强声响、反应镜头。
```

缓冲段：

```text
空镜、道具特写、环境声、角色沉默。
```

不要让整段视频一直高压、一直炫技、一直慢动作。

---

## 5.2 1 分钟视频分镜数量

通用真人 / 剪辑逻辑中，1 分钟视频通常约 15–20 个分镜，每镜 3–4 秒。

AI 视频生产中可以更细：

```text
情绪对话：每镜 2–4 秒。
动作高光：每镜 3–5 秒。
产品广告：每镜 3–5 秒。
复杂动作：宁愿拆成多个 3–5 秒镜头。
```

如果用户要求“每个镜头不超过 3 秒”，分镜表可以这样写，但生成时可采用：

```text
生成 4-5 秒素材
剪辑时取其中 2-3 秒最稳定部分
```

# 6. 分镜表规则

## 6.1 分镜表必须可执行

分镜表不是文学大纲，必须包含能交给生成工具执行的信息。

最低字段：

```text
拍摄场景
镜号
画面内容
景别
镜头运动
音乐 / 音效
台词 / 旁白
时长
文生图提示词
图生视频提示词
```

建议扩展字段：

```text
镜头功能
角色情绪
前景 / 中景 / 背景
关键道具
参考图职责
锁定元素
上一个镜头延续元素
下一步处理
```

---

## 6.2 分镜表模板

| 字段 | 说明 |
|---|---|
| project_name | 项目名 |
| scene_id | 场景编号 |
| shot_id | 镜头编号 |
| shot_function | 交代 / 推进 / 转折 / 反应 / 高潮 / 收束 / 卖点 |
| location | 场景地点 |
| continuity_from_prev | 与上一镜相同的元素、色调、角色位置、道具 |
| characters | 出镜角色 |
| character_emotion | 当前情绪，必须具象化 |
| visual_content | 画面内容 |
| shot_size | 远景 / 全景 / 中景 / 近景 / 特写 |
| camera_movement | 固定 / 推近 / 拉远 / 跟拍 / 横移 |
| foreground_midground_background | 前中后景 |
| sound_music | 音乐 / 音效 / 环境声 |
| dialogue_vo | 台词 / 旁白，后期处理 |
| duration | 镜头时长 |
| text2image_prompt | 文生图 / 关键帧 prompt |
| image2video_prompt | 图生视频 prompt |
| locked_elements | 不能变的主体、道具、场景、色调 |
| notes | 复盘 / 风险 / 备用方案 |

---

## 6.3 文生图 prompt 与图生视频 prompt 必须分工

文生图负责：

```text
主体是谁
场景在哪里
人物姿态
空间关系
光影色调
美术风格
构图
道具位置
```

图生视频负责：

```text
动作
镜头
表情变化
环境互动
时间线
光线稳定
锁定元素
```

不要把完整剧情全塞进文生图 prompt。

# 7. 分镜连续性规则

## 7.1 同一场景必须保留连续元素

如果一段戏发生在同一个场景中，每个镜头都要写：

```text
同一房间 / 同一宫殿 / 同一浴室 / 同一街道。
同一主色调。
同一光源方向。
同一关键道具位置。
同一人物服装和发型。
同一空间结构。
```

建议字段：

```text
continuity_from_prev：
延续上一镜的烛火、桌案、屏风、人物服装、冷暖色调和压抑氛围。
```

---

## 7.2 对话场景的空间稳定

对话戏常见失败：

```text
角色位置跳变。
两人突然换边。
场景变成另一个房间。
情绪断裂。
人物表情不接上一镜。
```

稳定规则：

```text
固定 A 在画面左侧 / 前景 / 坐位。
固定 B 在画面右侧 / 中景 / 站位。
明确谁看向谁。
用反打时也保持空间轴线。
每个镜头写与上一镜相同元素。
```

模板：

```text
场景仍然是同一间昏暗宫殿，烛火、屏风、桌案和冷暖混合光保持不变。
角色A仍位于画面左侧，角色B仍位于画面右侧，两人始终处于同一空间内。
本镜头只改变景别和情绪，不改变场景结构和人物服装。
```

# 8. 美术设计总规则

## 8.1 美术服务叙事

所有视觉元素都必须服务故事：

```text
场景
色彩
服装
道具
光影
构图
材质
气氛
```

禁止：

```text
为了炫技加入无关元素。
场景很好看但不反映人物状态。
服装华丽但和角色身份不匹配。
光影高级但破坏叙事焦点。
```

一句话：

```text
美术设计不是装饰，是无声叙事。
```

---

## 8.2 风格必须先定位

每个项目都要先定风格：

```text
现实主义
国风写实
国漫 3D
水墨写意
赛博朋克
古典宫廷
玄幻史诗
黑暗奇幻
商业广告高级感
儿童动画
```

然后写清视觉基调：

```text
主色调
光影类型
材质倾向
时代感
空间结构
参考文化符号
构图倾向
```

不要每个镜头都临时换风格。

---

## 8.3 场景即角色

场景不只是背景，它可以表现人物心理和剧情状态。

例：

```text
狭窄低矮空间 → 压迫、阶层困境、无路可退。
开阔高地 → 希望、自由、转折。
潮湿阴暗房间 → 恐惧、秘密、腐败。
高饱和对称空间 → 仪式感、荒诞、秩序。
破碎废墟 → 失控、战后、衰败。
```

AI prompt 中应写成可见结构：

```text
低矮天花板压近人物头顶，狭窄走廊两侧墙面逼近，人物被挤在画面边缘。
```

不要只写：

```text
压抑的场景。
```

# 9. 色彩设计规则

## 9.1 主色调奠定情绪

冷色调：

```text
孤独、压抑、冷静、悬疑、未来感。
```

暖色调：

```text
温暖、浪漫、危险、激烈、回忆。
```

低饱和：

```text
真实、压抑、历史感、沉重。
```

高饱和：

```text
童话、广告、喜剧、梦境、强视觉记忆。
```

对比色：

```text
冲突、分裂、欲望、危险、反差。
```

---

## 9.2 色彩随剧情变化

建议为项目写 Color Script：

```text
开场：主色调。
冲突升级：色彩变化。
高潮：对比增强或色彩极端化。
结尾：回归、反转或余韵色。
```

示例模板：

```text
第一幕：冷青灰，低饱和，表现压抑与未知。
第二幕：暗红与金色逐渐出现，表现危险和欲望。
第三幕：冷白月光压过暖色，表现真相揭开后的空寂。
```

---

## 9.3 角色专属色

核心角色建议有专属色：

```text
主角：冷白 / 青灰 / 月色。
反派：暗红 / 黑金 / 深紫。
神秘角色：银白 / 淡蓝 / 雾灰。
工匠：土黄 / 木褐 / 铁灰。
```

作用：

```text
强化身份记忆。
帮助多镜头中保持视觉一致。
方便生成角色资产和服装材质。
```

# 10. 服装与道具规则

## 10.1 服装是角色的第二层皮肤

服装必须反映：

```text
身份
阶层
时代
性格
心理变化
阵营
职业
地域文化
```

不要只写：

```text
华丽服装。
古装美女。
仙侠长袍。
```

更稳：

```text
深灰粗麻长袍，袖口有手工缝补痕迹，腰间挂旧铜钥匙和磨损布袋，显示长期劳作和贫寒身份。
```

---

## 10.2 服装弧光

角色变化可以通过服装表现：

```text
整洁 → 破损
浅色 → 暗色
普通材质 → 仪式性材质
束缚感服装 → 更自由的衣摆
隐藏身份 → 标志性符号暴露
```

AI 短片中，如果角色跨多镜头变化，必须建立：

```text
A-CH 初始服装资产
A-CH 转折服装资产
A-CH 终局服装资产
```

## 10.3 道具必须有叙事功能

关键道具可用于：

```text
推动剧情

与角色关系

```

# 11. 光影与构图规则

## 11.1 光影引导注意力

光影要解决：

```text
观众应该先看哪里？
谁处于权力中心？
谁被隐藏？
谁处于危险？
情绪是温暖、压抑、恐怖还是希望？
```

不要只写：

```text
电影级光影。
```

更稳：

```text
唯一暖色烛光从画面左下方照亮桌案边缘，人物半张脸在阴影中，眼部有细弱高光，背景屏风沉入暗处。
```

---

## 11.2 光影象征关系

可用规则：

```text
强光区：权力、真相、暴露、神性。
阴影区：秘密、压迫、危险、罪责。
背光剪影：神秘、疏离、威胁。
闪烁光：不安、恐怖、崩坏。
柔和弥散光：回忆、温柔、梦境。
```

注意：若不想出现光柱，要沿用 V1.1 规则：

```text
没有可见光束，没有丁达尔效应，没有手电筒式聚光。
```

---

## 11.3 构图服务心理

对称构图：

```text
秩序、仪式感、压迫、权力结构。
```

失衡构图：

```text
混乱、焦虑、角色失控。
```

框架构图：

```text
窥视、束缚、秘密、被困。
```

边角构图：

```text
孤独、无助、被环境吞没。
```

中心构图：

```text
仪式、审判、英雄登场、产品 hero shot。
```

AI prompt 要写可见空间关系：

```text
角色被门框和窗格夹在画面中央，像被困在狭窄画框中。
人物被放在画面最右侧边缘，左侧大面积阴影空白压迫过来。
```
