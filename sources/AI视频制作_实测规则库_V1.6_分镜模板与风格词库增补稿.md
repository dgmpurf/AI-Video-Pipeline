# 0. 本次更新结论

V1.6 的核心不是新增某个具体项目 prompt，而是建立一套 **分镜模板库与风格词库的使用规则**。

本次吸收资料：

```text
AI视频脚本分镜模板_共300条.xlsx
5000+分镜画面提示词.xlsx
300+电影风格提示词.csv
脚本.xlsx
```

读取结果摘要：

```text
AI视频脚本分镜模板_共300条.xlsx：300 条模板，字段为：序号、场景、镜头动作、开头方式、结尾方式、画面风格。
5000+分镜画面提示词.xlsx：当前上传文件中可读 501 条记录，其中 400 条为结构化分类，101 条为未分类混合记录。
300+电影风格提示词.csv：300 条风格词，分为影视风格、画面质感、灯光色调、摄影手法、平台风格五类，每类 60 条。
脚本.xlsx：21 条名场面对话分镜表样例，字段为：拍摄场景、镜号、画面内容、景别、音乐/音效、台词、时长、文生图提示词、图生视频提示词。
```

新增核心规则：

```text
1. 分镜模板库是“选项池”，不是最终脚本。
2. 风格词库是“美术方向池”，不是 prompt 结尾万能后缀。
3. 电影风格词必须翻译成可见视觉语言，不要直接依赖导演名、IP名、平台名。
4. 5000+画面提示词应按镜头类型、人物动作、环境设定、情绪氛围、时间状态、未分类混合重新归档使用。
5. 300条分镜模板适合生成短视频脚本骨架，但必须再补镜头功能、人物动机、连续性、参考图职责和锁定元素。
6. 脚本表格样例证明：同一场景的多镜头对话戏必须锁定场景、角色位置、色调、光源和情绪递进。
7. 从词库选词时，每个镜头最多选 1 个主镜头类型、1 个主体动作、1 个环境/时间状态、1 个主风格、1 个光影色彩。
8. 不要随机组合“场景 + 镜头动作 + 风格”，必须先由剧情功能驱动。
9. 词库内容必须先转成项目世界观语言，再进入文生图 / 图生视频 prompt。
10. V1.6 后，Source 不再吸收大批量原始词条，只吸收使用规则和精选模板结构。
```

# 2. 分镜模板库的定位

## 2.1 分镜模板不是剧本

`AI视频脚本分镜模板_共300条.xlsx` 的字段是：

```text
场景
镜头动作
开头方式
结尾方式
画面风格
```

这些字段适合快速搭建短视频骨架，但不等于可执行 AI 视频分镜。

它缺少：

```text
镜头功能
角色动机
人物情绪具象化
前景 / 中景 / 背景
参考图职责
连续性锁定
文生图 prompt
图生视频 prompt
素材命名
task_type
```

所以进入本项目时，必须升级成 V1.5 的可执行分镜表。

---

## 2.2 300条模板的正确用途

可以用于：

```text
快速找短视频开头方式；
快速找结尾方式；
快速找场景类型；
快速找镜头动作候选；
快速找画面包装风格；
帮助构建不同版本脚本方案。
```

不建议用于：

```text
随机组合成完整项目；
直接喂给文生视频；
直接变成 DreaminaBatcher manifest；
直接替代剧本和分镜设计；
直接生成连续角色短片。
```

# 3. 分镜模板改写流程

## 3.1 原始模板 → 项目分镜

原始模板：

```text
场景：街头采访
镜头动作：快剪回忆片段
开头方式：开场介绍主题
结尾方式：呼吁点赞评论
画面风格：竖屏构图+强节奏剪辑
```

不能直接使用。

应改写为：

```text
项目场景：某城外旧茶棚 / 月下空地 / 战后船坞 / 宫殿内室。
镜头功能：钩子 / 交代 / 推进 / 转折 / 反应 / 高潮 / 收束。
人物动机：主角为什么进入这个场景，为什么说这句话或做这个动作。
画面内容：主体 + 行为 + 环境 + 空间关系。
文生图 prompt：锁定构图、角色、场景、光影。
图生视频 prompt：锁定动作、镜头、时间线、情绪变化。
```

---

## 3.2 模板改写公式

```text
原始模板字段
→ 剧情功能
→ 项目世界观改写
→ 镜头调度六要素
→ 文生图 prompt
→ 图生视频 prompt
→ 生成任务字段
```

可复用流程：

```text
1. 从 300条模板中选一个“骨架”。
2. 把场景改写为项目内真实场景。
3. 把镜头动作拆成主体动作和摄影机动作。
4. 把开头方式改写为钩子或建立镜头。
5. 把结尾方式改写为情绪收束、悬念、转场或 closing frame。
6. 把画面风格改写为光影、色彩、构图和材质。
7. 写入 V1.5 分镜表字段。
8. 再进入 Composition Matrix 或 prompt。
```

# 4. 5000+ 分镜画面提示词库使用规则

## 4.1 当前可读结构

本次上传的 `5000+分镜画面提示词.xlsx` 当前可读 501 条记录，结构如下：

| 分类 | 数量 | 用途 |
|---|---:|---|
| 镜头类型 | 80 | 机位 / 景别 / 转场 / 摄影方式候选 |
| 人物动作 | 80 | 主体行为、微动作、情绪动作候选 |
| 环境设定 | 80 | 场景种子、空间气氛、地点候选 |
| 情绪氛围 | 80 | 情绪基调、声音、光影、镜头节奏候选 |
| 时间状态 | 80 | 时间、天气、季节、节日、事件状态候选 |
| 未分类混合 | 101 | 需二次分类后使用 |

---

## 4.2 画面提示词不是完整 prompt

例：

```text
鸟瞰镜头：从高空俯拍城市街区，车流穿梭，建筑鳞次栉比
女孩奔跑在雨中，镜头跟随其背影快速移动
城市夜景：灯火辉煌，街道车流不息
紧张：人物喘息声，镜头快速摇晃切换
清晨：城市刚苏醒，街道空荡，晨光温柔
```

这些是 **片段词**，不是最终 prompt。

最终 prompt 还需要补：

```text
项目ID；
主体身份；
角色参考图职责；
场景参考图职责；
前景 / 中景 / 背景；
主体站位；
视线方向；
运动路径；
锁定元素；
不允许变化项。
```

---

## 4.3 片段词组合上限

每个镜头最多选择：

```text
1 个镜头类型；
1 个人物动作；
1 个环境设定；
1 个情绪氛围；
1 个时间状态；
1 个主风格 / 光影方向。
```

不要同时堆：

```text
鸟瞰 + 特写 + 旋转 + 跟拍 + 快切 + 慢动作 + 长镜头
```

如果想用多个镜头类型，必须拆成多个 SHOT。

---

## 4.4 片段词项目化模板

原始片段：

```text
女孩奔跑在雨中，镜头跟随其背影快速移动。
```

项目化：

```text
角色A 从画面左下方湿滑石阶上快速奔跑，背对镜头，肩膀和发丝被雨水打湿。
摄影机位于角色身后偏低位置，低机位跟拍，不绕到正面。
前景有雨滴打在镜头边缘，中景是角色奔跑路径，背景是昏暗巷口。
保持角色服装、发型、场景结构和雨夜冷蓝色调不变。
```

# 5. 未分类混合词的处理规则

`5000+分镜画面提示词.xlsx` 中有 101 条未分类混合记录，例如：

```text
男人倚窗抽烟，烟雾缭绕眼神游离。
地铁站台：列车进站，警示灯闪烁。
欣慰：人物望向成果微笑，音乐温暖。
肩膀视角镜头：镜头从人物肩膀后紧随其步伐。
少女一边跳舞一边转圈，裙摆飞扬。
```

这些不能直接作为同一类处理。必须先二次分类：

| 类型 | 判断方法 | 进入字段 |
|---|---|---|
| 主体行为 | 有人物 + 动作 | visual_content / image2video_prompt |
| 场景环境 | 有地点 + 环境状态 | location / text2image_prompt |
| 情绪氛围 | 情绪词 + 表现方式 | character_emotion / sound_music |
| 镜头方式 | 明确镜头 / 机位 / 视角 | camera_movement / shot_size |
| 时间天气 | 时间、季节、天气、节日 | time_state / light_color |
| 复合片段 | 同时包含多类信息 | 拆分后使用 |

规则：

```text
未分类混合词不能直接进 prompt。
必须先拆成：主体、动作、环境、机位、情绪、光影、声音。
```

# 6. 300+ 电影风格词库使用规则

## 6.1 当前结构

`300+电影风格提示词.csv` 当前包含 300 条，五类：

```text
影视风格：60 条
画面质感：60 条
灯光色调：60 条
摄影手法：60 条
平台风格：60 条
```

这五类不能同等使用。

---

## 6.2 五类风格词的正确位置

| 类型 | 用途 | 是否进最终 prompt |
|---|---|---|
| 影视风格 | 观察美学方向 / 找视觉参考 | 慎用，优先改写 |
| 画面质感 | 胶片、颗粒、材质、清晰度 | 可用，但只选 1 个主质感 |
| 灯光色调 | 主光、环境光、色彩基调 | 可用，必须写光源逻辑 |
| 摄影手法 | 手持、航拍、POV、环绕等 | 可用，但每镜最多 1 个主手法 |
| 平台风格 | 小红书、抖音、B站等包装语感 | 不建议进生成 prompt，适合发布包装 |

---

## 6.3 影视风格词要视觉化

原始词：

```text
王家卫电影风格：慢镜头+模糊边缘+饱和色彩
诺兰式倒叙结构：非线性时间轴+暗色调
黑泽明剧场感：高反差黑白影调+固定机位
宫崎骏动画感：自然色彩+清新构图+手绘质感
漫威大片感：大场景+爆炸特效+CG合成
```

不建议直接写：

```text
王家卫风格
诺兰风格
宫崎骏风格
漫威风格
```

更稳写法：

```text
慢速镜头，边缘轻微柔焦，室内霓虹反光，红绿高饱和色块，人物在狭窄空间中沉默对视。
非线性回忆段落，暗色调，冷白顶光，低饱和写实画面，时间信息通过道具和场景变化区分。
高反差黑白影调，固定机位，舞台式横向空间，人物站位清晰，光影形成强烈剪影。
自然清新的手绘质感，柔和绿色与天空蓝，低对比阳光，风吹草叶和云影缓慢移动。
大型灾难场景，广角全景，烟尘、碎石、强逆光和远处爆炸作为背景层，不让特效遮挡主体。
```

规则：

```text
具体导演 / IP / 品牌 / 平台名 → 只作为内部参考。
进入 prompt 时必须转译为：色彩、光影、镜头、构图、材质、节奏。
```

---

## 6.4 平台风格不等于画面风格

原始词：

```text
小红书爆款视频风格
抖音人物特写
Tiktok 变装卡点
Bilibili 二次元 UP 主风格
电商短视频展示风
```

这些更适合：

```text
封面
标题
剪辑节奏
字幕包装
发布平台适配
BGM选择
文案风格
```

不适合直接作为：

```text
角色关键帧 prompt
剧情镜头 prompt
资产生成默认风格
```

平台风格应进入 `EDIT / PUBLISH` 阶段，而不是 `SHOT-KF / SHOT-V` 阶段。

# 7. 风格词组合规则

## 7.1 每个镜头只保留一个主风格

稳定组合：

```text
主风格：国风写实
画面质感：轻微胶片颗粒
灯光色调：冷白月光 + 弱暖烛光
摄影手法：缓慢推近
```

高风险组合：

```text
王家卫 + 诺兰 + 黑泽明 + 宫崎骏 + 漫威 + 小红书 + 抖音 + 赛博朋克
```

风险：

```text
风格互相抵消；
模型抓不到主体；
画面像混乱风格拼贴；
角色一致性降低。
```

## 7.2 风格选择顺序

先选：

```text
项目主风格
```

再选：

```text
场景色彩
```

再选：

```text
单镜头光影
```

最后选：

```text
局部质感 / 摄影手法
```

不要先选一堆电影风格，再硬套剧情。

# 9. 词库到分镜表的标准转换

## 9.1 推荐字段

在 V1.5 分镜表基础上，V1.6 建议新增可选字段：

```text
template_source
template_scene
template_camera_action
template_opening
template_ending
template_style
shot_prompt_type
action_fragment
environment_fragment
mood_fragment
time_fragment
style_fragment
style_translation
```

用途：

```text
记录这个镜头是从哪个模板 / 词库改写而来；
便于复盘哪些模板有效；
便于后续自动化生成和 A/B 测试。
```

---

## 9.2 从模板到 prompt 的字段映射

| 词库字段 | 进入分镜字段 | 注意 |
|---|---|---|
| 场景 | location / visual_content | 必须项目化 |
| 镜头动作 | camera_movement 或 visual_content | 区分摄影机动作和主体动作 |
| 开头方式 | shot_function / hook | 改成钩子或建立信息 |
| 结尾方式 | shot_function / transition_to_next | 改成转场、悬念或收束 |
| 画面风格 | style_fragment / light_color | 视觉化，不堆词 |
| 镜头类型 | shot_size / camera_position | 每镜最多一个主镜头 |
| 人物动作 | visual_content / image2video_prompt | 必须补终点和情绪 |
| 环境设定 | location / text2image_prompt | 必须补主体关系 |
| 情绪氛围 | character_emotion / sound_music | 具象化 |
| 时间状态 | time_state / light_color | 影响光源和天气 |
| 风格提示词 | style_translation | 必须转译 |

# 10. 自动组合时的禁止规则

如果以后用脚本或 LLM 自动组合词库，禁止：

```text
随机选一个场景 + 随机选一个动作 + 随机选一个风格直接输出。
```

必须先给出：

```text
项目类型；
故事目的；
主角；
镜头功能；
场景连续性；
角色当前情绪；
上一镜状态；

```

自动组合顺序：

```text
先确定 shot_function。

再确定 location/time。
再确定 camera。
再确定 light/style。
最后添加 locked_elements。
```

# 12. Prompt 片段库使用检查清单

```text
这个词库片段是否服务当前镜头功能？
有没有把通用场景改成项目场景？
有没有把通用人物改成项目角色？

有没有写参考图职责？
```
