# 0. 本次更新结论

本次更新不是新增单个项目 prompt，而是补充一类稳定问题：

```text
固定场景参考图中，需要同时控制：
1. 背景镜头位置不变；
2. 前景空地保留；
3. 近景/中景/远景人物分层；
4. 双人对峙方向；
5. 近大远小比例；
6. 角色外观参考不干扰空间站位；
7. 近景角色不要贴边，脚下落点要进入指定场地区块。
```

本轮赤焰天穹古寺庭院 SHOT-01-KF 实测表明：

```text
复杂双人对峙关键帧中，模型容易把“角色参考图”理解为“需要近景展示的主体”。
当远景角色使用完整角色参考图时，模型常把该角色拉近、放大、提高存在感。
当近景角色需要从画面边缘往内收时，单纯写“往里一点、后退一点、缩小一点”效果有限。
更有效的是：用边框约束、脚下落点和明确场地区块，把角色钉在世界空间中的位置。
```

新增核心规则：

```text
1. 场景构图锁定与角色外观锁定必须分层。
2. 远景角色不能只写“站远一点”，必须降低其展示优先级。
3. 远景角色可使用文字轮廓替代完整角色参考图。
4. “男远女近 / 男小女大”等关系要成对表达。
5. 前景空地不是气氛描述，而是空间控制锚点。
6. 站位与朝向必须分开描述。
7. 长 prompt 会稀释空间主约束；即梦更吃短、狠、单线程的空间描述。
8. 近景角色内收时，优先使用“不要贴边框 + 脚下落点 + 场地区块”，而不是“往里一点 / 后退一点 / 略微缩小”。
9. 若同一 prompt 已经把空间推到 75%-85% 接近，但远景角色仍不稳定，应切换为构图锚图、局部编辑或接受候选，而不是无限堆长 prompt。
```

# 1. 问题表现

在赤焰天穹古寺庭院双人对峙关键帧中，使用：

```text
A-SC-TEMPLE-COURTYARD-001 古寺庭院场景图
A-CH-A-SUBJECT-001 焚狩主体角色图
A-CH-B-SUBJECT-001 霜玑主体角色图
```

多次生成出现以下问题：

```text
1. 两个角色都被放到前景或中前景，像双人展示图。
2. 男角色本应处于远处，却被拉近、放大，成为近景主角。
3. 女角色位置逐渐正确，但朝向容易变成正面展示。
4. 如果强写“只有两个人”，模型可能删除正确远处角色，保留错误近处角色。
5. 如果强调角色外观完整，模型会提高角色展示权重，破坏远近关系。
6. “左边 / 右边 / 面对彼此”不足以稳定表达世界空间中的远近层级。
7. “中景对峙”“角色清楚可见”“完整展示两人”等词会把远景角色拉近。
8. 霜玑会贴在右前景边缘，像前景展示主体，而不是站在广场内部。
9. 写“向画面内部回收 / 略微缩小 / 后退一点”后，女角色可能仍然偏近、贴边或脚下落点不对。
```

本质问题：

```text
这是“空间调度失败”，不是单纯角色失败或场景失败。
模型在处理角色参考图时，会优先展示角色，而不是遵守远近 blocking。
模型对“人物位置”的理解经常偏向画面二维位置和主体展示，而不是地面脚点和世界空间落点。
```

# 2. 规则 A：场景构图锁定与角色外观锁定分离

## 2.1 稳定原则

当已有满意的场景参考图时，场景图应负责：

```text
镜头位置
画面构图
透视关系
前景/中景/背景
建筑布局
光影气氛
```

角色图应只负责：

```text
脸型 / 发型 / 服装 / 配色 / 体型 / 气质 / 材质
```

不要让角色图负责：

```text
站位
姿态
透视比例
画面层级
谁在前景谁在远景
```

## 2.2 稳定写法

```text
@场景图 作为唯一场景构图与镜头锁定参考。完全保留镜头位置、透视关系和画面构图不变。
@角色图 只作为角色外观参考，只保持发型、服装、体型和气质，不用于站姿构图。
```

## 2.3 风险写法

```text
参考这些图生成两人对峙。
保持角色完整清楚，站在场景里。
```

风险原因：

```text
模型可能把角色参考图理解为“角色展示主体”，导致角色被拉近、放大。
```

# 3. 规则 B：远景角色降权法

## 3.1 稳定原则

如果某个角色应处于远景，不要只写：

```text
角色站远一点。
角色更小一点。
```

必须明确降低该角色的视觉优先级：

```text
该角色在画面中的存在感要被主动压低。
该角色属于远景对峙目标，而不是近景展示主体。
该角色只需作为远处识别轮廓出现。
```

## 3.2 高价值有效表达

```text
焚狩在画面中的存在感要被主动压低，属于远景对峙目标，而不是近景展示主体。
```

本句在赤焰天穹古寺庭院对峙测试中显示明显有效，能降低模型把焚狩拉近展示的倾向。

## 3.3 可复用模块

```text
【远景角色】在画面中的存在感要被主动压低，属于远景对峙目标，而不是近景展示主体。
【远景角色】位于画面远处，只需看出较小的人物轮廓和主色块，不要求近景细节展示。
不要把【远景角色】拉到前景，不要让他/她成为近景主角。
不要让【远景角色】进入中景主体区，不要让【远景角色】变成清晰大人物。
```

# 4. 规则 C：远景角色参考降权 / 文字轮廓替代法

## 4.1 问题

完整角色参考图会提高角色权重。若远景角色使用完整主体参考图，模型可能自动：

```text
放大角色
拉近角色
展示角色脸和服装细节
让该角色成为前景主体
```

## 4.2 稳定策略

当空间站位优先于角色细节时：

```text
1. 保留场景参考图；
2. 保留近景/中景主承载角色参考图；
3. 远景角色暂时不用完整角色参考图；
4. 用文字描述远景角色轮廓、颜色和身份；
5. 先把远近空间关系做对，再考虑补外观细节。
```

## 4.3 可复用写法

```text
远处男性角色为黑发高束、黑红重甲的小轮廓，只需清楚辨认红黑主色块和男性武者形象。
不要把远处男性角色做成清晰近景展示人物。
```

## 4.4 适用条件

适用于：

```text
远景角色只需要作为对峙目标、敌人、远处观察者、远方来者出现；
不需要在该关键帧里展示其脸部细节和服装结构；
角色外观可在其他资产图或后续镜头中补足。
```

# 5. 规则 D：远近关系必须成对表达

## 5.1 稳定原则

不要只描述一个角色的位置，应成对描述：

```text
A 远 / B 近
A 小 / B 大
A 远景目标 / B 近景承载主体
A 降权 / B 主承载
```

## 5.2 有效表达

```text
男角色远，女角色近；男角色小，女角色大；前景空地明显；镜头和背景构图不变。
```

```text
焚狩位于画面左侧远处庭院角落，站在远处建筑前方，只能看到较小的人物轮廓，必须明显更远、更小，严格遵循近大远小。

霜玑位于画面中央偏右的近处中景，站在前方广场区域，人物尺寸明显大于焚狩。
```

## 5.3 风险写法

```text
焚狩在左边，霜玑在右边。
两人面对彼此。
```

风险原因：

```text
模型可能只执行左右站位，不执行远近层级。
```

# 6. 规则 E：前景空地是空间控制锚点

## 6.1 稳定原则

在广场、庭院、街道、球场、山门前等大空间中，前景空地不是气氛词，而是用于控制纵深的空间锚点。

必须明确：

```text
前景保留大面积空地
前景无人站立
人物进入中景/远景
两人之间隔着空地
```

## 6.2 可复用写法

```text
前景保留大面积无人站立的湿润石板地，用来体现空间深度、雨水反光和电影感。
两人隔着大面积空旷石板地面对面对峙。
```

## 6.3 风险写法

```text
湿石板地有反光。
场景很有纵深。
```

风险原因：

```text
这只是氛围描述，不能防止角色被拉到前景。
```

# 7. 规则 F：站位与朝向分开描述

## 7.1 问题

模型可能做到：

```text
人物位置正确，但朝向错误。
```

例如：

```text
女角色站在正确近景层，但正面看向镜头，而不是看向远处对手。
```

## 7.2 稳定规则

站位和朝向必须分开写：

```text
位置：位于画面哪个空间层、哪个场景锚点附近、远近关系如何。
朝向：身体朝哪里、头部朝哪里、目光看向谁、是否允许看镜头。
```

## 7.3 有效表达

```text
霜玑位于画面右侧前中景，身体朝向左前方，视线明确看向左后方远处的焚狩，不要正面朝向镜头。

焚狩位于画面左侧远景，身体朝向右侧，头部和目光看向霜玑。
```

# 8. 规则 G：长 prompt 会稀释空间主约束

## 8.1 问题

本轮实测中，过长 prompt 虽然包含很多限制，但效果反而不如短句 agent prompt。

原因：

```text
当 prompt 同时强调角色外观、完整展示、对峙、动作、构图、方向、负面限制时，模型会分散注意力。
```

## 8.2 稳定策略

当目标是强空间调度时，优先使用短、狠、单线程写法：

```text
1. 锁场景构图不变；
2. 写远景角色极远 / 小轮廓；
3. 写近景角色近处中景 / 更大；
4. 写近大远小；
5. 写不要多余人物。
```

## 8.3 有效短 prompt 结构

```text
完全保留参考图雨中古寺庭院的镜头位置和画面构图不变，大面积空前景湿润石板地带强烈反光积水质感，彩色经幡、木质大殿、左后方塔楼、冷色雨雾氛围全部原样保留。

极远处左侧庭院角落位置只能看到极小轮廓的黑发高束黑红重甲男性角色，近处中景位置站立银蓝高束发身着蓝白银轻甲长衣的女性角色，近大远小透视自然，整体半写实3D武侠电影关键帧质感，冷灰蓝低饱和冷雨光线，无多余人物无文字水印。
```

# 9. 规则 H：近景角色内收必须用“边框约束 + 脚下落点 + 场地区块”

## 9.1 问题表现

霜玑等近景角色容易出现：

```text
贴右边框；
站在右前景边缘；
视觉上好像缩小了，但脚下仍在边缘；
画面右侧没有呼吸空间；
像贴边前景展示主体，而不是站在广场内的近处中景人物。
```

## 9.2 本轮实测修正

以下两句在本轮测试中被证明**效果有限**，不能作为核心控制语：

```text
霜玑从右前景边缘向画面内部回收，人物落点位于右侧偏中的广场区域，右侧要保留明显呼吸空间。
霜玑人物尺寸略微缩小一点，并向画面内部后退一点，但仍明显大于远处焚狩。
```

它们可以作为辅助说明，但不足以稳定解决贴边/偏前问题。

真正更有效的是以下硬约束：

```text
霜玑不要贴右边框，霜玑脚下落点位于画面右侧偏中的石板广场内侧区域。
```

## 9.3 为什么有效

```text
“不要贴右边框”是明确禁止项；
“脚下落点”直接约束角色与地面的关系；
“石板广场内侧区域”把角色绑定到场景空间区块，而不是只控制画面中的视觉大小。
```

## 9.4 可复用 Prompt 模块：近景角色内收

```text
【近景角色】位于画面右侧偏中的近处中景，站在主殿前方的石板广场内侧区域。
【近景角色】不要贴右边框，脚下落点必须位于画面右侧偏中的石板广场内侧区域。
【近景角色】不是贴边前景展示主体，而是位于广场内部的近处中景人物。
【近景角色】身体朝向画面左侧远处目标，视线明确看向该目标，不要正面看镜头。
```

## 9.5 高风险弱描述

以下词语可以作为补充，但不应作为核心站位控制：

```text
往里面一点
后退一点
略微缩小
靠里一点
不要太靠边
不要太近
站在中景
从边缘回收
向画面内部后退
```

风险原因：

```text
这些词缺少明确的地面区域、边界约束和脚下落点，模型可能只做视觉大小变化，而不改变真实空间落点。
```

# 10. SHOT-01 古寺庭院对峙实测有效基准结构

以下结构是本轮最值得保留的验证样本，应作为后续同类镜头的基准模板，而不是逐字固定模板。

## 10.1 基准结构

```text
@场景图 作为唯一场景构图与镜头锁定参考。完全保留 @场景图 的镜头位置、透视关系和画面构图不变：雨中古寺庭院、湿润石板广场、大面积前景空地、强反光积水、主殿、左后方塔楼、彩色经幡、冷灰蓝雨雾氛围全部保留。

@霜玑角色图 作为霜玑角色外观参考，只用于角色外观，不用于站姿构图。保持银蓝高束长发、蓝白银轻甲长衣、女性气质。

生成一张半写实3D武侠电影关键帧。

画面中只有两个人：一个焚狩，一个霜玑。没有第三人，没有重复角色，没有分身。

最重要的空间关系如下：

焚狩位于画面左侧极远处庭院角落，站在远处建筑与回廊前方，只能看到很小的黑红男性轮廓。焚狩必须明显更远、更小，严格遵循近大远小。焚狩在画面中的存在感要被主动压低，属于远景对峙目标，而不是近景展示主体。不要把焚狩拉近，不要让焚狩进入中景主体区，不要让焚狩变成清晰大人物。

霜玑位于画面右侧偏中的近处中景，站在前方广场内部，但比当前常见结果更往画面里面收一点，不要贴右边框，不要站在最靠镜头的位置。霜玑仍然明显比焚狩更近，因此人物尺寸明显大于焚狩，但请把霜玑控制在“近处中景”而不是“超近景前景特写”。霜玑身体为侧身或轻微侧背，朝向左侧远处的焚狩，视线明确看向焚狩，不要正面看镜头。霜玑不要贴右边框，霜玑脚下落点位于画面右侧偏中的石板广场内侧区域。

两人隔着大面积空旷湿石板地面对面对峙。重点是：男角色远，女角色近；男角色小，女角色大；焚狩保持左侧远景小轮廓；霜玑位于右侧偏中的近处中景；前景空地明显；镜头和背景构图不变。

保留冷灰蓝低饱和雨天光线、湿石板反光、细密雨丝、电影感、肃杀安静氛围。

不要文字，不要水印，不要设定卡，不要三视图，不要四联图，不要多余人物，不要第二个焚狩，不要第二个霜玑，不要把焚狩拉近，不要让霜玑贴到右边缘，不要让霜玑正面朝向镜头。
```

## 10.2 基准结构中的核心有效点

```text
1. 场景图作为唯一构图锁定参考。
2. 霜玑角色图只负责外观，不负责站姿构图。
3. 焚狩不挂强角色参考图时，更容易被压成远景小轮廓。
4. 焚狩不仅写远，还写“存在感主动压低 / 远景对峙目标 / 不是近景展示主体”。
5. 霜玑不仅写右侧中景，还写“不要贴右边框 / 脚下落点 / 石板广场内侧区域”。
6. 人数限制、重复角色限制必须保留。
```

# 11. 失败/风险词汇表

## 11.1 容易拉近远景角色的词

```text
两人都清楚可见
完整展示两人
中景对峙
角色主体突出
双人并列
角色居中
焚狩蓄势前冲

沉重突进
前冲
突进
准备扑向对方
```

风险原因：

```text
这些词会把远景角色拉近，或把对峙图变成双人展示图 / 动作冲刺图。
```

## 11.2 对近景角色内收控制较弱的词

```text
往里面一点
后退一点
略微缩小
回收一点
靠里一点

```

风险原因：

```text
这些词太软，缺少明确空间锚点，模型可能只做局部视觉微调，而不改变脚下落点。
```

# 12. 证据等级与后续复测

当前等级：B 级暂行规则。

原因：

```text
本项目同一镜头多次实测显示有效趋势，但尚未在其他场景、其他人物、其他平台中重复验证。
```

升级到 A 级需满足：

```text
1. 至少两个不同镜头复测有效；
2. 至少两个不同场景类型复测有效，例如庭院 + 街道 / 广场 / 山门；
3. 生成 3-5 张中至少 2 张稳定体现远近关系；
4. 未出现显著新副作用，例如远景角色丢失、重复角色、场景大幅改构图；
5. “脚下落点 + 场地区块”在其他近景角色内收任务中同样有效。
```

# 13. 一句话总结

```text
固定场景里的远近对峙，不是靠“角色站左/右”解决，而是靠“场景构图锁定 + 远景角色降权 + 近大远小成对表达 + 前景空地锚定 + 朝向单独约束”解决。

当近景角色贴边或过近时，不要只写“往里一点 / 缩小一点”，要写“不要贴边框 + 脚下落点 + 明确场地区块”。
```

# V1.9.2 双远景同层左右分置规则

> 建议合并位置：`AI视频制作_实测规则库_V1.9_空间调度与远近站位控制_完整版_修正版.md` 之后  
> 来源：赤焰天穹 / 古寺庭院 SHOT-01-KF 双远景同层对峙分支实测  
> 证据等级：B 级暂行规则。本项目同一场景内多轮对比有效，尚需在其他场景复测。  
> 适用范围：固定场景参考图中的“双远景同层对峙”“左右分置远景人物”“大面积前景空地 + 两端远景站位”镜头。

---

## 1. 本次补充结论

当目标从“女近男远”改成“双远景同层对峙”时，核心控制目标会发生变化：

```text
女近男远：重点是压低远景角色，同时保留近景主控角色。
双远景同层：重点是同时压低两名角色，并锁定左右分置的远处落点。
```

本轮实测表明：

```text
“远近”相对容易被执行；
“左右具体落点”更容易漂移；
如果两人共享同一个中心建筑锚点，模型会把两人收拢到主殿前同一片区域，甚至产生重复人物。
```

因此，双远景同层对峙必须同时写清：

```text
1. 两人都远；
2. 两人都小；
3. 两人处于近似同一纵深层；
4. 两人分别占据左右两个远处区块；
5. 中间必须留出大面积空旷石板地；
6. 禁止把两人收拢到同一侧或同一片主殿前区域。
```

---

## 2. 问题表现

在古寺庭院场景中，将霜玑也推到远景后，出现过以下问题：

```text
1. 女角色确实远了，但没有落在期望的主体建筑前区域；
2. 两人都远了，但被模型收拢到右侧中后部；
3. 焚狩被从左侧远景拉到主殿前中部；
4. 左侧残留或新增一个黑红远景小人，出现重复焚狩风险；
5. “主殿正门前方 / 石阶下方 / 石栏杆前方左右侧”这类中心化锚点，会让模型优先围绕主殿正门生成两人，而不是保持左右分置。
```

本质问题：

```text
模型能理解“两人都远”，但不一定能理解“两人远且左右分开”。
如果锚点太集中，模型会牺牲左右分离，优先生成一个构图集中、人物更清楚的中后景对峙画面。
```

---

## 3. 规则 A：双远景同层时，不要让两人共享同一个中心建筑锚点

## 3.1 风险写法

以下写法在“左右分置”任务中存在风险：

```text
焚狩和霜玑都位于主殿正门前方区域。
二人的脚下落点都围绕主殿正门前方展开。
焚狩在主殿石阶下方左侧，霜玑在主殿石阶下方右侧。
两人都在主殿前方广场的中后部区域。
```

这些句子不是绝对不能用，但如果目标是“左右明显分开”，它们容易造成：

```text
1. 两人被吸到主殿正门附近；
2. 焚狩被拉近到中景主体区；
3. 两人站得过近；
4. 左侧原本应有的远景位置丢失；
5. 生成重复远景小人。
```

## 3.2 稳定原则

双远景同层对峙要用**左右独立场地区块**，不要只用一个共享中心锚点。

更稳结构：

```text
焚狩绑定左侧远处区块；
霜玑绑定右侧远处区块；
两人之间保留空旷广场；
禁止两人收拢到同一片主殿前区域。
```

---

## 4. 规则 B：左右独立区块锚定法

## 4.1 焚狩左侧远处区块

比“主殿正门前方左侧”更稳的写法：

```text
焚狩位于画面左侧远处区域，落点在左侧回廊与左后方建筑前的湿石板地带，只能看到较小的黑红男性轮廓。
```

这个写法的价值：

```text
1. 明确焚狩属于左侧远处区块；
2. 绑定左侧回廊和左后方建筑；
3. 避免焚狩被拉到主殿正门中部；
4. 继续保留远景降权。
```

## 4.2 霜玑右侧远处区块

比“主殿正门前方右侧”更稳的写法：

```text
霜玑位于画面右侧远处区域，落点在主殿右前方广场的右侧内侧区域，只能看到较小的银蓝女性轮廓。
```

这个写法的价值：

```text
1. 明确霜玑属于右侧远处区块；
2. 仍然和主殿右前方广场有关，而不是完全跑到右侧建筑边缘；
3. 避免霜玑停留在右前景主体位；
4. 继续保留远景降权。
```

---

## 5. 规则 C：横向分离必须硬写

仅写“焚狩在左、霜玑在右”不够。必须加横向分离和禁止收拢：

```text
两人必须分别占据画面左右两侧的远处区域，不能靠在一起，不能同时落在同一侧区域。
两人之间必须隔着明显的大面积空旷湿石板地，中间留出清楚的对峙空间。
焚狩在左，霜玑在右，二人横向距离要明显拉开。
不要把两人收拢到主殿前同一片区域，不要让两人站得过近。
```

这组句子在本轮实测中比“二人都围绕主殿正门前方展开”更符合左右分置目标。

---

## 6. 可复用 Prompt 模块：双远景同层左右分置

```text
画面中只有两个人：一个焚狩，一个霜玑。没有第三人，没有重复角色，没有分身。

最重要的空间关系如下：

焚狩位于画面左侧远处区域，落点在左侧回廊与左后方建筑前的湿石板地带，只能看到较小的黑红男性轮廓。焚狩必须明显远离镜头，属于远景小人物，严格遵循近大远小。焚狩在画面中的存在感要被主动压低，属于远景对峙目标，而不是近景展示主体。不要把焚狩拉近，不要让焚狩进入中景主体区，不要让焚狩变成清晰大人物。

霜玑位于画面右侧远处区域，落点在主殿右前方广场的右侧内侧区域，只能看到较小的银蓝女性轮廓。霜玑也必须明显远离镜头，人物尺寸与焚狩接近，可略大一点点，但仍然严格属于远景层。霜玑在画面中的存在感同样要被主动压低，属于远景对峙目标，而不是近景展示主体。不要把霜玑拉近，不要让霜玑进入中景主体区，不要让霜玑变成清晰大人物。霜玑身体轻微侧身，朝向左侧的焚狩，视线明确看向焚狩，不要正面看镜头。

这是双远景同层对峙。两人必须分别占据画面左右两侧的远处区域，不能靠在一起，不能同时落在同一侧区域。两人之间必须隔着明显的大面积空旷湿石板地，中间留出清楚的对峙空间。焚狩在左，霜玑在右，二人横向距离要明显拉开。不要把两人收拢到主殿前同一片区域，不要让两人站得过近。

重点是：两人都远，两人都小，但左右明确分开；焚狩是左侧远景小轮廓，霜玑是右侧远景小轮廓；二人之间有明显的大面积空广场；镜头和背景构图不变。
```

## 7. 风险词与风险结构

## 7.1 容易让两人收拢的结构

```text
两人都位于主殿正门前方。
两人都在主殿石阶下方、石栏杆前方。
两人的脚下落点都围绕主殿正门前方展开。
两人都位于主殿前方广场的中后部区域。
```

风险：

```text
当目标是左右分置时，这些中心化锚点可能把两人吸到主殿中部附近，削弱左右距离。
```

## 7.2 容易拉近角色的结构

```text
两人尺寸接近且都清楚可见。
双人清晰对峙。
完整展示两人。
主殿前双人对峙。
中景双人对峙。
```

风险：

```text
模型会为了清晰展示两人，把其中一个或两个角色拉近到中景主体区。
```

## 8. 与 V1.9 / V1.9.1 的关系



```text

```

三者不是互相替代，而是对应三种不同站位目标：

```text
1. 女近男远：用 V1.9。
2. 女近但不要贴边：用 V1.9.1。
3. 双远景同层左右分置：用 V1.9.2。
```
