# 0. 本次更新结论

本次 V1.3 更新的核心不是新增某个具体项目 prompt，而是新增一套 **Source 进化方法**。

```text
我们不是把别人的 prompt 变成 source。
我们是把别人案例暴露出的模型规律，变成 source。
```

新增核心规则：

```text
1. 外部公开案例只能作为观察样本，不能直接作为项目资产。
2. 不复制外部 prompt 原文，只提炼结构、触发词、风险词和修正方法。
3. 图片 prompt 可以采用“自然语言画面描述 + 短词美学描述”的结构。
4. 构图词必须翻译成可见空间关系，不能只写“大师构图 / 黄金螺旋 / 强透视”。
5. 关键帧少放动态特效，视频 prompt 负责爆发、扩散、飞散、流动、变身过程。
6. 同一 prompt 在不同模块中差异很大，记录时必须标注平台、模型和模块。
7. 评价图片和视频时，全部结果轻记录，少数高信息量结果深分析。
8. Pro / Think 模式主要用于批量归纳规律，而不是逐张图写观后感。
```

# 1. 外部公开案例的定位

## 1.1 外部案例定义

外部公开案例包括：

```text
即梦社区公开作品
可灵社区公开作品
Runway / Luma / Hailuo 等平台公开作品
Midjourney / ComfyUI / Stable Diffusion 等公开图像案例
他人公开发布的 prompt、图片、视频、生成结果和作者评论
```

这些内容在本项目中只作为：

```text
模型行为观察样本
prompt 结构研究样本
风险词与触发词研究样本
构图、动作、特效、光影的观察材料
```

不作为：

```text
项目资产
locked_refs
正式角色 / 场景 / 道具参考图
正式商业素材
可直接复制使用的 prompt 模板
```

---

## 1.2 外部案例使用原则

可以做：

```text
观察某个词为什么有效。
观察某个词为什么失效。
观察某个构图词为什么没有执行。
观察某个动作为什么错位。
观察某个特效为什么死板。
观察同一 prompt 在不同模块中为什么差别大。
把观察结果提炼为待复测假设。
```

不建议做：

```text
直接复制他人完整 prompt 作为项目正式 prompt。
直接把他人生成图作为项目参考图。
直接把他人图放入 00_locked_refs。
未经复测就把外部观察升级为稳定规则。
把外部案例的成功等同于本项目可复用。
```

一句话规则：

```text
外部案例是矿石，不是成品；source 里只放提炼后的规则，不直接堆别人 prompt。
```

# 2. 证据等级规则

为了避免外部案例污染正式 source，所有规则必须标注证据等级。

```text
A级：本项目重复验证 2 次以上，可作为稳定规则。
B级：本项目单次验证，可作为暂行规则。
C级：外部公开案例观察，只能作为假设规则。
D级：网上教程 / 经验说法，仅供参考。
```

## 2.1 升级流程

```text
外部公开案例观察
→ C级假设规则
→ 使用本项目素材复测
→ 复测成功，升级为 B级暂行规则
→ 多次成功，升级为 A级稳定规则
→ 复测失败，降级为风险规则或删除假设
```

## 2.2 规则记录格式

```text
规则名称：
证据等级：A / B / C / D
来源：本项目实测 / 外部公开案例 / 教程经验
适用范围：图片 / 视频 / 动作 / 构图 / 特效 / 光影 / 角色一致性
问题表现：
可能原因：
稳定写法：
风险写法：
禁止写法：
是否需要复测：是 / 否
```

# 4. 图片 Prompt 结构增补规则

## 4.1 推荐结构

图片 prompt 可以采用：

```text
连贯自然语言描述画面内容
+
短词语描述画面美学
```

即：

```text
第一段：主体 + 行为 + 环境 + 空间关系
第二段：风格 + 色彩 + 光影 + 构图 + 材质 + 情绪
第三段：关键限制
```

推荐模板：

```text
资源ID。画面中不要出现任何文字、编号或标识。

【画面内容】
用连贯自然语言描述主体、行为、环境、时代、空间关系。

【构图与机位】
景别、机位、主体位置、前中后景、引导线、留白、对称/非对称/中心/对角线/重复韵律等。
不要只写构图名，要写可见位置关系。

【主体细节】
人物身份、服装、道具、材质、姿态、朝向、视线。

【美学短词】
风格、色彩、光影、质感、画面类型。

【限制】
不要文字，不要水印，不要多余人物，不要主体变形。
关键帧不要爆炸、冲击波、强动态模糊、飞散碎片。
```

---

## 4.2 构图词必须具体化

不要只写：

```text
大师构图
黄金螺旋
强透视
中心构图
对角线构图
引导线构图
非对称构图
震撼构图
```

更稳写法：

```text
画面左下方的石阶和溪流形成引导线，视线沿 S 形曲线引向画面右上方的古城门。
主角位于画面右上黄金分割区域，左侧保留大量暗色留白。
城门位于画面中央偏下，左右两侧建筑对称排列，纵深道路通向城门。
前景残破石柱从画面左下斜向右上，形成对角线压迫感。
重复排列的屋檐从画面下方逐层向上递进，形成重复韵律和纵深。
```

规则：

```text
构图术语可以写，但必须跟可见空间关系绑定。
```

---

## 4.3 美学词可以用，但不能替代主体与结构

可用美学词：

```text
低饱和冷色调
暗金高反差
柔和弥散光
弱轮廓光
伦勃朗式明暗
极繁主义
水墨写实
厚涂 CG
2.5D cel shading
页游 CG 感
电影级光影
```

但不要只堆：

```text
震撼
大师级
高级感
视觉冲击
宿命感
故事感
极致细节
8K
杰作
```

原因：

```text
这些词能提升氛围，但无法稳定解决主体、空间、动作和构图问题。
```

# 5. Midjourney 式写法的适用边界

Midjourney 式结构可以借鉴：

```text
主体描述 + 场景描述 + 风格词 + 光影色彩 + 构图词
```

但在即梦 / Seedance / 可灵视频项目中，还必须增加：

```text
空间关系
镜头调度
首尾状态
运动路径
锁定元素
参考图职责
```

## 5.1 不建议把 MJ 参数混进即梦 prompt

不建议在即梦 / DreaminaBatcher prompt 中写：

```text
--ar 3:4
--s 250
--v 6
```

原因：

```text
比例、模型、分辨率、时长应由 manifest / CLI 字段管理。
prompt 只负责画面内容、构图、风格、动作和限制。
```

# 6. 关键帧特效规则

## 6.1 核心原则

```text
关键帧负责锁定：主体、姿态、空间、构图、光影基调。
视频 prompt 负责生成：爆发、扩散、飞散、流动、变身、冲击波、粒子运动。
```

这条规则适用于：

```text
图生视频
首帧视频
首尾帧视频
多模态参考视频
动作片
变身镜头
能量特效
爆炸冲击
空间裂隙
高速运动
```

---

## 6.2 关键帧可以保留的弱特效

关键帧可以有：

```text
弱雾气
静态冷光
微弱粒子
地面已有裂纹
角色身边淡淡光晕
远处静止火光
背景中的烟尘层
衣摆轻微受风
水面轻微反光
```

这些属于“环境状态”，可以帮助视频保持氛围。

---

## 6.3 关键帧尽量避免的动态特效

关键帧尽量不要直接放：

```text
爆炸正在爆开
冲击波已经成环形扩散
大量碎石正在飞
能量正在喷射
角色正在半变身
强动态模糊
闪电正在击中目标
刀光已经划满画面
敌人已经被震飞在半空
```

原因：

```text
这些是“时间过程”，如果烙进关键帧，图生视频容易变成海报抖动。
模型可能只让已有特效轻微晃动，而不是生成从无到有的动态过程。
```

---

## 6.4 变身 / 爆气 / 冲击波推荐拆法

关键帧写：

```text
角色站在破碎地面中心，衣摆微微抬起，身体周围只有淡淡能量光晕。
周围敌人仍在近处，地面有细小裂纹，但没有爆炸和强冲击波。
```

视频写：

```text
0-1秒：能量从角色脚下和胸口聚集，衣摆和头发被向上托起。
1-2秒：清晰的环形冲击波从角色脚下向四周扩散。
2-3秒：周围敌人、碎石和尘埃沿远离角色中心的方向被推出。
3-5秒：尘土和粒子继续向外翻滚，中心区域被清空，角色保持站立。
```

限制写：

```text
不要超级赛亚人式金发，不要自动变成动漫固定 IP 风格。
敌人必须沿半径方向向外飞，不要只是原地抖动。
```

# 7. 视频特效运动规则

## 7.1 视频 prompt 必须写特效运动路径

不要只写：

```text
粒子飞舞
冲击波爆发
陨石飞掠
星环流动
烟尘翻滚
```

更稳写法：

```text
特效从哪里出现
向哪个方向运动
速度如何
经过哪个空间层
最后停在哪里或消失在哪里
```

模板：

```text
特效起点：
特效方向：
特效速度：
前景 / 中景 / 背景关系：
是否遮挡主体：
结束状态：
```

---

## 7.2 宇宙飞行类示例

弱写法：

```text
快速向前运动，蓝色光粒子如流光环绕轨迹，陨石火球迎面散射飞掠，星球星环发光流动尾迹。
```

更稳写法：

```text
摄影机沿宇宙隧道中心轴高速向前推进。
蓝色光粒子从画面前方两侧掠过镜头，向画面后方拉成长线，形成向后流动的速度感。
陨石从画面右上前景高速掠过镜头，向画面左下方飞离，不要停在画面中心。
带星环的星球固定在画面左侧远处背景，星环边缘有稳定微光，尾迹向画面后方拖拽。
整个画面运动方向保持向前，不要随机旋转，不要变成静态粒子海报。
```

# 8. 同一 prompt 不同模块差异规则

## 8.1 必须记录模块

同一 prompt 在不同模块中可能差异很大，所以记录时必须标注：

```text
平台：即梦 / 可灵 / 其他
入口：网页端 / API / CLI
模块：text2image / image2image / text2video / image2video / frames2video / multiframe2video / multimodal2video
模型：具体模型版本，如果未知写 unknown
是否有参考图：有 / 无
是否有参考视频：有 / 无
是否有首尾帧：有 / 无
```

## 8.2 不允许跨模块直接套结论

不要这样判断：

```text
这个 prompt 在 text2image 里好看，所以 image2video 一定稳定。
这个 prompt 在可灵网页端成功，所以即梦 CLI 一定成功。
这个 prompt 在单图视频有效，所以多模态参考也有效。
```

必须这样记录：

```text
这个 prompt 在 text2image 中风格有效。
在 image2video 中需要重新写运动路径。
在 frames2video 中需要控制首尾帧差距。
在 multimodal2video 中需要写清每张参考图职责。
```

# 9. 图片评价表

评分统一：

```text
0 = 失败 / 没做到
1 = 部分做到
2 = 做得好
```

状态统一：

```text
LOCK = 可锁定参考
GOOD = 好但暂不锁定
MIXED = 局部成功局部失败，值得分析
FAIL = 失败但有分析价值
NOISE = 无价值，不分析
RETEST = 值得复测
```

| case_id | 来源 | 平台/模块 | 模型 | 用途 | prompt_id | 图片文件/链接 | 主体准确 0-2 | 构图准确 0-2 | 风格美感 0-2 | 光影色彩 0-2 | 细节材质 0-2 | 主要成功点 | 主要失败点 | 触发误解词 | 可提炼规则 | 状态 | 下一步 |
|---|---|---|---|---|---|---|---:|---:|---:|---:|---:|---|---|---|---|---|---|
| IMG-0001 | 外部社区 / 自测 | 即梦 text2image | unknown | 风格观察 / 项目资产 / 关键帧 | P-001 | path/link |  |  |  |  |  |  |  |  |  | GOOD / MIXED / FAIL | 保留 / 复测 / 改写 / 入规则库 |

## 9.2 图片评价维度

| 维度 | 判断问题 |
|---|---|
| 主体准确 | 主角、道具、场景是不是目标内容？有没有多出或缺失？ |
| 构图准确 | 机位、主体位置、前中后景、引导线、尺度关系有没有执行？ |
| 风格美感 | 画风、时代感、质感是否符合项目？有没有页游感、AI味、跑题？ |
| 光影色彩 | 色调、明暗、光源逻辑是否稳定？有没有手电筒光、乱光、过曝？ |
| 细节材质 | 武器、衣服、建筑、皮肤、毛发、石头等是否可信？ |
| 主要成功点 | 哪些词或结构真的起作用？ |
| 主要失败点 | 是主体错、构图错、动作错、空间关系错，还是文字乱码？ |
| 触发误解词 | 哪个词导致模型联想到不该联想的东西？ |
| 可提炼规则 | 能不能变成“以后应该怎么写 / 不该怎么写”？ |

## 9.3 图片总分

```text
图片总分 = 主体准确 + 构图准确 + 风格美感 + 光影色彩 + 细节材质
满分 10
```

建议处理：

```text
8-10 分：候选 LOCK 或 GOOD
5-7 分：多半是 MIXED，值得分析
低分但问题典型：FAIL，深分析
低分且没新信息：NOISE，不分析
```

## 10.2 视频评价维度

| 维度 | 判断问题 |
|---|---|
| 起始状态 | 第一帧主体是否已经存在？有没有突然冒出来？ |
| 主体一致 | 人物脸、服装、道具、场景有没有漂移或变形？ |
| 动作执行 | 动作有没有按时间线完成？方向、力量、停顿是否正确？ |
| 运镜执行 | 固定、推近、拉远、跟拍、低机位等有没有执行？有没有乱转？ |
| 空间调度 | 前景 / 中景 / 背景，人物站位、视线、运动路径是否清楚？ |
| 特效运动 | 粒子、冲击波、爆炸、烟尘是否真的“运动”，还是只在画面上抖？ |
| 时间连续 | 有没有跳变、溶解、主体瞬移、动作断裂？ |
| 主要成功点 | 哪一段最稳？哪个词有效？ |
| 主要失败点 | 是动作错、运镜错、特效僵、角色变、还是空间关系错？ |
| 触发误解词 | 哪些词让模型联想到错误类型，比如超级赛亚人、游戏海报、横版格斗？ |
| 可提炼规则 | 能否变成后续视频 prompt 的稳定规则 / 风险规则 / 禁止规则？ |

## 10.3 视频总分

```text
视频总分 = 起始状态 + 主体一致 + 动作执行 + 运镜执行 + 空间调度 + 特效运动 + 时间连续
满分 14
```

建议处理：

```text
11-14 分：候选 LOCK 或 GOOD
7-10 分：多半是 MIXED，值得分析
低分但问题典型：FAIL，深分析
低分且没新信息：NOISE，不分析
```

# 12. 选图 / 选视频分析策略

不要全量深分析。

推荐比例：

```text
全部结果：保留原文件
全部结果：做轻记录
15%-25%：做深分析
1%-5%：进入 locked_refs
每 20-50 条案例：归纳一次规则
```

## 12.1 值得深分析的好图 / 好视频

```text
能直接变成风格模板的。
好看但有明显缺陷的。
某个词或结构明显有效的。
能为项目复用的。
和当前项目镜头类型高度相关的。
```

## 12.2 值得深分析的坏图 / 坏视频

```text
坏得有规律。
坏得会反复影响项目。
暴露了空间关系、动作方向、特效运动、构图执行问题。
和项目关键镜头有关。
```

## 12.3 不值得深分析的结果

```text
普通中庸图。
单纯画质差但没有新信息。
手指小错、脸轻微崩等模型常见问题。
重复成功图。
重复失败图。
和项目无关的漂亮废片。
```

一句话：

```text
好图记录“为什么能用”。
坏图记录“为什么不能用”。
中庸图只记录“它存在过”。
```

# 13. 外部案例可提炼的常见风险规则

以下规则来自外部公开案例观察，默认先标 C级，必须经本项目复测后才能升级。

---

## 13.1 “巨大 / 遮天蔽日 / 震撼”不能单独使用

问题：

```text
模型可能只生成普通大主体，或只表现成遮光效果，没有真实尺度压迫。
```

更稳：

```text
主体从画面左右边缘延伸到画面之外。
主体覆盖整个上半天空，只露出少量天空缝隙。
画面底部有渺小人类剪影、建筑、车辆、桥梁、塔吊等比例尺。
低机位贴近地面仰拍，主体像巨型飞机从头顶压过。
```

---

## 13.2 “航拍视角”不能单独使用

问题：

```text
只写航拍，模型可能生成高处看城市，但机位高度、俯仰角、视觉轴线不稳定。
```

更稳：

```text
摄影机位于城市上方约百米高度，斜向下俯拍。
画面下方是长街，中部是城门和门楼，上方是层层递进的山城建筑，最上方是夜空和远山。
建筑层级从画面下方向上递进，形成清晰纵深。
```

---

## 13.3 建筑蓝图中的文字不可靠

问题：

```text
模型可以生成专业蓝图感、标注线、编号框，但汉字容易扭曲。
```

规则：

```text
重要文字后期添加。
生图阶段只生成结构、引线、编号框、空白标注牌，不要求模型生成可读汉字。
```

---

## 13.4 武器空间关系必须写清

问题：

```text
只写“手拿双剑挥舞”，模型可能让刀刃穿过身体，或刀柄位置正确但刀刃方向错误。
```

更稳：

```text
双剑分别握在左右手中。
刀柄位于手掌内，刀刃从手掌外侧向画面两侧延伸。
刀刃位于身体前方或身体外侧，不穿过胸口，不贴在身体内部。
```

---

## 13.5 “随机风格 / 随机镜头 / 随机人物”容易生成漂亮废片

问题：

```text
画面可能很好看，但主题空泛，不服务叙事目标。
```

风险词：

```text
随机元素
随机风格
随机镜头
随机人物
巨大奇幻空间
宏大
故事感
神秘
随机模糊
```

修正：

```text
必须指定主体、主体位置、空间结构、尺度关系、镜头轴线、运动方向。
```

---

## 13.6 主体承载关系必须写清

问题：

```text
如果没有写角色站在天上、云上、地上、柱子上或空中，模型通常默认让主体踩在地面。
```

更稳：

```text
神兽悬浮在云层上方，四爪不接触地面。
脚下只有翻滚云雾，没有实体地面。
画面下方远处才是山脉和地面，作为背景比例尺。
```

或：

```text
神兽四爪稳稳踩在黑色岩地上，爪下有火焰和碎石。
```

二者必须明确其一。

---

## 13.7 高训练频概念会触发强联想

问题：

```text
“超级状态 / 爆气 / 变身”可能触发超级赛亚人式发型、金发、动漫 IP 联想。
“梵高 / 抽象星空”可能触发固定的旋涡星空。
```

修正：

```text
写清不要金发，不要超级赛亚人式尖刺发型。
如果要避开固定艺术家风格，改写为“旋涡笔触的深蓝星空”或“厚重短笔触的夜空”，不要直接依赖艺术家名。
```

# 14. Pro / Think 模式使用规则

Pro / Think 模式不建议用于逐张图写观后感。

更适合：

```text
每 20-50 条案例做一次聚类。
提炼稳定词、风险词、禁止词。
对比好图和坏图的 prompt 结构差异。
把 deep cases 归纳成 source 规则。
为下一轮复测设计实验组。
```

推荐提问方式：

```text
这里有 30 条外部案例观察，请按：
稳定词、风险词、禁止词、构图问题、动作问题、特效问题、模块差异
帮我归纳出可复测规则。
```

不推荐：

```text
这张图怎么样？
这张图好不好？
每张都详细评价一下。
```

# 16. 一句话总结

```text
外部案例用来训练判断力，不用来复制素材；
图片 prompt 先锁主体、空间、构图、光影；
视频 prompt 再写动作、运镜、扩散、飞散和时间线；
所有外部观察先进入 C级假设，复测成功后才能升级为项目 source 稳定规则。
```
