# 1. 规则 A：轻武器抗重武器时，必须写受力反应

## 1.1 问题表现

生成冷兵器打斗时，模型可能会让轻武器轻松挡住重武器，例如：

```text
小刀轻松挡住大棒；
长刀轻松挡住重刃枪；
格挡方身体没有任何受力反应；
武器接触点没有重量和冲击。
```

这会导致画面看起来“不符合物理”。

## 1.2 原因

只写：

```text
B 用长刀格挡 A 的重刃枪。
```

模型会理解为“动作成立”，但不会自动补足人体受力链。

## 1.3 稳定规则

如果出现：

```text
轻武器格挡重武器
小体型角色抵挡大力量攻击
快速角色硬抗重型角色攻击
```

必须写清：

```text
手腕被压低；
小臂和肩膀承受冲击；
上半身被带偏；
脚步向后滑退；
膝盖下沉缓冲；
武器接触点有短暂停顿和震颤；
火花、碎屑、水花或尘埃作为环境反馈。
```

## 1.4 可复用 Prompt 模块

```text
轻武器格挡重武器时，格挡方不能轻松挡住。
接触瞬间，格挡方手腕被压低，小臂和肩膀明显承受冲击，上半身被力量带偏，双脚向后滑退半步，膝盖下沉缓冲。
武器接触点出现短暂停顿和震颤，火花/水花/碎石从接触点爆开。
不要让格挡看起来毫无重量，不要让轻武器像没有阻力一样挡住重武器。
```

# 2. 规则 B：变身镜头必须锁定“对视轴线”

## 2.1 问题表现

变身镜头中，角色容易从对峙状态变成正面展示：

```text
两人变身时转向镜头；
两人面对观众摆姿势；
原本左右对峙关系消失；
变身后不再互相看向对方。
```

## 2.2 原因

模型会把“变身”理解为“角色展示图”或“英雄登场海报”，自动把主体转向镜头。

## 2.3 稳定规则

如果变身前是对峙关系，变身过程中必须锁：

```text
A 始终在左侧面向右侧；
B 始终在右侧面向左侧；
两人始终互相对视；
身体朝向彼此；
不要转向镜头；
不要面向观众摆姿势；
不要正面海报式站立。
```

## 2.4 可复用 Prompt 模块

```text
变身全过程中，角色A始终位于画面左侧并面向右侧，角色B始终位于画面右侧并面向左侧。
两人从人形到终极形态始终互相对视，身体朝向彼此，不要转向镜头，不要面向观众摆姿势，不要变成正面海报式展示。
```

# 3. 规则 C：巨物尺度不能只写“巨大”

## 3.1 问题表现

写“巨大怪物”“巨型机甲”时，模型经常只生成：

```text
比人略大一点；
普通怪物尺度；
小型外骨骼；
没有环太平洋级巨物感；
周围环境缺乏比例尺。
```

## 3.2 原因

“巨大”“超大”“山一样”是抽象形容词，不足以稳定生成尺度。

## 3.3 稳定规则

巨物尺度必须绑定可见比例尺：

```text
建筑；
桥梁；
塔吊；
船坞；
高墙；
断桥；
巨型残骸；
海浪；
车辆；
人类剪影。
```

必须写清：

```text
变身后高度远超周围建筑；
船坞和塔吊在它面前显得渺小；
断桥、巨构残骸、墙体作为比例尺；
从人类尺度跃迁为山岳级 / 环太平洋级巨物。
```

## 3.4 可复用 Prompt 模块

```text
变身完成后，主体成为山岳般巨大的终极形态，远超周围建筑、桥梁、塔吊、船坞和巨构残骸。
周围结构在它面前显得非常渺小，画面必须清楚显示尺度对比。
不要只变大一点，不要保持接近人类体量。
```

# 4. 规则 D：空间裂隙必须用具体视觉词，不用抽象词

## 4.1 问题表现

只写“空间撕裂”时，模型可能生成：

```text
普通爆炸；
场景整体碎裂；
地面或墙体破碎；
发光传送门；
抽象能量波；
物体被撕开而不是空间被撕开。
```

## 4.2 原因

“空间撕裂”是抽象概念，模型无法稳定判断具体视觉形状。

## 4.3 稳定规则

空间裂隙要写成可见对象，必须包含：

```text
裂隙形状；
边缘材质；
内部颜色；
与原场景差异；
周围空气扭曲；
吸附碎石/火花/尘埃；
禁止普通爆炸和发光传送门。
```

## 4.4 可复用 Prompt 模块

```text
原本完整的现实场景中央，突然出现一道竖直的不规则黑色裂隙。
裂隙像有人用锋利工具在现实画面上硬生生划开一道伤口。
裂隙边缘像破碎镜面一样尖锐、分叉、碎裂，带有细小裂纹。
裂隙内部是纯黑、无底、与当前场景完全不同的空洞，不显示原场景纹理。
它看起来像在写实画面上被粗黑墨水强行涂开的一道不规则裂口，边缘轻微抖动，周围空气发生扭曲。
附近碎石、火花、雨水、尘埃被吸向裂隙边缘。
不要做成普通爆炸，不要做成发光传送门，不要让整个场景整体裂成碎片。
```

# 5. 规则 E：快节奏打斗不等于整段慢动作

## 5.1 问题表现

为了追求“拳拳到肉”，模型可能生成：

```text
整段慢动作；
动作很重但节奏很拖；
每一招都停下来展示；
打斗像回合制；
缺少连续攻防。
```

## 5.2 原因

“重击”“48fps”“低动态模糊”“慢动作”等词容易让模型把整段动作升格慢放。

## 5.3 稳定规则

快节奏打斗应写成：

```text
整体真实速度；
快速出招；
快速格挡；
快速闪避；
快速反击；
不停顿；
只有关键命中瞬间有极短 hit-stop。
```

## 5.4 推荐时间控制

```text
普通动作：真实速度
关键命中：0.2-0.3 秒极短顿挫
重武器碰撞：0.2-0.4 秒极短停顿
禁止：整段慢动作、长时间升格、持续慢放
```

## 5.5 可复用 Prompt 模块

```text
整体节奏快速紧凑，保持真实速度，不要长时间慢镜头。
双方持续抢攻、格挡、闪避、反击，招式连续衔接，不要每打一招就停顿。
每个镜头至少包含一个连续交换回合：进攻 -> 格挡/闪避 -> 反击 -> 再追击。
只有关键命中瞬间保留 0.2 秒左右极短 hit-stop，其余动作保持真实速度。
不要整段升格，不要回合制展示，不要站桩互殴。
```

# 6. 规则 F：出现横版游戏感时，必须强制改机位描述

## 6.1 问题表现

打斗镜头容易生成：

```text
两人左右站在同一水平线；
纯侧面平拍；
像横版格斗游戏；
像舞台侧视图；
画面缺乏纵深；
镜头一直平行于角色移动。
```

## 6.2 原因

如果只写“A 在左、B 在右”，模型会自动形成横版游戏式构图。

## 6.3 稳定规则

当项目需要电影感动作镜头时，要明确禁止：

```text
不要横版游戏式纯侧面平拍；
不要舞台侧视图；
不要让两人始终贴在同一平面上；
不要固定横向平移跟拍到底。
```

同时指定电影机位：

```text
三分之二侧前方；
三分之二侧后方；
低机位斜侧跟拍；
近身手持跟拍；
肩位手持跟拍；
低角度推近；
宽景缓慢推进；
俯冲跟随；
落地低机位迎接冲击。
```

## 6.4 可复用 Prompt 模块

```text
不要横版游戏式纯侧面平拍，不要像舞台侧视图。
使用电影化机位：三分之二侧前方、三分之二侧后方、低机位斜侧跟拍、近身手持跟拍或低角度推近。
人物前后关系和景深要清楚，画面要有纵深，不要让两人一直像贴在同一个平面上。
镜头可以轻微手持、轻微推近或短距离跟移，但不要僵硬锁死在平行侧拍。
```

# 7. 高速坠落补充规则

## 7.1 问题表现

“下落”经常被模型生成：

```text
慢慢飘下；
缓慢坠落；
像漂浮；
像滑翔；
没有重力冲击；
落地不够重。
```

## 7.2 稳定规则

如果需要高速坠落，要明确写：

```text
瞬间失去支撑；
被重力猛地向下拽走；
几乎垂直高速坠下；
没有悬停；
没有漂浮；
没有慢慢掉落；
短时间内重重砸入下方场地；
落地伴随碎石、水花、烟尘爆开。
```

## 7.3 可复用 Prompt 模块

```text
结构突然崩断后，角色不是缓慢下落，而是瞬间失去支撑，被重力猛地向下拽走。
下坠速度很快，几乎垂直直坠，没有悬停，没有滑翔，没有慢慢漂落。
短时间高速下落后重重砸入下一个场地，伴随巨大冲击、水花、碎石和烟尘爆开。
落地后身体因冲击出现明显缓冲动作，然后立刻恢复战斗。
```
