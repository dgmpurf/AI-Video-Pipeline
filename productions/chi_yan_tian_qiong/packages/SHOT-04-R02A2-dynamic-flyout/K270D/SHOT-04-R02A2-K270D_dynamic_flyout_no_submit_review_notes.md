# SHOT-04 R02A2 K270D Dynamic Fly-Out No-Submit Review Notes

## Purpose

K270D creates no-submit package materials for SHOT-04 R02A2 Beat B: the dynamic fly-out / explosive displacement beat that should follow the existing Variant A CONTACT_HITSTOP_SHORT static/contact Beat A.

## Why B3 Is Primary

B3 result_only_flyout_after_hit begins after the hit-stop and does not ask the model to solve precise contact again. This directly addresses the K269Z human insight that the current material can provide static contact, but a separate fast displacement beat is still needed.

B3 should focus entirely on Shuangji already flying or sliding backward with readable distance, acceleration trails, wet-floor splash, rain spray, and debris response. This gives the generation a narrower goal than full close-combat choreography.

## Why B1 Is Secondary

B1 wide_side_flyout_result_shot uses side or three-quarter wide framing so displacement distance and speed can be read clearly. It is a useful backup if B3 becomes too result-only or loses the relation to Fenshou's follow-through.

B1 is secondary because it asks for more staging control: Fenshou left/near side, Shuangji launched right/backward, wide framing, and readable environment feedback all at once.

## Why B2 Is Not Created Now

B2 impact_detail_then_acceleration could be visually strong, but it requires a precise timing shift from slow impact detail to fast launch inside one generated shot. That risks reintroducing the same sustained-contact and timing-control problems that K269Z identified.

## Why B4 Is Not First Requirement

B4 wall_or_column_impact_result is not created now because prior wall-impact routes were brittle. Wall or column impact may become a later optional extension, but the first dynamic package should prioritize visible body travel and speed rather than collision complexity.

## Why No Current Cut Is Final

CONTACT_HITSTOP_SHORT is the best existing static/contact candidate, but it does not show snap-back, fast displacement, or fly-out by itself. CONTACT_HITSTOP_PADDED is only an alternate static beat. LATE_REACTION_EVIDENCE is reaction reference only. CONTACT_EVIDENCE_LONGER reintroduces sustained pressure.

## Why Future Submit Requires Separate Authorization

K270D is no-submit package creation only. It does not authorize Dreamina, submit, query, download, retry, resubmit, batch, media creation, final, or lock. Any future submit requires a separate human authorization phase after package review.

## Static/Dynamic Contrast Link

K269Z's core lesson is that impact force comes from contrast: a static hit-stop/contact beat followed by a dynamic high-speed fly-out over visible distance. K270D packages are designed to provide that Beat B while preserving the existing Beat A as supporting edit material only.
