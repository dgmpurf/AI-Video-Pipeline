## 9. Proposed Route-State Transition Model

No state below implicitly activates provider authority. Every transition is evidence-bound and atomic.

| State | Required transition evidence | Allowance and authority semantics |
| --- | --- | --- |
| `PLANNED_NO_LIVE_BINDING_NOT_CREATED` | This design report and human acceptance | No route file exists; all authority false. |
| `BINDING_CREATED_NOT_AUDITED` | Future binding and route-state files, hashes, and creation report | No preflight or provider authority. |
| `BINDING_AUDITED_NOT_PREFLIGHTED` | Independent audit decision with zero blocking/material findings | No canary or provider authority. |
| `PREFLIGHT_DESIGNED_NOT_RUN` | Fresh checkpoint-specific preflight plan bound to audited artifacts | Commands remain unauthorized. |
| `PREFLIGHT_PASSED_AWAITING_SUBMIT_AUTHORIZATION` | Re-readable evidence manifest proving every static, runtime, account, credit, and duplicate gate | Submit allowance unconsumed; provider authority false. |
| `PREFLIGHT_SAFE_BLOCKED` | Persisted exact stop reason and failed gate evidence | No provider invocation; allowance remains unconsumed, but a new checkpoint-bound decision is required. |
| `SUBMIT_AUTHORIZED_NOT_CONSUMED` | Fresh exact human authorization plus verified checkpoint and operation maxima | Exactly one submit is eligible, not yet invoked. |
| `SUBMIT_INVOCATION_STARTED_ALLOWANCE_CONSUMED` | Atomically persisted invocation reservation and duplicate snapshot immediately before runner start | Allowance becomes consumed permanently; submit invocation count becomes `1`; no second submit. |
| `SUBMITTED_PENDING_SEPARATE_QUERY_AUTHORIZATION` | Exact nonterminal parser proof, new IDs, one task, exact 70-credit reconciliation, and re-readable evidence | Route remains open only for a separate query decision; query/download authority remains false. |
| `CLOSED_SUBMIT_FAILED` | Failed status or provider terminal failure evidence | Route closed; no query/download/retry/resubmit. |
| `CLOSED_AMBIGUOUS_SUBMIT_RESULT` | Unknown or conflicting status/identity evidence | Route closed and allowance remains consumed. |
| `CLOSED_PREQUEUE_UPLOAD_FAILURE` | Exact upload marker, no provider task evidence, and zero-charge reconciliation | Route closed; returned handle quarantined. |
| `CLOSED_TASK_CREATION_PROOF_FAILURE` | Missing submit ID/logid, unproven task, count mismatch, or unknown classification | Route closed. |
| `CLOSED_CREDIT_RECONCILIATION_FAILURE` | Missing, nonnumeric, non-70, or unexplained credit evidence | Route closed even if task creation was otherwise proven. |
| `CLOSED_BOUNDARY_VIOLATION` | Any command/count/path/authority/protected-file violation | Route closed and escalation required. |
| `CLOSED_TERMINAL_SUBMIT_RESULT` | Parser reports terminal success/failure rather than an accepted nonterminal state | Route closed; no automatic query or download. |
| `CLOSED_RUNNER_INVOCATION_FAILURE` | Runner fails after allowance reservation | Route closed; invocation failure record preserved. |
| `CLOSED_EVIDENCE_PERSISTENCE_FAILURE` | Any required envelope, parsed record, transition, or fsync/re-read failure | Route closed; no later action. |

The allowance is consumed immediately before or atomically with starting the first provider invocation and can never be restored. A pre-provider preflight block does not consume it. Every failure or ambiguity after the provider boundary closes the route. Original F07 state is never mutated by any F07R transition.

- created task count exactly `1`;

Success does not increment fixed completion, change original F07 counts, authorize query/download, create review artifacts, or set final/lock.

Any unknown, failed, terminal, contradictory, incomplete, ambiguous, upload-failure, task-proof, credit, boundary, runner, or persistence result after invocation:

- leaves the allowance consumed;
- permits no second submit under the identity;
- closes the route with one exact terminal reason;
- preserves sanitized evidence;
- authorizes no query or download;
- recommends, but does not create or authorize, fallback `CAL001-F07R-MEDIAOPT-P1-R1`.
